# /explore

## 목적

5개 이상의 파일을 읽어야 하거나 코드베이스를 폭넓게 조사해야 할 때,
`codebase-explorer` sub-agent에 위임해 메인 컨텍스트를 보호한다.

## 사용 예시

```
/explore note/ 모듈에서 마크다운 직렬화가 어떻게 동작하는가
/explore Reminders 동기화 충돌 해결 로직이 어디 있는가
/explore canvas 렌더링 파이프라인 전체 흐름
```

## 실행 순서

### Step 1: 탐색 질문 명확화

탐색 목표를 한 문장으로 정의:
- 무엇을 알고 싶은가?
- 어떤 파일 범위인가?
- 무엇을 찾으면 완료인가?

### Step 2: codebase-explorer sub-agent 위임

`codebase-explorer` sub-agent에 다음을 전달:
- 탐색 질문 (1문장)
- 예상 범위 (모듈명 또는 파일 패턴)
- 찾아야 할 구체적 항목

### Step 3: 결과 수신

sub-agent가 반환하는 구조화된 요약만 메인 컨텍스트에 보유.
원본 파일 내용은 수신하지 않는다.

### Step 4: 필요시 후속 탐색

첫 탐색 결과에서 추가로 봐야 할 것이 있으면 동일 패턴으로 반복.
각 탐색 결과는 **요약 형식**으로만 누적.

## 완료 기준

- 탐색 질문에 답할 수 있는 핵심 사실을 확인했다
- 메인 컨텍스트에는 요약만 있고 파일 전체 내용은 없다
- 다음 구현 단계를 결정할 수 있다
