# /fix-bug

## 사전 조건

- 버그 설명 또는 재현 절차
- `make setup` 완료

## 실행 순서

### Step 1: 버그 이해

1. 관련 코드 파일을 읽는다 (`ARCHITECTURE.md`에서 담당 모듈 확인).
2. 기존 테스트 파일(`*.test.ts`)을 읽는다.
3. 버그의 기대 동작 vs 실제 동작을 명확히 정의한다.

### Step 2: 재현 테스트 작성

관련 `*.test.ts` 파일에 실패하는 테스트를 추가한다.

```bash
npm test -- --reporter=verbose  # 새 테스트가 실패하는지 확인
```

재현이 불가능한 경우: 이유를 기록하고 Step 3으로 진행 후 증거를 남긴다.

### Step 3: 수정 구현

최소 범위로 변경한다. 관련 없는 코드는 건드리지 않는다.

### Step 4: 검증

```bash
make verify  # 새 테스트 통과 + 기존 테스트 회귀 없음 확인
```

Electron/UI 관련 버그:
```bash
make dev-safe  # 격리된 환경에서 동작 확인
```

### Step 5: 문서 갱신 (해당 시)

- 버그가 설계 결정의 모호함에서 비롯되었으면 `docs/decisions/`에 기록.
- `docs/product/FEATURE-INVENTORY.md` 관련 항목 갱신.

## 완료 기준

- 재현 테스트가 존재하고 통과한다.
- `make verify` 종료 코드 0.
- 관련 없는 테스트 회귀 없음.

## 완료 보고 형식

```
수정 내용: [한 줄 설명]
재현 테스트: [파일:줄번호]
변경 파일: [목록]
make verify: 종료 코드 0 (X passed)
런타임 확인: [해당 시 설명]
남은 위험: [있다면 설명]
```
