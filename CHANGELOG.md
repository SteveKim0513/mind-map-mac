## [0.7.15] - 2026-06-24

### 버그 수정
- **이미지 외부 저장 누락 (EditorToolbar)** — 툴바 파일선택이 `fileToDataUrl`만 호출하고 `imagesWrite` IPC를 건너뛰어 base64가 .md에 직렬화되던 문제. `NoteEditor.insertImages`를 `onInsertImages` prop으로 전달해 paste/drop과 동일 경로 사용.
- **NotePopup 이미지 미표시** — `renderMarkdown`이 `data:image/` 접두사만 허용, 상대경로(`./note.assets/…`)는 텍스트로 처리. 팝업 마운트 시 IPC로 상대경로를 dataUrl로 치환 후 렌더링.
- **이름 변경 시 .assets 미갱신** — `fs:rename`이 `.md` 파일만 이름변경, `.assets` 폴더를 그대로 두어 이미지 로드 실패. 이름변경 후 `oldStem.assets` → `newStem.assets` 이름변경 추가.
- **파일 이동 시 .assets 미이동** — `fs:move`가 `.md`만 이동. 이동 후 `.assets` 폴더도 함께 이동.
- **휴지통 이동/복원/삭제 시 .assets 미처리** — `trash:move/restore/deleteOne/empty` 모두 `.assets` 폴더를 누락. 각 핸들러에 동반 처리 추가.
- **파일명 충돌 (동일 초 다중 삽입)** — 초 단위 타임스탬프로 이름 생성 시 같은 초에 여러 장 삽입하면 덮어씀. 밀리초(3자리) 추가.
- **사이드바에 .assets 폴더 노출** — `walk()`가 `.assets` 접미사 폴더를 필터링하지 않아 빈 폴더로 노출. 접미사 필터 추가.

## [0.7.8] - 2026-06-22

### 버그 수정
- **삭제 후 파일이 다시 나타나는 문제** — 노트 편집 중 삭제 시 800ms autosave 타이머가 `trashItem` 이후에 만료되어 `fs.writeFile`이 삭제된 경로에 파일을 재생성하던 레이스 컨디션. 삭제 전 `flushSaves`로 dirty 상태를 초기화해 타이머를 취소.
- **삭제 실패 시 탭·사이드바가 사라지는 문제** — `trashItem` 진짜 실패(권한 오류 등) 시에도 catch를 통과해 `onDeleted`가 호출되어 탭이 닫히고 사이드바에서도 사라지던 문제. 이제 실패 시 toast 표시 후 return, UI는 그대로 유지.
- **일괄 삭제(deleteMarked) 동일 수정** — 개별 삭제와 동일한 flushSaves + 실패 처리 적용.
- **NotePopup(peek 팝업) 미닫힘** — 사이드바에서 파일 삭제 시 열려 있던 peek 팝업이 닫히지 않던 문제.
- **삭제 실패 원인 로그** — `shell.trashItem` 실패 시 main 프로세스 로그에 기록.

## [0.7.6] - 2026-06-22

### 버그 수정
- **노트/파일 삭제 후 탭이 안 닫히는 문제** — `removeNode`/`deleteMarked`에 try-catch가 없어서 파일이 이미 없으면 `onDeleted()` 미호출 → 탭·사이드바 잔류. 이제 파일 삭제 실패(ENOENT 포함)에도 UI 정리를 보장.

- **ENOENT 탭 열기 오류** — 삭제된 파일 경로를 가진 탭을 열려 할 때 toast만 뜨고 탭이 남아있던 문제. 이제 ENOENT 시 해당 탭을 닫고 사이드바를 갱신.

## [0.6.0] - 2026-06-16

### 새 기능
- **노트 ↔ 노트 연결** — 본문에서 `[[노트 제목]]`으로 노트를 잇습니다. `[[` 입력 시 자동완성이 뜨고, 없는 노트는 그 자리에서 만들어 연결합니다. 링크를 누르면 **미리보기(peek)**가 떠 보던 노트를 잃지 않고, "열기"나 **⌘-클릭**으로 반대쪽 화면에 나란히 엽니다. 아직 없는 노트로의 링크는 흐리게 표시됩니다.

### 개선
- **탭 순서 드래그** — 탭을 끌어 순서를 바꾸고 좌우로 옮길 수 있습니다(드래그가 되지 않던 문제 해결).

- 노드에서 "새 노트 만들어 연결"이 조용히 실패하던 문제 — 이제 실패 시 알려주고, 한 단계가 실패해도 나머지는 진행됩니다.
- 노트 이름을 바꾸면 다른 노트의 `[[ ]]` 링크가 깨지던 문제 — 참조하는 모든 노트에서 자동으로 갱신됩니다.
