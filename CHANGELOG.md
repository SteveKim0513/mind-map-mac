## [0.11.1] - 2026-07-17

### 수정 — 집중 세션 데이터 안정화
- **집중 종료 시 노트 본문·기록이 유실되던 문제 수정** (스펙 §14-I) — 종료 직전 타이핑한 작업 로그가 자동저장 경합으로 사라지거나 세션이 "진행 중"으로 고착되던 것을, 종료 전 `flushSaves`로 라이브 본문을 먼저 디스크에 쓴 뒤 end를 스탬프하도록 고쳤다. `sessions()`는 `sessionId`로, `buildNoteIndex`는 경로로 dedup(중복 집계 방지). 고착된 옛 세션은 노트 배너 "종료 처리"로 복구.

### 변경 — 집중과 일정 분리 + 자동 배치
- **집중 게이트 해제** ([결정 0015](docs/decisions/0015-focus-independent-of-schedule.md)) — 할 일(todo) 노드에서 **완료·일정 지정·집중이 각각 독립 기능**. 집중은 더 이상 일정을 요구하지 않는다(0011 §3 완화). 툴바에 "일정 지정" 버튼 추가.
- **일정 없는 노드에서 집중하면 그 시각이 스케줄로 자동 배치** — 시작 시각을 `scheduleAt`으로 잡아 캘린더에 즉시 뜨고, 종료 시 실제 집중 분(`durationMin`)으로 블록 크기를 맞춘다.

### 변경 — 캘린더 UX
- **일정 클릭 = 미리보기 먼저** — 맵을 바로 열지 않고 그 노드+하위를 읽기전용 아웃라인(말줄임 없는 전문)으로 보여주고, 다시 누르면 우측 분할로 맵. (일=서랍/월=아래 패널/주=팝업)
- **"일정 추가" 진입점 일관화** — 일/주/월 **헤더 같은 자리**에 버튼 하나. 피커의 종일/시간 토글 항상 노출(주간 슬롯은 시각 프리필), Escape로 항상 닫힘, 빈 곳 hover 힌트.
- **실행은 일간 뷰에만** — 월·주 뷰에서 집중·완료 인라인 버튼 제거(조망·계획에 집중). 일간 집중·완료 버튼은 아이콘+라벨 pill로 통일.
- **겹치는 일정 UI 개선** — 나란히 열 분할 + 사이 간격, 겹칠 땐 중복 시각 숨김, hover 시 전체 폭으로 확대되어 읽힘.
- **월간 셀 스크롤** — "+N" 축약 없이 셀 안에서 스크롤. 주간 시간표 정렬 버그 수정(고정 px 스케일).

### 변경 — 생각·실행 경계 일관화
- 실행 기능(완료·일정·집중)이 **모든 표면에서 할 일 노드에만** 노출되도록 정리 — ⌘K 명령 팔레트(일반 노드엔 "할 일로 전환"만)와 누적 집중시간 칩(할 일 노드에만) 포함.

## [0.11.0] - 2026-07-16

### 새 기능 — 할 일(todo) 노드
- **생각과 실행을 가르는 "할 일" 상태** — 완료·일정·집중은 이제 **할 일 노드에서만** 보인다. 일반 노드(순수 생각)는 메뉴가 가벼워지고 "할 일로 전환"만 노출한다. `⌘Enter`로 일반→할 일 전환/할 일→완료 토글, 일정을 잡으면(수동·`@내일 3시`·캘린더) 자동으로 할 일이 된다. 할 일 노드는 제목 앞 체크박스 글리프로 구분([결정 0014](docs/decisions/0014-todo-node.md)). 데이터 `version` 무범프(backfill로 하위호환).

### 새 기능 — 캘린더 UX 개선
- **일정 클릭 = 미리보기 먼저** — 곧바로 맵을 열지 않고 그 노드+하위를 **읽기전용 미니뷰**로 미리본다(일간=카드 아래 서랍 / 월간=그리드 아래 패널 / 주간=✕ 닫기 팝업). 미리보기의 스케줄 노드를 다시 누르면 **오른쪽 화면 분할**로 맵이 열린다.
- **통합 일정 생성** — 주간 빈 칸·일간 "일정 추가"·월간 셀 "+"에서 **기존 노드를 검색해 일정을 잡거나**, "새로 만들기"로 **"오늘의 생각"**(⌥Space 캡처와 같은 인박스)에 새 일정을 담는다. 일·월은 **종일이 기본 + 시간 옵트인(HH:mm)**, 주간은 클릭한 칸 시각.
- **일간 집중·완료 버튼 통일** — 아이콘+라벨 pill 한 쌍으로.
- **월간 셀 스크롤** — 일정이 많아도 "+N" 축약 없이 셀 안에서 스크롤로 전부 표시.

### 수정
- **주간 시간표 정렬** — 창이 커지면 일정 블록이 왼쪽 시간 눈금과 어긋나던 문제를 고정 px 스케일로 수정.

> v0.9.1 태그는 CI의 E2E 단계에서 실패해(e2e 창을 화면 밖으로 옮기는 격리 로직이 실제 창 포커스를

### 새 기능
- **캘린더 탭 신설** — 사이드바 "오늘" 오버레이를 일/주/월 보기가 가능한 탭으로 승격했다. 기본 진입 뷰는 일간(오늘)이고, 주간은 7일 그리드에 시간순 칩, 월간은 전통적인 달력 그리드(일정 개수 배지)로 보여준다. 어느 뷰에서든 칸을 클릭하면 그 날짜의 일간 뷰로 전환된다. 사이드바 상단은 "캘린더 · 즐겨찾기" 2개만 남기고, "집중 기록"은 하단 관리 화면 묶음으로, "최근 수정"은 설정(⌘,) 안으로 옮겼다.

### 개선
- **집중 세션은 이제 일정이 지정된(스케줄) 노드에서만 시작할 수 있다** — 실행은 "정리된 것 중 일정이 잡힌 걸 완수하는 흐름"이라는 정의에 맞춰, 일정 없는 노드에서는 선택 툴바의 집중 버튼 자체가 안 보이고 우클릭 메뉴는 비활성 + 안내 문구를 보여준다.
- **우클릭 메뉴가 구조/정리/실행/통찰 4그룹으로 재편됐다** — 완료·일정·집중이 "실행" 그룹으로, 색·아이콘·메모·링크가 "정리" 그룹으로 모였다.
- **일정 팝오버에 "지금 집중 시작" 버튼 추가** — 날짜·시간을 정한 바로 그 화면에서 곧바로 집중 세션으로 넘어갈 수 있다.

### 버그 수정
- 설정에서 "최근 수정"을 열 때 설정 창이 그 위에 그대로 남아 클릭을 가로채던 문제 수정 — 이제 "최근 수정"을 열면 설정 창이 먼저 닫힌다.

## [0.9.0] - 2026-07-15

### 새 기능
- **커맨드 팔레트(⌘K)에서 선택한 노드에 바로 동작을 실행할 수 있습니다** — 지금까지는 노드를 검색해 선택·포커스만 시켰는데, 이제 "선택 노드: 일정 설정" · "색상 — 보라" · "링크 추가" · "노드에 노트 연결" · "집중 세션 시작" 같은 실행 가능한 명령이 함께 뜬다. 즐겨찾기·최근 수정 보기도 팔레트 명령으로 등록.
- **사이드바 파일/폴더 행에 우클릭 메뉴가 생겼습니다** — 즐겨찾기·이름 변경·삭제를 캔버스 노드·탭과 같은 방식(우클릭)으로도 실행할 수 있다. 호버 아이콘은 그대로 있음.

### 개선
- **선택 툴바가 6개 버튼으로 정리됐습니다**(색상·완료·메모·집중·자식추가·삭제) — 일정·링크·노드에 노트 연결은 값이 있을 때 나타나는 칩과 우클릭 메뉴로만 접근한다. 같은 기능이 칩·아이콘·메뉴 텍스트 세 가지 다른 모습으로 흩어져 있던 걸 정리.
- **색상 선택 UI를 하나로 통일** — 우클릭 메뉴와 선택 툴바가 각자 다른 색상 선택 구현을 갖고 있어, 선택 툴바 쪽은 이미 지정한 색을 다시 열어도 표시가 안 됐다. 이제 둘 다 같은 모양, 같은 "현재 색 표시".
- **접기 화살표·링크 제거 버튼이 평소에도 옅게 보입니다** — 이전엔 완전히 숨겨져 있어 호버하기 전엔 있는지도 몰랐다.
- **노트 에디터 툴바의 각 그룹(제목/서식/구조/삽입)이 시각적으로 구분됩니다.**
- **탭 우클릭 메뉴, "템플릿 추가" 드롭다운이 이제 Escape로 닫힙니다.**
- **정보 양식 삭제 시 확인을 먼저 받습니다** — 이 양식을 쓰는 모든 노트에서 함께 사라지는 되돌릴 수 없는 동작인데 확인이 없었다.
- 영어로 남아있던 라벨("Note Template"), 헷갈리던 용어("템플릿"/"양식" 혼용), 원인·다음 행동이 없던 오류 메시지 여러 건을 정리.

### 버그 수정
- **탭을 닫을 때 편집 중이던 마지막 입력이 자동저장 전에 사라지던 문제 수정** — 편집 직후 바로 탭을 닫으면 아직 디스크에 안 쓰인 내용이 그대로 사라졌다. 이제 편집 중인 탭은 닫기 전에 즉시 저장한다.

- **노드 순서 드래그** — 형제 노드를 끌어 순서를 바꿉니다(위/아래 틈에 놓으면 형제로 삽입, 가운데에 놓으면 자식으로).

### 개선
- **탭 순서 드래그** — 탭을 끌어 순서를 바꾸고 좌우로 옮길 수 있습니다(드래그가 되지 않던 문제 해결).

### 버그 수정
- 노드에서 "새 노트 만들어 연결"이 조용히 실패하던 문제 — 이제 실패 시 알려주고, 한 단계가 실패해도 나머지는 진행됩니다.
- 노트 이름을 바꾸면 다른 노트의 `[[ ]]` 링크가 깨지던 문제 — 참조하는 모든 노트에서 자동으로 갱신됩니다.
