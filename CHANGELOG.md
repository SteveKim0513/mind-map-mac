# Changelog

모든 주요 변경을 이 파일에 기록합니다. 형식은 [Keep a Changelog](https://keepachangelog.com/ko/),
버전은 [유의적 버전(SemVer)](https://semver.org/lang/ko/)을 따릅니다.
이 파일은 앱의 "업데이트 내역"에도 그대로 표시됩니다.

## [0.7.5] - 2026-06-22

### 버그 수정
- **노트 목차 한글 깨짐** — 정보 패널(목차)의 `note-toc-list`에 적용된 `pillPop` 애니메이션(`transform: scale(0.96)`)이 GPU 컴포지팅 레이어를 생성하고, 그 안에서 한글 텍스트가 잘못 렌더링되던 문제. 해당 애니메이션 제거로 해결.
- **링크 노트 중복 생성** — Enter 키 반복 입력 시 React state 업데이트 비동기 지연 창에서 `importNoteFromUrl`이 동시에 여러 번 호출되어 노트가 중복 생성되던 레이스 컨디션. `importingRef` 동기 가드로 첫 호출 이후 중복 실행을 차단.

## [0.7.4] - 2026-06-17

### 버그 수정
- **노트 표 소실** — 셀에 블록이 2개 이상일 때(셀 안에서 Enter) tiptap-markdown이 표 전체를 리터럴 `[table]`로 직렬화해 저장/탭전환 시 표가 사라지던 문제. 항상 유효한 GFM으로 직렬화하는 커스텀 Table 직렬화기(`tableMarkdown.ts`, 멀티블록 셀은 한 줄로 평탄화)로 해결.
- **체크리스트** — tiptap-markdown이 task item 사이에 빈 줄을 넣어 "느슨한" 목록으로 저장하던 것을 표준 tight GFM(`- [ ] a\n- [ ] b`)으로 정리(onChange 후처리).

## [0.7.3] - 2026-06-17

### 버그 수정 · 개선
- **다중 선택 우클릭 메뉴를 단일과 분기** — 선택이 2개 이상이면 "N개 선택됨" 헤더와 함께 선택 전체에 적용되는 항목만 노출(두 노드 연결·섹션으로 묶기·색상·완료·삭제). 단일 전용 항목(편집·자식/형제 추가·메모·링크·노트·스케줄·집중·보기 등)은 숨김.
- **일괄 색상** `setColorSelected` + **일괄 완료** `toggleDoneSelected` 추가. 메뉴 삭제가 `deleteNode(id)`(한 노드) → `deleteSelected`(전체)로 바뀌어 키보드 Delete와 일치.

## [0.7.2] - 2026-06-17

### 개선
- **업데이트 확인 — 즉시 인앱 팝업** — 메뉴/설정의 "업데이트 확인"이 곧바로 팝업을 띄우고 상태를 스트리밍(확인 중 → 최신/다운로드 N%/재시동/오류)합니다. 네트워크가 끝나야 뜨던 느린 네이티브 다이얼로그를 대체. 설정에 "업데이트 확인" 버튼 추가. 백그라운드 자동 확인은 그대로.
- **변경 내역 분리** — 앱에 노출되는 사용자용 `CHANGELOG.user.md`(고객 언어·간결)와 개발용 `CHANGELOG.md`(기술 상세)로 나눔. 앱의 `CURRENT_VERSION`·"업데이트 내역"·"새로운 점" 카드는 사용자용을 번들.

## [0.7.0] - 2026-06-16

### 새 기능
- **전체 검색** — 모든 지도의 노드와 모든 노트(제목·내용)를 한 번에 검색합니다. 결과를 누르면 노드는 해당 지도를 열어 가운데로, 노트는 바로 엽니다. **⌘⇧F**, 사이드바 검색, 또는 노트·홈에서 **⌘F**로 엽니다.

### 개선
- **사이드바 · 설정 정리** — 사이드바를 검색 · 오늘/돌아보기 · 라이브러리 중심으로 가볍게 다시 구성했습니다. 만들기는 풋터의 **＋ 하나**(마인드맵·노트·링크·폴더)로, 설정은 **⚙ / ⌘,** 로 엽니다.
- **설정은 진짜 옵션만** — 테마 · 글자 크기 · 워크스페이스 · 버전. 단축키와 사용법은 설정의 **"사용 안내"**(섹션으로 넘기는 매뉴얼)로 옮겼습니다.

### 버그 수정
- **루트 노드를 다른 노드의 자식으로** 만들 수 있습니다 — 루트를 끌어 노드 위에 놓으면 그 자식이 됩니다(빈 곳에 놓으면 기존처럼 자유 배치).

## [0.6.1] - 2026-06-16

### 개선
- **탭 순서 조정** — 탭 사이의 또렷한 세로 막대로 놓일 위치를 보여주고, 커서가 탭의 왼/오른쪽 어디에 있는지로 앞·뒤를 정해 정확히 옮깁니다.
- **화면 분할 가이드** — 탭을 아래 화면 영역으로 끌어내렸을 때만, 더 작고 차분한 안내로 표시됩니다(순서만 바꿀 땐 뜨지 않음). 가리키는 쪽이 강조됩니다.

## [0.6.0] - 2026-06-16

### 새 기능
- **노트 ↔ 노트 연결** — 본문에서 `[[노트 제목]]`으로 노트를 잇습니다. `[[` 입력 시 자동완성이 뜨고, 없는 노트는 그 자리에서 만들어 연결합니다. 링크를 누르면 **미리보기(peek)**가 떠 보던 노트를 잃지 않고, "열기"나 **⌘-클릭**으로 반대쪽 화면에 나란히 엽니다. 아직 없는 노트로의 링크는 흐리게 표시됩니다.
- **백링크(양방향)** — 노트 제목 옆 "정보"에서 *이 노트를 참조하는 노트들*을 봅니다. 한쪽만 링크해도 양쪽에서 오갈 수 있습니다.
- **노트 "정보" 패널** — 목차 · 연결된 노드 · 백링크를 한곳에 모았습니다(기본 닫힘, 있을 때만 표시).
- **노드 순서 드래그** — 형제 노드를 끌어 순서를 바꿉니다(위/아래 틈에 놓으면 형제로 삽입, 가운데에 놓으면 자식으로).

### 개선
- **탭 순서 드래그** — 탭을 끌어 순서를 바꾸고 좌우로 옮길 수 있습니다(드래그가 되지 않던 문제 해결).

### 버그 수정
- 노드에서 "새 노트 만들어 연결"이 조용히 실패하던 문제 — 이제 실패 시 알려주고, 한 단계가 실패해도 나머지는 진행됩니다.
- 노트 이름을 바꾸면 다른 노트의 `[[ ]]` 링크가 깨지던 문제 — 참조하는 모든 노트에서 자동으로 갱신됩니다.

## [0.5.1] - 2026-06-14

### 버그 수정
- **노드에서 만든 노트가 사이드바에 보입니다** — 숨김 폴더로 들어가던 문제 수정(이제 노드의 맵과 같은 폴더에 생성). 이전 버전에서 숨어 있던 노드 노트도 앱을 열 때 자동으로 복구됩니다.

## [0.5.0] - 2026-06-12

### 새 기능
- **집중 세션** — 노드에서 집중 세션을 시작해 목표·과정·결과를 남깁니다. 타이머, 작업 로그 노트, 완료 카드.

- **노트 표(테이블)** — GFM 마크다운.

### 개선
- 접근성(WCAG AA), 디자인 일관성, 사이드바 내비 정리, 크래시 진단 로깅.
