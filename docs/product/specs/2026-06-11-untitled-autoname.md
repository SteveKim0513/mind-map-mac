# "제목 없음" 자동 이름 — 기능 명세

> 작성: 2026-06-11 · 작성자: 기획(Claude) · 상태: 구현·E2E 검증 완료
> 출처: [UX-AUDIT-2026-06 U3](../reports/UX-AUDIT-2026-06.md)

## 1. 문제 (Why)

홈 CTA "새 마인드맵"은 이름 단계 없이 "제목 없음.mind"를 만들고 끝난다. 사이드바 경로만 인라인 이름 변경에 진입한다. 결과: 실제 워크스페이스에 "제목 없음", "제목 없음 2"가 누적 — 파일을 다시 찾을 수 없게 된다. 핵심 경험 1번(속도)을 지키면서 풀어야 한다: **이름 짓기를 강요하는 다이얼로그는 답이 아니다.**

## 2. 해결 (What)

**첫 중심 토픽의 텍스트가 곧 파일 이름이 된다** (Notion의 제목=첫 줄 관성).

- 트리거: 맵의 파일명이 `제목 없음` 패턴(`제목 없음`, `제목 없음 2`, …)이고, 첫 루트 노드에 비어 있지 않은 텍스트가 커밋된 뒤 자동 저장이 도는 시점.
- 동작: 루트 텍스트를 파일명으로 정제(금지 문자 제거, 공백 정리, 최대 60자)해 파일을 rename. 충돌 시 기존 디듀프 규칙(`이름 2`)을 따른다. 탭 제목·최근 목록·열린 세션 경로는 기존 rename 인프라(`flushSaves` → `fs:rename` → `renamePath` → 워크스페이스 refresh)를 그대로 탄다.
- 한 번만: rename 후에는 파일명이 패턴을 벗어나므로 자연히 재발동하지 않는다. 사용자가 직접 지은 이름은 절대 덮어쓰지 않는다(패턴 매칭이 가드).
- 루트 편집 중에는 발동하지 않는다(타이핑 중간 이름 방지 — `editingId` 가드).
- undo 대상 아님(파일명은 문서 데이터가 아님). 리마인더 동기화 영향 없음. 파일 포맷 변경 없음.

## 3. 범위에서 뺀 것 (Not now)

- 노트(.md)의 동일 규칙 — 노트는 프론트매터 title과 파일명이 분리돼 있어 별도 설계 필요(제목 입력 시 파일명 동기화). 후속 명세로.
- 루트 텍스트 변경 시 파일명 추적(계속 따라가기) — 의도적으로 안 함. 첫 명명 이후 파일명은 사용자 소유.
- 홈 CTA에 이름 입력 단계 추가 — 속도 원칙에 반함.
