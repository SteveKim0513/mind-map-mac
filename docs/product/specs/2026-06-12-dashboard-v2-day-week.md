# 돌아보기 v2 — 하나의 스코프(오늘 ↔ 이번 주), 계층 정리

> 작성: 2026-06-12 · 상태: 명세(합의 완료, 구현 예정) · 출처: 사용자 피드백(기본을 오늘로 / 주간은 옵션 / 빨강 밴드 무의미 / 주·일 정보 혼재). [decisions/0007](../../decisions/0007-focus-positioning.md) 맥락, [v1 redesign](2026-06-12-work-dashboard-redesign.md) 보강.

## 1. 문제

돌아보기(회고)에는 리듬이 다른 두 가지가 한 화면에 **섞여** 있었다:
- **매일 점검** — "오늘 뭘 얼마나 집중했나" (자주, 즉각)
- **주간 성찰** — "이번 주 패턴·추세·시간 배분" (가끔, 패턴)

v1은 *주 히어로 + 일 목록 + 주간 띠 + 일 드릴*을 한 번에 보여줘 **주·일 계층이 어긋났다.** 또 맨 위 **빨강 플래그 밴드**(파편화/마감)는 매일 보는 화면에 알람처럼 떠 과했고, 마감 정보는 회고가 아니라 *계획(오늘 뷰)*의 몫이다.

## 2. 해결 — 한 번에 한 축

상단 **세그먼트 토글 [오늘 | 이번 주]**, **기본값 = 오늘**. 항상 한 스코프만 — 주·일 절대 안 섞임.

### 오늘 (day, 기본)
- 헤더: 토글 + `◀ {오늘/어제/날짜} ▶` (날짜 이동, 미래 불가).
- 히어로: `{날짜} 집중` + 큰 총합 + `N세션` (+ 어제 대비 델타, 선택).
- **무엇을 했나**: 그날 세션 목록(시간 역순) — `시간 · 주제 · 성과 · 길이`, 펼치면 목표↔성과↔노트.
- 빈 상태: "이 날은 집중 기록이 없어요."
- *차트·주간 띠·인사이트 없음.* 매일 열어도 부담 없는 단순 화면.

### 이번 주 (week, 옵션)
- 헤더: 토글 + `◀ {이번 주/지난주/날짜범위} ▶` (주 이동).
- 히어로: `{주} 집중` + 큰 총합 + 지난주 대비 `↑+40%`.
- **주간 띠**: 7칸(월~일), 오늘 강조. **칸 클릭 → 오늘(day) 스코프로** 그날 드릴(스코프 전환).
- **주제별 칩**: `전체 + 상위 주제(시간)` — 누르면 바로 아래 목록을 그 주제로 필터(컨트롤=결과 인접).
- **무엇을 했나(주)**: 그 주 세션을 날짜별로(주제 필터 반영).
- **이렇게 일했어요**: 차분한 인사이트 1~3줄(딥워크·최장·파편화) — *빨강 아님, 알람 아님.*
- **▸ 더보기**(접힘, 읽기 전용): 시간대 · 8주 추세.

## 3. 범위에서 뺀 것

- **마감 vs 실제(빨강 플래그)** — 돌아보기에서 **제거**. 마감은 *오늘 뷰*(지남/오늘)가 담당. → `priorityVsActual`/`allScheduledNodes` 의존 제거(코드 단순화).
- 월 단위/사용자 지정 기간 — 후속(오늘·이번 주 두 축으로 충분).
- 주 모드의 풀 내러티브를 일 모드와 중복 노출하지 않음(상세는 띠 클릭으로 일 모드).

## 4. 완료 기준

- 기본 진입 = **오늘** 스코프. 토글로 이번 주 전환, 다시 오늘.
- 두 스코프가 **각자 자기 레벨 정보만** 노출(주·일 안 섞임).
- 주간 띠 칸 클릭 → 오늘 스코프로 그날 표시.
- 주제 칩이 바로 아래 목록을 필터(시야 안).
- 빨강 플래그 밴드 없음. 패턴은 주 모드의 차분한 문장.
- 기존 순수 집계 함수 재사용, 회귀 없음. 격리 E2E로 양 스코프·드릴·필터 검증.
