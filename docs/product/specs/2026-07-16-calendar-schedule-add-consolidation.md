# 캘린더 "일정 추가" 진입점 일관화 — UX 개선안

> 작성: 2026-07-16 · 상태: **기획(구현 전)** · 방법: ux-audit 스킬로 라이브 워크스루(일/주/월 각 뷰의
> 일정 추가 플로우, 스크린샷 11장 + 콘솔 0 에러) + `/design-ui` 원칙 적용.
> 기반: [calendar-ux-overhaul](2026-07-16-calendar-ux-overhaul.md)(SchedulePicker·진입점 도입).

## 1. 문제

일정 추가의 **진입 방식이 뷰마다 다르고 대부분 보이지 않는다.** 같은 작업(일정 추가)인데 뷰를 바꾸면
방법이 바뀌어 사용자가 매번 다시 학습해야 한다. 원칙 5("같은 역할의 요소는 앱 전체에서 같은 모양")·
원칙 3(점진적 공개) 위반.

## 2. 현재 진입점·프로세스 (워크스루로 확인)

| 뷰 | 진입점 | 발견성 | 시간 지정 | 빈 공간 클릭 의미 |
|---|---|---|---|---|
| **일간** | 본문 우상단 **`+ 일정 추가` 버튼** | ✅ 보임(유일) | 종일 기본 + `시간 지정` 토글 | — |
| **주간** | **빈 시간 슬롯 클릭**(암묵) | ❌ 안 보임(힌트 없음) | **슬롯 위치=시각 고정(토글 없음)** | 슬롯 클릭 = **추가** |
| **주간** | **종일 띠 클릭**(암묵) | ❌ 안 보임 | 종일(토글 있음) | — |
| **월간** | **셀 hover 시 나타나는 `+`** | ❌ hover 전 opacity 0 | 종일 기본 + `시간 지정` 토글 | 셀 본문 클릭 = **일간 이동** |

**공통 프로세스(진입 후는 동일)**: 진입 → `SchedulePicker` 팝업 → (a) 기존 노드 검색·선택 → 그 노드에
일정, 또는 (b) `새로 만들기` → 오늘의 생각. 피커 자체는 뷰와 무관하게 일관적이다(스크린샷 02·06·10).

### 관찰된 결함 (증거 기반)
- **F1 · 진입 발견성 불균형** — 일간만 라벨 버튼이 보이고, 주간·월간엔 **보이는 추가 버튼이 없다**
  (주간 헤더/월간 헤더에 버튼 없음, 스크린샷 05·08). 주간은 "슬롯을 클릭하면 된다"는 신호가 전무하고,
  월간 `+`는 hover 전 **opacity 0**(측정값 0 → hover 후 0.98).
- **F2 · "빈 공간 클릭"의 의미 충돌** — 주간 빈 슬롯 클릭 = **추가**, 월간 빈 셀 클릭 = **일간 이동**.
  같은 제스처가 뷰마다 반대 결과 → 예측 불가.
- **F3 · 시간 지정 모델 불일치** — 주간 슬롯은 시각이 위치로 고정(종일 불가·토글 없음), 일간·월간은
  종일 기본+토글. "언제"를 정하는 방식이 뷰마다 다르다.
- **F4 · 피커 Escape가 안 먹는 경우** — 검색 입력에 포커스가 있을 때만 Escape로 닫힌다. 토글/시간
  필드를 만진 뒤 Escape를 누르면 **안 닫히고 scrim이 다음 클릭을 막는다**(워크스루에서 실제로 hang 유발).
- **F5 · 피커 빈 검색 상태** — 검색어가 없으면 워크스페이스 전체 노드가 나열되고 "새로 만들기"는
  검색어가 있어야 나온다 → 첫 사용자가 "어떻게 새로 만들지" 헷갈릴 수 있음(부차적).

## 3. 개선안 — "한 개의 보이는 입구 + 뷰별 가속기"

핵심 원칙: **진입은 어디서나 같은 버튼 하나로 보장하고(발견성), 뷰별 클릭은 그 위의 가속기로 둔다.**
피커는 이미 일관적이므로, **진입만 통일**하면 된다.

### 3.1 (F1) 공용 헤더 `+ 일정 추가` 버튼 — 모든 뷰
- 캘린더 **헤더**(현재 `오늘` 버튼 옆)에 **항상 보이는 `+ 일정 추가` 버튼**을 둔다 — 일/주/월 **동일 위치**.
  누르면 그 뷰의 현재 기준일(일간=그 날, 주간=오늘 또는 주의 오늘, 월간=오늘)로 `SchedulePicker`.
- 일간 본문 우상단의 기존 버튼은 **헤더로 승격**(중복 제거). 주간·월간도 이 버튼으로 발견성 확보.
- → 뷰를 바꿔도 **같은 자리, 같은 라벨, 같은 결과**. 원칙 5 충족.

### 3.2 (F3) 피커의 종일/시간 토글을 **항상 노출**(문맥 프리필)
- 지금은 주간 슬롯 진입 시 토글을 숨긴다 → 뷰마다 피커가 달라 보인다. **토글을 항상 보이게** 하고,
  진입 문맥으로 **프리필만 다르게** 한다: 주간 슬롯 진입 → `시간 지정` + 그 슬롯 시각, 그 외 → `종일`.
- → 모든 추가가 **동일한 피커·동일한 시간 컨트롤**을 지난다. "언제"의 정신 모델 하나로 통일(F3 해소).
  슬롯을 잘못 찍어도 피커에서 시각·종일을 바로 고칠 수 있다.

### 3.3 (F2) "빈 공간 클릭" 의미를 명확히 + 힌트
- 의미는 뷰 목적에 맞게 **유지하되**(주간=시간 배치 표면이라 슬롯 클릭=추가 / 월간=조망 표면이라
  셀 클릭=일간 이동), **헤더 버튼(3.1)이 일관 입구**라 충돌의 부담이 사라진다.
- 발견성 보강(가속기 힌트):
  - **주간 슬롯 hover** → 그 슬롯에 옅은 `+ 일정` 고스트 라벨(hover 시에만). 클릭 가능함을 알린다.
  - **월간 `+`** → hover 전에도 **아주 옅게(opacity ~0.35) 상시 표시**, hover 시 진하게. 완전 투명 제거.

### 3.4 (F4) 피커 Escape·바깥클릭 닫기 정상화
- 포커스 위치와 무관하게 **Escape로 항상 닫힌다**. `SchedulePicker` 루트에 keydown(Escape→onClose)
  핸들러를 두거나 스크림에 위임. 바깥 클릭 닫기는 이미 동작(scrim onClick)하나 Escape 경로를 보강.
- 접근성(원칙 4, 키보드로 모든 작업).

### 3.5 (F5) 피커 빈 상태 카피(경미)
- 검색어가 없을 때 상단에 옅은 안내 한 줄("노드를 검색하거나, 입력해서 새로 만드세요"). 새로 만들기가
  검색어 입력으로 열린다는 걸 알린다.

## 4. 범위에서 뺀 것
- 피커 자체 재설계(이미 일관적). 자연어 시각 입력(별도). 반복 일정(Phase 4).
- 월간 셀-클릭 의미를 "추가"로 바꾸는 것(일간 이동이 유용 — 헤더 버튼으로 추가는 커버).

## 5. 가드레일 정합성
- 원칙 5(패턴 반복)·3(점진적 공개)·4(키보드) 충족. 새 색 없음(기존 `--state-scheduled`).
- 스키마·store 무변경 — 순수 UI 배선(진입점 이동 + 토글 상시 + Escape 핸들러 + hover 힌트 CSS).
- 실행/사고 분리(0014/0015)와 무관 — 캘린더 계획 표면 안의 일관화.

## 6. 완료 기준
- [ ] 일/주/월 **모두** 헤더에 `+ 일정 추가` 버튼이 **같은 자리**에 있고, 누르면 그 기준일로 피커가 열린다.
- [ ] 피커의 종일/시간 토글이 **항상 보이고**, 주간 슬롯 진입 시 `시간 지정`+슬롯 시각으로 프리필된다.
- [ ] 주간 슬롯 hover 힌트 + 월간 `+` 상시 옅게 표시(hover 전 opacity > 0).
- [ ] 피커가 **어떤 포커스에서든 Escape로 닫힌다**(재현: 토글 클릭 후 Escape).
- [ ] E2E: 세 뷰 헤더 버튼 → 피커 오픈, 주간 슬롯 프리필, Escape 닫힘.
- [ ] `make dev-safe` 다크/라이트 육안.
