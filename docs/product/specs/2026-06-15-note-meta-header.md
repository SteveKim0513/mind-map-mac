# 노트 상단 메타 영역 — UI 기획

> 작성: 2026-06-16 · 상태: **구현 완료** — 제목 옆 "정보" 버튼 → 아코디언 패널에 **목차 + 연결된 노드 + 백링크** 통합(count 배지, 기본 닫힘, 메타 없으면 버튼 숨김). 목차는 에디터에서 패널로 이전(`NoteHeadingsProbe`로 헤딩 lift), `TableOfContents.tsx` 제거. E2E 검증.
> 출처: 사용자 요청("목차 영역과 더불어 상단 메타 영역 UI 개선, 기존 화면과 잘 융화되게"). 백링크 상단 배치([[note-to-note-links]]) 전제.
> 관련: [[note-feature-architecture]], 목차 기능(2026-06-15 릴리즈), 색상 3계층(ADR 0001).

## 1. 문제 (Why)

노트 상단에 "이 노트에 대한 메타" 정보가 **세 곳에 흩어져** 있고 스타일이 제각각이다.

- `note-head`: 제목 입력 + 저장 상태 + 연동 버튼.
- `note-links`(에디터 **밖**): 연결된 노드 칩들.
- `TableOfContents`(에디터 **안**, `note-rich` 최상단): 목차 칩.

여기에 곧 **노트↔노트 백링크**가 상단에 추가된다([[note-to-note-links]]). 지금처럼 두면 메타가 4종으로 더 흩어지고, 목차(에디터 안)와 노드 칩(에디터 밖)의 경계가 어긋나 시각적으로 따로 논다.

## 2. 해결 (What) — 단일 "정보" 버튼 → 하나의 아코디언

칩 여러 개로 늘어놓지 않는다. `note-head`의 **연동 버튼 옆에 "정보" 버튼**(이름 잠정, 예: 정보 / ⓘ)을 두고, 누르면 **현재 목차가 있던 그 자리**(제목 아래·본문 위)에 **하나의 아코디언 패널**이 열려 노트 관련 메타가 모두 들어간다. 다시 누르면 닫힘. 기본 닫힘.

```
┌ note-head ─────────────────────────────────────┐
│ [제목 입력................]   저장됨   ⓘ정보  ＋연동 │
└────────────────────────────────────────────────┘
  ▼ (정보 누르면 본문 위에 한 패널이 펼쳐짐)
  ┌ note-info (본문 폭 정렬, 낮은 강도) ───────────┐
  │  목차                                          │
  │   • 제목 1 …  • 제목 2 …  (헤딩 점프)            │
  │  ─────────────────────────────────────────────│
  │  연결된 노드        ↗이동  ✕해제   ＋노드 연동   │
  │  ─────────────────────────────────────────────│
  │  백링크 (이 노트를 참조한 노트)                  │
  │   • 노트 A   • 노트 B   (클릭 → 탭 열기)         │
  └────────────────────────────────────────────────┘
  (에디터 본문…)
```

- **한 패널, 섹션 스택**: 패널 안에 `목차 / 연결된 노드 / 백링크` 섹션을 hairline으로 구분해 위→아래로. 칩을 따로 토글하지 않으니 "어느 칩을 열까" 고민이 없다.
- **있는 섹션만 표시**: 헤딩<2면 목차 생략, 노드 연결 0이면 연결 섹션 생략(단 "＋노드 연동" 줄은 남겨 빈 노트에서도 연동 가능), 백링크 0이면 백링크 섹션 생략. 패널을 열었는데 아무 메타도 없으면 "이 노트에 대한 정보가 아직 없어요" 한 줄.
- **정보 버튼 신호**: 백링크나 연결이 있으면 버튼에 **작은 점/카운트**(예: `ⓘ 정보 ·3`)로 "볼 게 있다"를 은은히 표시. 없으면 점 없음.
- **목차 동작 유지**: 헤딩 추출·클릭 점프는 기존 `TableOfContents` 로직 그대로, 위치만 패널 안으로.
- **세션 노트**: 패널은 목차만(노드 연동은 불변 → `SessionMetaBanner` 유지).

### 정렬·융화 원칙 (디자인)

- 패널은 **본문 컬럼 폭에 정렬**(좌우 여백 = 에디터 본문) → 목차가 에디터 밖으로 나오며 생기던 경계 어긋남 해소.
- 색: 패널은 중립(낮은 강도) 배경 + `--ink-muted`. 섹션 아이콘만 정체성색 — 노드=`--state-map`, 백링크/노트=`--state-note`(ADR 0001, 아이콘으로 의미 전달).
- 펼침: 위가 출렁이지 않도록 본문은 패널 높이만큼 자연스레 밀림(레이아웃 흐름), 모션 없거나 ≤120ms — 앱의 절제된 모션과 일치.

## 3. 구조 변경 (How)

- 새 컴포넌트 `NoteInfoPanel`(`src/note/NoteInfoPanel.tsx`): 한 패널 + `목차 / 연결 / 백링크` 섹션. 열림 상태(boolean)는 NotePane이 보유(정보 버튼이 토글).
  - 목차: `TableOfContents`의 헤딩 추출 로직을 훅/컴포넌트로 흡수해 패널 섹션으로 렌더.
  - 연결: `note.links` + `revealNode`/`unlink` + 연동 추가(NotePane에서 끌어올림 / 기존 `＋연동`과 동작 공유).
  - 백링크: `useWorkspace.backlinks(note.id)`([[note-to-note-links]]에서 추가).
- `NotePane`: `note-head`에 "정보" 토글 버튼 추가, 기존 `note-links` 블록 제거(연결은 패널로), `NoteEditor` 위에 `{infoOpen && <NoteInfoPanel/>}` 배치. `TableOfContents`는 `NoteEditor` 내부에서 제거(패널로 이전).

## 4. 범위에서 뺀 것 (후속)

- 패널 고정(sticky) 스크롤, 백링크 호버 미리보기, 목차 현재 위치 하이라이트, 생성/수정 시각·단어 수 같은 부가 메타 — 후속.

## 5. 완료 기준

- [ ] `note-head` 연동 옆 "정보" 버튼 → 본문 위 단일 아코디언 패널 토글(기본 닫힘)
- [ ] 패널에 목차·연결된 노드·백링크가 한 패널 안 섹션으로, 본문 폭에 정렬
- [ ] 있는 섹션만 표시(빈 노트는 "＋연동"만), 메타 전무 시 안내 한 줄
- [ ] 정보 버튼에 백링크/연결 있을 때 점·카운트 신호
- [ ] 노드 연결 이동/해제·＋연동이 패널에서 동일 동작, 목차 점프 회귀 없음
- [ ] 세션 노트는 패널에 목차만
- [ ] FEATURE-INVENTORY 갱신

## 6. 의존성

- 백링크 섹션은 [[note-to-note-links]]의 `noteRefs`/`backlinks` 인덱스에 의존 → 노트↔노트 연결과 **함께 구현**하거나, 패널(목차+연결 통합)을 먼저 짓고 백링크 섹션만 그 기능 구현 시 채우는 2단계도 가능(목차·연결 통합만으로도 즉시 가치).
