# "오늘" 뷰 — 흩어진 일정을 모아 바로 몰입으로

> 작성: 2026-06-12 · 상태: 명세(합의 후 구현) · 출처: 사용자(킬러기능 검토 — todo 앱의 "Today"를 우리 정체성으로 흡수). 관련 [decisions/0007](../../decisions/0007-focus-positioning.md) 보강.

## 1. 문제

일정/마감(`scheduled`/`scheduleAt`)이 **여러 맵에 흩어져** 있어, "오늘 뭘 해야 하지"를 한 화면에서 못 본다. todo 앱이 가장 사랑받는 이유가 바로 그 "Today" 한 화면인데 우리에겐 없다. 동시에 우리는 그걸 그대로 베끼는 대신 **우리만의 집중 세션과 결합**할 수 있다.

## 2. 해결

전 워크스페이스의 예정 노드를 한 곳에 모은 **"오늘" 뷰**. 핵심 차별점: 각 항목에서 **클릭 한 번에 집중 세션 시작** → 보면서 바로 실행 → 과정·결과가 작업 기록에 쌓임. (todo 앱은 "할 일 목록"에서 멈추지만 우리는 거기서 몰입에 들어가고 그게 측정된다.)

**구성 (위→아래)**
- **지남(overdue)**: `scheduleAt < 오늘 0시` AND `!done` — 빨강 틴트, 맨 위(놓친 것 먼저).
- **오늘**: `scheduleAt`가 오늘 — 시간 있으면 시간순.
- **다가오는(7일)**: 내일~+7일, 날짜별 그룹.
- 각 행: `[시간] 노드텍스트 · 맵이름   [⏱ 집중] [✓ 완료]`
- 상단 한 줄 요약: "오늘 N건 · 지남 M건 · 오늘 집중 X" (집중량은 기존 집계 재사용).

**상호작용** (기존 자산 재사용)
- 행/노드텍스트 클릭 → `revealNode`(맵 열고 노드 센터) — 이미 있음.
- **[집중]** → 맵이 열려 있으면 `requestFocusStart(store, nodeId)`(목표 팝업), 닫혀 있으면 먼저 reveal 후 안내.
- **[완료]** → 열린 맵은 `toggleDone` 직접, 닫힌 맵은 reveal(캔버스에서 토글) — *v1 한계*.

**데이터**: 기존 `allScheduledNodes`(열린 맵 + 디스크 `.mind` 스캔, WorkHistory에 있음)를 일반화해 `done`·시간 포함하는 `agendaItems()`로 추출. 그룹핑/정렬은 **순수 함수**(테스트 가능, report.ts 패턴).

**진입점**(매트릭스 준수): 사이드바 하단 **"오늘"**("작업 기록" 위) + `⌘K` "오늘 열기". 빈 상태: "오늘 예정된 일정이 없어요 — 노드에 날짜·시간을 설정하면 여기 모여요."

## 3. 범위에서 뺀 것 (v1)

- 닫힌 맵 노드의 **인라인 완료/집중**(reveal로 우회) — fast-follow.
- 자연어 날짜 입력, 반복 일정, 드래그 재일정 — 후속 후보(별도).
- todo식 프로젝트/라벨/필터/우선순위 등급 — *우리는 todo 앱이 아니다*(가드레일).

## 4. 완료 기준

- 여러 맵의 예정 노드가 지남/오늘/다가오는으로 한 화면에 모인다(열린 맵 + 디스크).
- 행에서 **클릭 한 번에 집중 세션 시작**(목표 팝업까지) 동작.
- 노드 클릭 → 해당 맵/노드로 이동.
- 일정 0건 빈 상태 안내.
- 그룹핑/정렬 순수 함수 단위 테스트.
- 일반 사용(비집중) 경험·성능 회귀 없음.
