# 동작별 표준 진입점 매트릭스 — 기능 명세

> 작성: 2026-06-11 · 작성자: 기획(Claude) · 상태: 1차 구현·검증 완료 (매트릭스 팀 합의 대기)
> 출처: [UX-AUDIT-2026-06 U4](../UX-AUDIT-2026-06.md), [FEATURE-INVENTORY 부록 B](../FEATURE-INVENTORY.md)

## 1. 문제 (Why)

같은 동작의 진입점이 화면마다 다르게 흩어져 있고(아이콘은 팝오버에만, 노트 해제는 노트 패널에만, 분할은 드래그·⌘K뿐), 기준이 없어 새 기능마다 진입점 배치가 취향 논쟁이 된다.

## 2. 해결 (What) — 표준 매트릭스

각 레이어의 역할을 고정한다. **새 동작을 추가할 때 이 표에 행을 추가하는 것이 기본 절차다.**

| 레이어 | 역할 | 원칙 |
|---|---|---|
| 단축키 | 파워유저 고속 경로 | 고빈도 동작만, README·⌘K에 항상 병기 |
| 우클릭 메뉴 | **전체 카탈로그** — 노드에 가능한 모든 동작 | 여기 없는 노드 동작은 없어야 한다 |
| 선택 툴바 | 고빈도 6±2개 | 메뉴의 부분집합. 추가 시 하나 빼는 것 검토 |
| 노드 팝오버 | 속성 상세 편집(메모·링크·아이콘) | 메뉴의 "…" 항목이 여는 시트 |
| 네이티브 메뉴(메뉴바) | 앱·문서 수준 동작 | 캔버스 노드 동작은 넣지 않는다 |
| ⌘K 팔레트 | 모든 명령의 검색 가능한 합집합 | 위 레이어들의 미러 |

### 현재 위반 2건 → 이번에 해소

1. **노트 연동 해제가 노드 쪽에 없음** — 연결은 노드에서 했는데 해제는 노트 패널에서만 가능(비대칭). → 노드 칩이 여는 노트 팝업 헤더에 "연동 해제" 추가. 닫힌 노트 파일도 처리하도록 `removeLinkFromNoteFile` 신설(추가와 대칭).
2. **화면 분할이 메뉴바에 없음** — 탭 드래그/⌘K만. → 보기 메뉴에 "화면 분할 토글" 추가(기존 `toggleSplit` 재사용).

### 보류 (합의 후 별도 진행)

- 우클릭 메뉴 14항목 축소·그룹핑 — 매트릭스 합의 후 메뉴가 카탈로그 역할이면 길이는 수용 가능할 수도. 스크롤/그룹 구분선로 풀지 팀 논의.
- 아이콘 지정은 "메뉴 → 팝오버" 경로가 매트릭스에 부합하므로 위반 아님 (메뉴 라벨 "아이콘…"이 팝오버를 여는 것 유지).

## 3. 범위에서 뺀 것 (Not now)

- 선택 툴바 구성 변경, 메뉴 재배치 — 매트릭스 위반이 아니므로 보류.
- ⌘K에 노드 동작 추가 — 선택 컨텍스트 설계가 필요해 별도.
