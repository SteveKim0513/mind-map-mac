# 노트 이미지 — 기능 명세

> 작성: 2026-06-12 · 갱신: 2026-07-14 (저장 방식 변경 반영) · 상태: 구현·검증 완료
> 출처: 사용자 요청(노트에 이미지 추가).
> 저장 방식 변경 이력: `docs/exec-plans/completed/2026-06-23-note-image-external-storage.md`(base64→외부 파일), `docs/decisions/0010-note-image-assets-hidden-folder.md`(숨김 폴더).

## 1. 문제 (Why)

노트가 텍스트만 지원해 스크린샷·다이어그램을 붙일 수 없다. 기획·리서치 노트의 자연스러운 사용을 막는다.

## 2. 해결 (What)

TipTap Image 확장. 저장 방식은 두 번 바뀌었다 — **현재(v0.7.15 이후)는 외부 파일 저장**이 기본이며, 아래는 현재 동작을 기술한다.

- **저장 방식**: 파일이 있는 노트는 이미지를 노트와 같은 폴더의 `.notename.assets/`(이름 앞에 점 — 숨김 폴더, v0.8.3부터) 안에 실제 파일로 저장하고, 본문엔 `![](./.notename.assets/image-*.ext)` 상대경로만 남긴다. 저장된 적 없는 세션 노트(파일 경로가 없는 경우)만 base64 인라인으로 폴백한다. `electron/main.ts`의 `images:write`/`images:read` IPC, `src/note/imageInsert.ts`.
- **폴더 동기화**: 노트 이름 변경·이동·휴지통 이동·복원 시 `.assets` 폴더도 함께 옮겨지고 본문의 이미지 참조 경로도 자동으로 다시 쓰인다(`electron/main.ts`의 `findAssetsDir`/`relocateAssetsDir`). v0.8.3 이전에 만들어진 점 없는 폴더(`notename.assets`)도 계속 인식되며, 위 동작 중 하나가 그 노트에 처음 일어날 때 숨김 이름으로 자동 승격된다.
- **삽입 경로**: (1) 툴바 이미지 버튼 → 파일 선택, (2) 붙여넣기, (3) 드래그&드롭.
- **자동 축소**: 삽입 시 캔버스로 최대 변 2400px로 축소, 투명도 있으면 PNG 유지·없으면 JPEG(품질 0.92)로 재인코딩. 작고 가벼운 이미지는 원본 유지.
- **표시**: 에디터·미리보기(NotePopup) 모두 `max-width: 100%`, 클릭 시 lightbox 확대(`src/note/ImageLightbox.tsx`). 미리보기 렌더러(markdown.tsx)는 `data:image/`로 시작하는 src만 허용(XSS 방어) — 외부 파일 이미지는 표시 전 IPC로 읽어 data URL로 변환 후 넘긴다.
- 마크다운 왕복: tiptap-markdown이 `![]()`를 직렬화/역직렬화.

## 3. 범위에서 뺀 것

- 이미지 리사이즈 핸들·정렬·캡션.
- 이미지 외 파일(pdf 등) 첨부.
- v0.8.3 이전에 만들어진 점 없는 폴더의 일괄(워크스페이스 전체) 마이그레이션 — 노트를 건드리는 시점(이름 변경 등)에 개별적으로만 승격된다(사유는 0010 결정 기록 참조).

## 4. 완료 기준

- [x] 툴바·붙여넣기·드롭으로 이미지 삽입
- [x] 큰 이미지는 축소되어 들어감
- [x] 저장→재열기 후에도 이미지 유지(마크다운 왕복)
- [x] 노드 칩 미리보기 팝업에서도 이미지 표시, data: 외 src 차단
- [x] 이미지 첨부 폴더는 숨김 폴더이며, 노트 이름 변경·이동·삭제·복원 시 폴더와 본문 참조가 함께 동기화된다
