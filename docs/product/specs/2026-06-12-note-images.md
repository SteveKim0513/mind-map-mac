# 노트 이미지 — 기능 명세

> 작성: 2026-06-12 · 상태: 구현·검증 완료
> 출처: 사용자 요청(노트에 이미지 추가).

## 1. 문제 (Why)

노트가 텍스트만 지원해 스크린샷·다이어그램을 붙일 수 없다. 기획·리서치 노트의 자연스러운 사용을 막는다.

## 2. 해결 (What)

TipTap Image 확장 + **base64 임베딩** (사용자 결정 2026-06-12).

- **저장 방식**: 이미지를 마크다운 본문에 `![](data:image/...;base64,...)`로 직접 포함. .md 파일 하나만 옮기면 이미지까지 따라가는 완전 이식성. 표준 마크다운이라 다른 에디터에서도 열림.
- **삽입 경로**: (1) 툴바 이미지 버튼 → 파일 선택, (2) 붙여넣기, (3) 드래그&드롭.
- **자동 축소**: 삽입 시 캔버스로 최대 변 1600px로 축소, 투명도 있으면 PNG 유지·없으면 JPEG(품질 0.82)로 재인코딩 → base64 용량 억제. 작고 가벼운 이미지는 원본 유지.
- **표시**: 에디터·미리보기(NotePopup) 모두 `max-width: 100%`. 미리보기 렌더러(markdown.tsx)는 `data:image/`로 시작하는 src만 허용(XSS 방어).
- 마크다운 왕복: tiptap-markdown이 `![]()`를 직렬화/역직렬화.

## 3. 범위에서 뺀 것

- 첨부 파일(.assets/) 방식 — 이식성 단순함을 위해 base64 선택. 대용량 이미지가 문제되면 후속 명세로 전환 검토.
- 이미지 리사이즈 핸들·정렬·캡션.
- 이미지 외 파일(pdf 등) 첨부.
