# 캘린더 뷰 — "오늘" 뷰를 일/주/월 탭으로 승격

> 작성: 2026-07-15 · 상태: 구현 완료 · 출처: 사용자 요청(기능 계층 재편의 일부,
> [FEATURE-HIERARCHY-VISION](../FEATURE-HIERARCHY-VISION-2026-07.md) §4-4, [결정 0011](../../decisions/0011-three-axis-thinking-model.md)).
> 기반: [2026-06-12-today-view.md](2026-06-12-today-view.md)(오버레이 "오늘" 뷰) — 이 명세로 대체되지
> 않고 승격된다. 데이터 계층(`collectAgenda`/`agenda.ts`)은 그대로 재사용.

## 1. 문제

지금 "오늘"은 사이드바에서 여는 오버레이(모달형 팝업)이고, 지남/오늘/다가오는(7일) 3구간만 보여준다.
"이번 주에 뭐가 있지", "이번 달은 어때" 같은 진짜 캘린더 질문에는 답을 못한다. 또한 오버레이라
다른 작업과 나란히 열어두고 참고할 수 없다(닫아야 다음 걸 함).

## 2. 해결

**사이드바 상단의 "오늘" 자리를 "캘린더"로 바꾸고, 클릭하면 탭으로 연다**(오버레이 아님 — 다른 탭처럼
전환·병렬로 열어둘 수 있음). 탭 안에서 **일간(day) / 주간(week) / 월간(month)** 세 보기를 토글.
**기본 진입은 일간 보기 + 오늘 날짜**(사용자 지시: "첫 화면은 오늘 일정이 잘 보이게").

### 공통 헤더

`[일 | 주 | 월]` 세그먼트 토글 + `◀ {기간 라벨} ▶`(이전/다음 기간 이동) + `오늘` 버튼(현재 선택 기간과
무관하게 오늘로 즉시 복귀). 기존 돌아보기 v2([specs/2026-06-12-dashboard-v2-day-week.md](2026-06-12-dashboard-v2-day-week.md))의
`◀ {라벨} ▶` 패턴을 그대로 재사용해 이미 있는 관성과 맞춘다.

### 일간(day) — 기본

기존 "오늘" 오버레이의 본문을 그대로 가져오되, **"오늘"에 고정되지 않고 선택된 날짜 기준**으로
동작하도록 일반화한다:
- 선택 날짜가 **오늘**이면: 지남(overdue, 전체 기간 기준) → 오늘 항목 순으로 보여준다(기존과 동일).
- 선택 날짜가 **오늘이 아니면**: 지남 섹션 없이 그 날짜의 항목만 보여준다(과거 날짜를 봐도 "지남"
  경고를 반복해서 보여줄 필요 없음 — 지남은 "지금 기준 처리 안 된 것"이라는 개념이라 오늘 뷰에서만
  의미가 있다).
- 각 행: 기존과 동일(`[시간] 텍스트 · 맵이름 [집중] [완료]`) — `requestFocusStart` 재사용.
- 빈 상태: "이 날은 예정된 일정이 없어요."

### 주간(week) — 7칸 그리드

월~일(또는 로캘 기준) 7칸, 각 칸에 날짜 + 그 날의 항목을 압축 표시(최대 3개 텍스트 + "+N개"). 칸
클릭 → 그 날짜로 일간 보기 전환(돌아보기 v2의 "주간 띠 클릭 → 오늘 스코프" 패턴과 동일 상호작용).
오늘 칸은 강조(테두리/배경).

### 월간(month) — 전통적 달력 그리드

해당 월의 모든 주(앞뒤 달의 넘치는 날짜 포함, 회색 처리) × 7칸. 각 칸에 날짜 숫자 + 항목이 있으면
작은 점(또는 개수 뱃지, 3개 초과 시 "+N"). 칸 클릭 → 그 날짜로 일간 보기 전환. 오늘 칸 강조.

## 3. 범위에서 뺀 것 (v1)

- 드래그로 일정 재조정(캘린더 칸 간 드래그) — 별도 후속.
- 반복 일정 — 데이터 모델에 없음(기존 한계, 이 명세로 안 건드림).
- 여러 워크스페이스/여러 캘린더 소스 통합 — 지금처럼 열려 있는 워크스페이스 하나만.
- **월간 보기의 성능 최적화**(대형 워크스페이스에서 수백 개 항목) — v1은 `collectAgenda()`를 그대로
  써서 매번 전체 스캔한다. 느리면 후속에서 캐시 도입.

## 4. 진입점 변경

- 사이드바 상단 내비게이션: **"오늘" 버튼 제거 → "캘린더" 버튼으로 교체**, 탭으로 열림.
- `⌘K`: "오늘 열기" 명령을 "캘린더 열기"로 이름 변경, 동작은 캘린더 탭을 열거나 이미 열려 있으면
  활성화.
- 캘린더는 **싱글턴 탭**이다 — 여러 번 열어도 탭이 하나만 존재(이미 있으면 활성화만).

## 5. 집중 기록(돌아보기) 접근 경로 변경

기존엔 사이드바 상단에 "집중 기록"이 있었다. 캘린더 도입으로:
- **캘린더 탭 헤더에 "돌아보기" 아이콘 버튼** 추가 — 일정(실행)을 보다가 바로 회고(통찰)로 이어지는
  동선.
- **사이드바 맨 아래(하단 바)**에도 유지 — 휴지통·Note Template과 같은 "관리형 화면" 자리.

## 6. 완료 기준

- [ ] 사이드바 상단이 "캘린더 · 즐겨찾기" 2줄이다.
- [ ] 캘린더를 누르면 탭으로 열리고(오버레이 아님), 다른 탭과 나란히 유지된다.
- [ ] 기본 진입은 일간 보기 + 오늘 날짜, 오늘의 지남/오늘 항목이 보인다.
- [ ] 일/주/월 토글이 전부 동작하고, 주/월에서 날짜 클릭 시 그 날짜의 일간 보기로 전환된다.
- [ ] 일간 보기의 각 항목에서 여전히 클릭 한 번에 집중 세션 시작이 된다(기존 "오늘" 뷰 동작 유지).
- [ ] 캘린더 탭 안에서, 그리고 사이드바 맨 아래에서 모두 돌아보기(대시보드)로 갈 수 있다.
- [ ] 기존 `collectAgenda`/`agenda.ts` 순수 함수는 그대로 재사용(회귀 없음), 새 날짜 그룹핑 함수는
      순수 함수로 분리해 단위 테스트.
