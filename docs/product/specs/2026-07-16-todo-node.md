# 할 일(todo) 노드 — 생각과 실행의 분리

> 작성: 2026-07-16 · 상태: 구현 예정 · 출처: 사용자 요청("done이 아무 노드나 가능해 붕 뜬 느낌 —
> todo 노드로 전환을 경계로 두고, 완료·집중은 todo 노드에서만"). 결정: [0014](../../decisions/0014-todo-node.md)
> (0011 §4 실행 그룹 개정), 데이터 [0012](../../decisions/0012-node-duration-field.md) 선례.

## 1. 문제

마인드맵은 브레인스토밍(생각)이 기본인데, `done`이 아무 노드에나 붙어 실행 흐름 밖에 떠 있다. 완료·일정·
집중이 모든 노드 메뉴에 노출돼 (1) 생각용 노드 메뉴가 무겁고 (2) "우리는 todo 앱이 아니다" 정체성이 흐려진다.

## 2. 해결

노드에 명시적 **할 일(todo)** 상태를 두고, 생각(일반 노드)과 실행(할 일 노드)을 가른다.

### 데이터
- `MindNode.todo?: boolean` — 선택 필드, **version 무범프**. 로드 시 `done||scheduled||reminderOn` → `todo=true` backfill.

### 일반 노드 (todo 아님)
- 메뉴/툴바: 정리(색·아이콘·메모·링크)·구조(자식·형제·편집·확대·복제·삭제)·통찰(이 노드만 보기·노트 연결)만.
- **완료·일정·집중은 아예 없음(숨김, 비활성 아님).** 실행 입구는 **"할 일로 전환"** 하나.
- 체크박스·일정 칩·집중 버튼 없음.

### 할 일 노드 (todo)
- **스타일 구분**: 제목 앞 체크박스 글리프(빈 원 ○ / 완료 ✓) + 살짝 다른 배경/보더.
- 완료(체크박스 클릭 또는 ⌘Enter), 일정 등록, (일정 있으면) 집중 세션 — 기존 규칙 그대로(집중은 이미 일정 게이트).
- 메뉴에 "일반 노드로 되돌리기" 추가.

### 전환 규칙
- **일반 → 할 일**: 메뉴 "할 일로 전환" / 툴바 버튼 / **⌘Enter**(일반 노드에서 누르면 할 일로 전환, 다시 누르면 완료 토글).
- **자동 승격**: 일정을 잡으면(수동 SchedulePopover·`@내일 3시` 자연어·캘린더 빈 슬롯 캡처) 자동으로 할 일이 됨.
- **할 일 → 일반(되돌리기)**: 완료·일정·리마인더를 정리. 반드시 `mapStore` 경유로 `reminderOn/reminderId` 고아 방지.
- 전환은 **노드별**(자식에 전파 안 함). 일정의 서브트리 전파(`setScheduled` 캐스케이드)는 유지.

## 3. 범위에서 뺀 것
- 우선순위·라벨·todo 리스트·마감 정렬(= todo 앱 기능). 가드레일.
- 서브트리 통째 "할 일로 전환" 옵션 — 후속(노드별로 충분).
- 반복 일정(별도 트랙).

## 4. 가드레일 정합성
- **"todo 앱 아님"**: 오히려 강화 — 실행은 옵트인, 지도는 순수 생각. 우선순위/리스트 없음.
- **reminder 불변조건**: 되돌리기·언스케줄은 `mapStore` 경유.
- **진행적 공개(원칙 3)**: 일반 노드는 실행 UI를 숨김.
- **색 억제**: 새 색 없음. 체크박스 글리프 + 기존 시맨틱.
- 데이터: version 무범프, backfill로 하위호환.

## 5. 완료 기준
- [ ] `todo?` 필드 + 로드 backfill(`done||scheduled||reminderOn`→todo). 라운드트립/마이그레이션 단위 테스트.
- [ ] 일반 노드 메뉴·툴바에 완료/일정/집중이 **안 보이고**, "할 일로 전환"만 보인다.
- [ ] 할 일 노드에 체크박스 글리프(클릭=완료 토글) + 시각 구분, 완료/일정/집중 노출, "일반 노드로 되돌리기".
- [ ] 일정 지정(수동·자연어·캡처) 시 자동으로 할 일이 된다.
- [ ] ⌘Enter: 일반→할 일 전환, 할 일→완료 토글.
- [ ] 되돌리기 시 완료·일정·리마인더 정리(리마인더 고아 없음).
- [ ] E2E: 전환→실행 기능 노출 / 일반 노드 숨김 / 자동 승격 / 되돌리기 정리.

## 6. 결정된 세부
- **다중 선택 "완료"**: 선택에 일반 노드가 하나라도 섞이면 **완료 버튼을 숨긴다**(전원 할 일일 때만 노출, 2026-07-16 확정). 스토어는 `done ⟹ todo` 불변식을 방어(`toggleDone`/`toggleDoneSelected`가 done 표시 시 `todo=true`도 세팅).
- **자식·형제는 실행 상태를 상속하지 않는다**(2026-07-16 확정) — 일정 노드 아래 새 자식/형제는 순수 생각(일반 노드). 명시적 전환이나 일정 지정으로만 할 일이 된다. (내가 눌러 하는 "하위까지 일정 지정" 캐스케이드는 유지.)
- 하위 완료 롤업 배지는 그대로(할 일 자식만 done 가능하므로 자연히 todo 기준).
