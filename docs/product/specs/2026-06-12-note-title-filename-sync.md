# 노트 제목 ↔ 파일명 동기화 — 기능 명세

> 작성: 2026-06-12 · 상태: 구현·검증 완료
> 출처: 사용자 보고(제목 편집이 사이드바·탭에 반영 안 됨). U3 명세에서 "후속"으로 미뤄둔 항목.

## 1. 문제 (Why)

노트 제목은 프론트매터 `title`에 저장되고, 사이드바·탭은 **파일명**을 표시한다. 둘이 분리돼 있어 에디터에서 제목을 바꿔도 파일명·사이드바·탭은 그대로다 → 제목 "시장 검토"인데 목록엔 "ㄱㅅㄷ"처럼 어긋난다. 맵은 U3에서 자동 명명을 넣었지만 노트는 빠져 있었다.

## 2. 해결 (What)

**노트 제목이 곧 파일명이 된다** (Notion·Bear 방식, 맵의 U3와 같은 철학).

- 트리거: 제목 입력이 멈추면(디바운스 600ms) 정제된 제목으로 `.md` 파일을 rename.
- 정제·충돌: `fileNameFromTitle`(autoName.ts) 재사용, 충돌 시 기존 디듀프(`이름 2.md`).
- 빈 제목이면 rename 안 함(파일명 유지). 정제 결과가 현재 파일명과 같으면 no-op.
- 첨부 노트(`.notes/`)는 그 폴더 안에서 rename(경로의 dirname 유지).
- 기존 rename 인프라 사용: 진행 중 자동저장 flush → `window.api.rename` → `renamePath`(탭 경로·제목 갱신) → 워크스페이스 refresh.
- 노드→노트 링크는 노트 본문의 `{mapId,nodeId}`로 매칭되고 인덱스는 refresh로 재구축되므로 rename에 안전(경로가 바뀌어도 링크 유지).

## 3. 범위에서 뺀 것

- 파일명을 직접 바꿨을 때 제목으로 역동기화 — 사이드바 인라인 rename은 파일명만 바꾼다(제목 프론트매터는 그대로). 단방향(제목→파일명)만.
- 맵의 자동 명명 로직 변경.

## 4. 완료 기준

- [x] 노트 제목 편집 → 600ms 후 파일명·탭·사이드바가 따라감
- [x] 빈 제목·동일 제목은 rename 안 함
- [x] 첨부 노트는 `.notes/` 안에서 rename
- [x] 노드 연동이 rename 후에도 유지
