# 작업 기록 대시보드 재기획 — 아이디어

> 작성: 2026-06-12 · 작성자: 기획(Claude) · 상태: 아이디어 (구현 전, 합의 후 선별 구현)
> 출처: 현재 대시보드 효용이 낮다는 피드백. 목적 재확인 후 재설계.

## 1. 목적 재확인 — 누가, 무엇을 위해 보는가 (2026-06-12 확정)

**1차 사용자: 본인.** 자기 일을 돌아보고 다음을 계획한다.
**그러나 결정적 맥락: 본인이 이걸 관리자에게 보여주고 피드백·평가를 받는다.** 즉 대시보드는 *내 도구*이되 **관리자 앞에 내놓을 수 있는, 신뢰할 수 있는 기록**이어야 한다.

이 이중성이 설계를 좌우한다:
- **정직하고 객관적**이어야 한다 — 자화자찬이 아니라 *공백·미달도 솔직히* 드러나야 평가 자료로서 신뢰를 얻는다(게임화/허위가 보이면 가치 0).
- **내놓을 수 있는 산출물**(주간 요약 리포트)이 1급 기능 — "이번 주 이렇게 일했습니다"를 한 장으로.
- **스스로 서사를 말하게** — 사용자가 화면을 보며 "무엇을, 왜, 어떻게 했는지" 자연스럽게 설명할 수 있는 구조.
- 동기 장치(연속 기록 등)는 *부차적* — 평가 자료의 신뢰성을 해치지 않는 선에서만.

→ 핵심: 대시보드는 **숫자(데이터)**가 아니라 **판단(인사이트)과 이야기(narrative)**를 줘야 한다. 지금은 "오늘 3분, 히트맵, 막대"라는 *데이터*만 있고, 평가 자료로서 답해야 할 질문에 답하지 않는다.

## 2. 현재의 문제 진단

- **"얼마나"만 있고 "어떻게/무엇을/왜"가 없다.** 총 시간·히트맵은 양(量)이지, 일이 잘 됐는지를 말하지 않는다.
- **성과(outcome)가 묻혀 있다.** 관리자가 가장 보고 싶은 "무엇을 끝냈나"가 한 줄로 흘러간다.
- **우선순위·계획과의 대조가 없다.** "중요하다고 표시(리마인더/일정)해 놓고 정작 집중 안 한 것"을 못 본다.
- **집중의 질이 안 보인다.** 8분짜리 20개(파편화)와 2시간 1개(딥워크)가 같은 "총 X분"으로 뭉뚱그려진다.
- **추세가 없다.** 이번 주가 지난주보다 나아졌는지, 습관이 자라는지 모른다.

## 3. 재설계 원칙

1. **인사이트 먼저, 데이터는 근거로.** 화면 맨 위는 숫자가 아니라 **자동 생성된 한 줄 요약/플래그**.
2. **성과 중심.** 시간보다 "무엇을 끝냈나"가 주인공.
3. **우선순위 대조.** 실제 투입 시간 vs 계획(일정·마감)을 나란히.
4. **질(質) 신호.** 평균 세션 길이·파편화·딥워크 블록을 드러낸다.
5. **두 층위.** 30초 스캔(요약·플래그) → 클릭 한 번 드릴다운(세션 감사).

## 3.5 [P0] 기간 네비게이션 — 지난 주·지난 날을 골라 본다 (필수 구조)

> 피드백: "지난 요일·지난 주를 선택해서 볼 수도 있는 구조인가?" → 현재는 "최근"만 보여 **불가**. 평가 맥락에선 *특정 주*를 골라 보여주는 게 필수다.

- 상단에 **기간 선택기**: `◀ 이번 주 ▶` (주 단위 이전/다음). 임의 주로 이동. "이번 주/지난주/이번 달" 프리셋.
- 선택한 기간이 **모든 패널에 일관 적용**: 요약·다이제스트·우선순위 대조·질 지표 전부 그 주 기준. (히트맵만 장기 맥락이라 전체 유지하되, 선택 주를 하이라이트.)
- **하루 드릴다운**: 히트맵 칸이나 다이제스트의 날짜를 클릭 → 그날만. ← 이전/다음 날.
- 기본값은 "이번 주". URL/상태로 기억할 필요는 없음(열 때마다 이번 주).

이건 단순 표시가 아니라 **대시보드의 골격**이다 — 모든 집계 함수가 `(from, to)` 구간을 받도록 설계한다(순수 함수, 테스트 가능).

## 4. 구성 아이디어 (위 → 아래, 우선순위 표시)

### 4.1 [P0] 인사이트 스트립 — 자동 생성 한 줄들
화면 최상단, 숫자 카드 대신 **문장형 신호** 2~4개. 규칙 기반으로 자동 생성:
- "이번 주 집중 **6h 20m** — 지난주보다 **+40%** ↑"
- "가장 깊은 몰입: 화요일 **2h 12m**, ‘결제 모듈’"
- ⚠️ "‘인증 시스템’ 마감 D-2인데 이번 주 집중 **0분**" (일정/리마인더 대조)
- ⚠️ "평균 세션 **9분** — 파편화 신호(잦은 중단)"
→ 관리자가 가장 먼저 읽는 "so what". 좋은 신호와 주의 신호를 색으로 구분.

### 4.2 [P0] 주간 작업 다이제스트 — 성과 narrative
"이번 주 / 지난주" 토글. **날짜별로 무엇을 끝냈나**를 보고서처럼:
```
화 · 2h 12m
  · 결제 모듈 — 환불 플로우 설계 완료
  · 인증 — JWT 검증 함수 작성
수 · 1h 05m
  · 결제 모듈 — 웹훅 재시도 로직
```
- 각 줄 = 한 세션의 **성과(회고)**. 클릭 → 그 세션 노트(대시보드 닫고).
- 성과 미기록 세션은 흐리게 "— 성과 미기록"으로 → 관리 신호(회고 습관 유도).
- 맨 아래 "주간 요약 노트로 내보내기" → 회고/보고용.

### 4.3 [P1] 우선순위 vs 실제 — 정렬 점검
주제(노드)별로 **계획 신호와 실제 집중 시간을 나란히**:
| 주제 | 마감 | 실제 집중(주간) |
|---|---|---|
| 인증 시스템 | D-2 🔴 | 0분 ⚠️ |
| 결제 모듈 | — | 3h 17m |
- 일정/리마인더가 걸린 노드 = "계획상 중요". 거기에 시간이 갔는지 한눈에.
- "마감 임박 + 집중 0"을 빨갛게 → 관리자의 핵심 관심사.

### 4.4 [P1] 집중의 질 — 딥워크 vs 파편화
- **하루 타임라인(간트형):** 가로축=시간(09–22시), 세션을 블록으로. 빈틈·파편·긴 블록이 한눈에. "오전엔 딥워크, 오후엔 잘게 쪼개짐" 같은 패턴.
- **질 지표:** 평균 세션 길이, 최장 세션, 일평균 세션 수, 집중 시간대(언제 잘 몰입하나).

### 4.5 [P2] 추세
- 주별 집중 시간 꺾은선(최근 8주) — 습관이 자라나/줄어드나.
- 연속 기록(streak) — 단, 압박 안 되게 "최근 N일 중 집중일".
- 히트맵은 유지(장기 패턴)하되 메인에서 **보조**로 강등.

### 4.6 [P2] 세션 감사(드릴다운)
세션 클릭 시(또는 노트에서) **목표 vs 성과**를 대비:
- "🎯 목표: JWT 검증 끝내기 / ✅ 성과: 검증 함수 완성, 만료 처리 남음"
- 관리자가 "세운 목표를 달성했나"를 본다. 목표·성과는 둘 다 노트에 있으니 추출만 하면 됨.

## 5. 데이터로 가능한가 (실현성)

대부분 **이미 있는 데이터로 가능**하다:
- 성과 = `session.reflect`. 목표 = 노트 본문 "🎯" 섹션(파싱). 시간·세션 = `session.start/end/durationSec`.
- 우선순위 대조 = 노드의 `scheduled/scheduleAt/reminderOn`(이미 있음)과 세션 시간 매칭.
- 추세·질 = 세션들의 시간 분포 집계(순수 함수, 테스트 가능).
- 인사이트 스트립 = 위 집계 위에 **규칙 몇 개**(임계값 비교)면 생성.

신규로 필요한 건 거의 없고, **집계 + 규칙 + 표현**의 문제다.

## 6. 권장 1차 범위 (효용 빠르게 끌어올리기)

만약 다 만들지 않는다면, **효용 대비 비용**이 가장 좋은 세 가지:
1. **인사이트 스트립(4.1)** — "so what"을 맨 위에. 가장 큰 체감 변화.
2. **주간 작업 다이제스트(4.2)** — 성과 중심 narrative + 주간 요약 내보내기.
3. **우선순위 vs 실제(4.3)** — 마감 임박 미집중 플래그. 관리자 핵심 질문.

나머지(질·추세·감사)는 그다음.

## 7. 접근 경로 / 저장 — work-log는 사이드바에서 숨김 (2026-06-12 확정)

세션 노트는 **사이드바에 표시하지 않는다**(피드백 반영). 접근은 두 경로로 충분:
- **대시보드 → 날짜/세션 클릭** → 그 세션 노트 열기.
- **노드의 집중 칩 / 노트 팝업** → 노드에서 바로 열기.
- work-log 폴더·파일은 트리에서 숨고, 이름 변경·이동도 불가(불변 기록). 단 인덱싱(대시보드 데이터)은 그대로 스캔.

## 8. 열어둔 결정 (합의 필요)

- ~~1차 독자~~ → **본인(관리자에게 제시)**로 확정. 톤 = 정직한 객관 리포트.
- "주간 요약 내보내기"의 형식: 노트로 내보내기 vs 화면 그대로 PDF/캡처? (보고용)
- 인사이트 규칙의 임계값(파편화 = 평균 몇 분 미만? 마감 임박 = D-며칠? 등)을 어떻게 정할까. → 기본값 제안 후 조정.
- 여러 사람(팀)을 한 곳에서 보는 건 범위 밖(지금은 단일 워크스페이스 = 한 사람). 관리자는 사용자가 *보여주는* 방식이지, 관리자가 직접 여러 명을 보는 구조는 아님.
