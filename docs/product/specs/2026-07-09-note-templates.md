# 노트 템플릿 — 기획

> 작성: 2026-07-09 · 상태: **구현 완료** (v1) — 실행 계획: [[note-templates-exec-plan]] (`docs/exec-plans/completed/2026-07-09-note-templates.md`)
> 출처: 사용자 요청 — 라이브러리에 "Note Template" 폴더, 노트 편집 중 "템플릿+"로 템플릿 삽입, 설정 토글로 켬/끔.
> 관련: [[note-feature-architecture]], [[color-system]], `src/note/EditorToolbar.tsx`(메타+), `src/store/trashStore.ts`(시스템 폴더 선례)

## 0. 용어 정리 — 시작 전에 반드시 확인

사용자 요청 원문의 "메타+"는 **이미 존재하는 별개 기능**이다 (`MetaAddButton`, `src/note/EditorToolbar.tsx:242-289`). 노트 하나에 구조화된 메타데이터 필드셋(예: 프로젝트 정보 칸)을 끼워 넣는 기능으로, `userData/meta-templates.json`에 저장되고 **워크스페이스 파일이 아니다**.

이번에 만들 "노트 템플릿"은 **노트 본문 전체를 복사해 넣는 별개 기능**이다. 사용자가 "메타+처럼"이라 한 것은 **버튼 배치·드롭다운 UI 패턴을 따라 하라는 의미**로 해석했고, 메타 시스템 내부에 얹는 것이 아니라 **툴바에 나란히 놓인 새 독립 버튼("템플릿+")**으로 설계한다. 두 시스템은 저장 위치·데이터 모델이 다르므로 합치면 오히려 사용자 요청 4번째 문단(확장자/파일명 구분, 폴더 상시 존재)과 충돌한다.

→ **결정**: "메타+"와 "템플릿+"는 툴바에 나란히 있는 두 개의 독립 버튼. 기능은 완전히 분리.

## 1. 문제 (Why)

반복적으로 쓰는 노트 구조(회의록, 회고, 주간 계획 등)를 매번 처음부터 작성해야 한다. 메타+ 필드셋은 짧은 구조화 데이터에는 맞지만 **본문 전체(제목·문단·표·체크리스트 등)를 통째로 재사용**하는 용도로는 안 맞는다.

## 2. 해결 개요 (What)

- 워크스페이스마다 숨김 시스템 폴더(가칭 `.templates/`)를 두고, 그 안의 `.md` 파일 하나하나가 템플릿 노트다.
- 노트 편집 툴바에 **"템플릿+"** 버튼을 메타+ 옆에 추가. 누르면 검색창이 달린 드롭다운이 뜨고, 항목을 고르면 **현재 커서 위치에 해당 템플릿의 본문이 삽입**된다.
- 사이드바 라이브러리에 **"Note Template" 가상 섹션**을 둬서 템플릿 파일을 보고 새로 만들 수 있게 한다(트리 탐색이 아니라 휴지통과 같은 전용 패널 방식).
- 설정에 **"노트 템플릿" 토글**을 추가. 끄면 사이드바 섹션이 숨겨진다(툴바 버튼도 함께 숨김). 켜면 시스템 폴더가 없을 시 자동 생성.

## 3. UX 설계

### 3.1 툴바 — 템플릿+ 버튼과 드롭다운

```
┌ EditorToolbar ──────────────────────────────────────────┐
│  B  I  ...  │  메타+  템플릿+  │  ...                     │
└───────────────────────────────┬──────────────────────────┘
                                 ▼ (클릭)
                    ┌ tpl-menu (검색 포함 드롭다운) ────────┐
                    │ 🔍 템플릿 검색...                      │
                    │ ──────────────────────────────────── │
                    │  📄 주간 회고                          │
                    │  📄 회의록                             │
                    │  📄 프로젝트 킥오프                     │
                    │ ──────────────────────────────────── │
                    │  ＋ 새 템플릿 만들기                    │
                    └────────────────────────────────────────┘
```

- 검색창은 `GlobalSearch`(Quick Open, `src/search/GlobalSearch.tsx`)의 `qo-input`/`qo-item` 시각 패턴을 재사용해 앱 전체에서 검색 UI가 하나의 인상으로 보이게 한다.
- 방향키(↑↓)로 항목 이동, Enter로 삽입, Esc로 닫힘 — Quick Open과 동일한 키보드 규약.
- 항목 없음(첫 실행 등): "아직 템플릿이 없어요 · ＋ 새 템플릿 만들기" 빈 상태 문구 (원칙 문서의 "비어있는 상태도 UI다").
- 선택 시 시각 피드백 없이 바로 삽입 + 드롭다운 닫힘 (원칙: "상태 변화 없이 모달/토스트만 띄우는 액션" 금지 — 삽입 자체가 상태 변화이므로 별도 토스트 불필요).

**메타+ 드롭다운도 동일하게 검색창을 상단에 추가한다** (사용자 요청 3번째 문단). 현재는 `templates.map()`로 나열만 하므로, 항목이 늘어나면 스크롤이 길어지는 문제가 이미 있다 — 이번 작업에서 함께 리트로핏.

### 3.2 사이드바 — "Note Template" 가상 섹션

`.trash`(main.ts:307 dot-prefix 스킵)와 동일하게 실제 파일은 일반 트리 워크에서 제외하되, **휴지통과 같은 방식으로 전용 UI 섹션**을 라이브러리 하단(또는 워크스페이스 트리와 별도 고정 섹션)에 노출한다. 일반 폴더처럼 트리에 섞이면 실수로 자식 노트를 만들거나 옮길 위험이 있어 **전용 패널이 트리 노출보다 안전**하다.

```
┌ 라이브러리 ──────────────────┐
│  📁 워크스페이스 트리...       │
│  ...                         │
├───────────────────────────────┤
│  🗑  휴지통 (3)                │   ← 기존
│  🧩 Note Template (5)         │   ← 신규, 설정 토글 꺼지면 이 줄 자체가 숨김
└───────────────────────────────┘
```

- 패널 열면 템플릿 목록(이름 + 수정일) + "＋ 새 템플릿" 버튼. 새 템플릿은 빈 노트 탭으로 열리고 저장 시 `.templates/`에 기록.
- 기존 노트를 "템플릿으로 저장"하는 것은 **1차 범위에서 제외**(§6) — 처음에는 템플릿 전용 패널에서 직접 작성/편집만 지원.

### 3.3 설정 — 토글

`Settings.tsx`에 다른 온오프 토글과 같은 행 스타일로 추가:

```
[ 노트 템플릿 사용    ● 켜짐 ]   사이드바에 템플릿 폴더 표시, 편집기에 템플릿+ 버튼 표시
```

- 켜짐(●)/꺼짐(○) — 원칙 문서의 "채워진 원 대 빈 원" 라디오/토글 표기 그대로.
- 기본값은 **켜짐**으로 제안(실제 파일은 숨김 폴더라 꺼도/켜도 사용자 데이터 위험 없음, 신기능 발견성이 우선). 팀에서 기본값을 끔으로 바꾸고 싶다면 손쉽게 반대로 뒤집을 수 있는 구조로 구현.

### 3.4 디자인 원칙 체크 (`/design-ui` 기준)

- [x] 색상: 신규 UI는 기존 CSS 변수만 사용(`--state-note` 보라 = 노트 정체성이므로 템플릿 아이콘도 이 색 재사용, 새 hue 추가 안 함)
- [x] 폰트: 드롭다운 항목 13px, 힌트·빈 상태 문구 12px — 11px 미만 없음
- [x] 키보드: 방향키+Enter+Esc, 토글은 Tab/Space로 접근 가능
- [x] 다크 모드: CSS 변수만 사용하므로 자동 대응
- [x] 상태 표시: 토글 켜짐/꺼짐 색 + 텍스트("켜짐"/"꺼짐") 동시 표기(색약 고려)

## 4. 데이터/저장 구조

- 위치: 워크스페이스 루트의 `.templates/` (dot-prefix → `electron/main.ts`의 `walk()`가 자동으로 트리에서 제외, `.trash`와 동일 메커니즘).
- 파일 형식: 기존 `NoteDoc`(.md + frontmatter) **그대로 재사용** — 새 파서·새 확장자 없음. `MindMapDoc.version`, `NoteDoc` 스키마 변경 없음.
- **"확장자나 파일명으로 구분" 요청에 대한 결정**: 폴더 위치 자체를 구분자로 쓴다(=`.templates/` 안에 있으면 템플릿). 커스텀 확장자(`.tmpl.md` 등)는 IO 파서 분기가 하나 더 늘고 "다른 마크다운 편집기로 열면 안 열림" 같은 부작용이 생겨 채택하지 않음. 폴더 경계만으로 "템플릿인지 아닌지"가 100% 구분되므로 사용자가 원한 효과(구분 가능함)는 동일하게 달성.
- 자동 재생성: 워크스페이스 오픈 시 또는 설정 토글을 켜는 시점에 `.templates/` 존재 확인 → 없으면 생성(Trash의 지연 생성과 달리 **즉시 생성**해야 "시스템 폴더처럼 상시 존재" 요구를 만족).
- 삽입 시 frontmatter는 제외하고 본문만 Tiptap `insertContent`로 커서 위치에 삽입(SlashMenu의 `run: (chain) => …` 체인 재사용).

## 5. 설정 스키마

- `settings.json`에 `templatesEnabled?: boolean` 추가(default `true`). **주의**: 이 프로젝트 최초의 "영속 기능 온오프 토글"이라 전례가 없다 — `electron/main.ts`의 `settingsPath()` 읽기/쓰기 로직 확장 + 새 IPC 필요.
- 렌더러 쪽은 새 `templateStore`(Zustand) 또는 기존 설정 스토어에 `templatesEnabled` 추가 — `store/`는 `ui/`를 import하지 않는다는 경계 규칙 유지.

## 6. 범위에서 뺀 것 (후속)

- 기존 노트를 "템플릿으로 저장" 버튼(현재는 템플릿 전용 패널에서 새로 작성만).
- 템플릿 하위 폴더/카테고리, 템플릿 내 변수(`{{날짜}}` 같은 플레이스홀더).
- 템플릿 단축키(예: `⌘⇧T`로 템플릿+ 바로 열기).
- 워크스페이스 간 템플릿 공유/동기화.
- 동명 템플릿 충돌 처리(1차는 저장 시 파일명 뒤 숫자 증가로 단순 처리).
