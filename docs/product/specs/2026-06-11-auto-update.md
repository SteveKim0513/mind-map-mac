# 자동 업데이트 — 기능 명세

> 작성: 2026-06-11 · 작성자: 기획(Claude) · 상태: 구현·E2E 검증 완료 (로컬 피드로 감지→다운로드→무결성 검증→재시동 호출 확인)
> 선결 조건: 코드 서명·공증 (2026-06-11 완료 — Squirrel.Mac이 서명 검증을 요구)

## 1. 문제 (Why)

dmg 수동 교체 배포는 사용자가 새 버전을 알 방법이 없고, 알아도 다운로드→교체를 직접 해야 한다. v0.2.0→v0.2.1처럼 데이터 손실급 수정이 나가도 전달이 안 된다. "데이터 신뢰"가 차별점인 제품에서 수정 사항이 사용자에게 도달하지 않는 것은 약속 위반이다.

## 2. 해결 (What)

electron-updater + GitHub Releases(`SteveKim0513/mind-map-mac`, 공개).

- **확인 주기**: 앱 시작 10초 후 1회 + 이후 4시간마다. 백그라운드 다운로드.
- **UX (오류 친절 원칙 적용)**:
  - 다운로드 완료 시에만 사용자에게 노출 — 대화상자: "새 버전 vX.Y.Z가 준비되었습니다" / [지금 재시동] [나중에]. "나중에"면 다음 종료 시 자동 적용.
  - 확인·다운로드 실패는 사용자에게 침묵 (로그만) — 업데이트 실패가 사용을 방해하면 안 됨.
  - 메뉴바: MindMap(앱 메뉴) → "업데이트 확인…" 수동 확인 (진입점 매트릭스: 앱 수준 동작 = 메뉴바). 수동 확인일 때만 "최신 버전입니다" / 실패 안내를 보여줌.
- **적용 범위**: 패키징된 정식 빌드만. `MindMap Dev`·`npm run dev`에서는 비활성.
- **테스트 훅**: `MINDMAP_UPDATE_URL` env가 있으면 generic 피드로 대체 (E2E용 — `MINDMAP_USER_DATA`와 같은 패턴).
- **빌드 변경**: mac.target에 `zip` 추가 (Squirrel.Mac은 zip으로 업데이트). 릴리즈 시 dmg·zip·blockmap·`latest-mac.yml`을 GitHub Release에 업로드 (`--publish` 또는 gh).
- 로그: electron-log로 updater 이벤트 전부 기록.

## 3. 범위에서 뺀 것 (Not now)

- 릴리즈 노트 in-app 표시 (대화상자에 버전만)
