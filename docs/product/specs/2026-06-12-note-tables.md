# 노트 표 — 기능 명세

> 작성: 2026-06-12 · 상태: 구현·검증 완료
> 출처: 사용자 보고(표를 만들 수 없음). 표준 마크다운(GFM) 기능인데 누락돼 있었음.

## 1. 문제 (Why)

노트 에디터에 표 기능이 없다. 기획·비교·일정 노트에서 표는 기본 도구이고, GFM 마크다운은 파이프 문법으로 표를 지원하므로 우리 "표준 마크다운 + 이식성" 원칙과도 맞는다.

## 2. 해결 (What)

TipTap Table 확장(table / row / header / cell) 추가.

- **삽입**: 슬래시 메뉴 "표"(3×3, 헤더 행 포함) + 툴바 버튼.
- **편집**: 셀 내 Tab/Shift-Tab 이동, 셀에서 우클릭 없이도 — 행/열 추가·삭제는 1차로는 키보드/기본 동작에 맡기고, 표 위 플로팅 컨트롤은 후속.
- **마크다운 왕복**: tiptap-markdown이 GFM 표를 직렬화/파싱(prosemirror-markdown 기반) → `| a | b |` 형태로 저장, 다른 에디터에서도 열림.
- **미리보기**(NotePopup의 markdown.tsx): GFM 파이프 표를 `<table>`로 렌더. 헤더 구분선 행(`---`)은 표시하지 않음.
- **CSS**: 에디터·미리보기 공통 테두리·헤더 강조, 셀 패딩.

## 3. 범위에서 뺀 것

- 셀 병합, 정렬 지정(GFM `:---:`), 표 리사이즈 핸들 — 후속.
- 표 안의 이미지/중첩 블록(기본 동작 허용 범위만).

## 4. 완료 기준

- [ ] 슬래시 "표"·툴바로 3×3 표 삽입
- [ ] 셀에 입력 → 저장 → 재열기 후 표·내용 유지(마크다운 왕복)
- [ ] 노드 칩 미리보기 팝업에서 표가 `<table>`로 보임
- [ ] FEATURE-INVENTORY 갱신
