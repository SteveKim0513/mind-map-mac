# 집중 세션 (Focus Session) — 기능 명세

> 작성: 2026-06-12 · 상태: v1 구현·E2E 검증 완료 (§14 보강 반영)
> 출처: 사용자 요청. 하나의 노드를 대상으로 집중 작업 시간을 측정·기록하고, 그 기록을 대시보드로 돌아보는 기능.
>
> ⚠️ 아래 §1~§13은 초안. **§14가 비판 검토(행동심리·데이터무결성·엔지니어링) 결과로 목적 달성에 필수인 보강을 담으며, 충돌 시 §14가 우선한다.**

## 0. 한눈에

노드를 골라 **집중 세션을 시작**하면 → 화면에 **카운트업 타이머**가 뜨고 → `work-log/` 폴더에 **세션 노트**가 자동 생성·해당 노드와 연동되며 → 노트 상단에 **메타(시작/종료/대상)와 템플릿(집중할 일·목표·작업 기록)**이 깔린다. 사용자는 그 노트에 작업을 기록한다. **종료**를 누르면 종료 시각·총 시간이 노트와 기록에 박히고, 누적된 세션들은 **작업 기록(Work History) 대시보드**에서 직관적으로 돌아본다.

---

## 1. 문제 (Why)

지금은 "이 노드(주제)에 내가 실제로 얼마나, 언제, 무엇을 했는지"가 어디에도 남지 않는다. 마인드맵은 *구조*를, 노트는 *내용*을, 리마인더는 *마감*을 담지만 **실제 투입한 집중 시간과 그때의 작업 로그**라는 축이 비어 있다. 이 기능은 제품의 핵심 경험인 "통합"(생각·기록·할 일을 한 곳에)에 **실행/시간 추적**을 더한다 — 노드가 곧 작업 단위가 되고, 시간이 데이터로 쌓인다.

## 2. 용어와 이름 충돌 정리 (선결)

캔버스에는 이미 **"이 노드에 집중"**(`setFocus` — 그 노드 서브트리만 보이게 하는 *보기 필터*)이 있다. 새 기능도 "집중"이라 둘이 헷갈린다. 정리:

- 새 기능 = **"집중 세션"** (시간·작업 추적). 메뉴 라벨: **"집중 세션 시작"**.
- 기존 보기 필터의 라벨을 **"이 노드만 보기"**로 변경(동작은 그대로) → "집중"이라는 단어를 더 의미 있는 쪽(작업 집중)에 양보. ADR로 기록.

## 3. 목표 / 범위에서 뺀 것

**목표**
- 한 번에 **하나의** 활성 세션. 노드 1개 대상. 카운트업(스톱워치) 방식.
- 세션 노트 자동 생성 + 노드 양방향 연동 + 메타·템플릿.
- 종료 시 시간 데이터 확정. 앱 재시작에도 진행 중 세션 복구.
- 작업 기록 대시보드: 일별 집중량 히트맵 + 최근 세션 타임라인 + 노드별 누적.

**범위에서 뺀 것(후속 후보)**
- 포모도로식 카운트다운/목표 시간/자동 휴식 알림.
- 일시정지(pause) — v1은 시작·종료만(아래 "일시정지" 결정 참고).
- 다중 동시 세션, 팀 공유, 외부 캘린더 연동.
- 유휴(idle) 자동 감지 후 시간 차감.

## 4. 진입점 (entry-point 매트릭스 준수, decisions/0004)

- **시작**: 노드 우클릭 → **"집중 세션 시작"** (카탈로그 = 우클릭 메뉴). + `⌘K` 명령. (선택 툴바 버튼은 빈도 검증 후 후속.)
- **종료**: 타이머 위젯의 **종료** 버튼. + `⌘K` "집중 세션 종료".
- **대시보드 열기**: 사이드바 하단 **"작업 기록"** 항목 + `⌘K` "작업 기록 열기".
- **세션 노트 열기**: 타이머 위젯의 대상 노드명 클릭 → 세션 노트 활성화(오른쪽 분할).

## 5. 세션 생명주기

### 5.1 시작 (Start)
1. 대상 노드 확정 → 이미 활성 세션이 있으면 "진행 중인 세션을 종료하고 새로 시작할까요?" 확인.
2. `work-log/` 폴더 **보장**(없으면 생성, 있으면 그대로) — *원 요청의 "없으면 생성, 있으면 노트 생성"을 이렇게 해석: 폴더는 항상 보장하고, 노트는 매 세션 새로 만든다.*
3. 세션 노트 생성: `work-log/<제목>.md`, `emptyNote` 기반 + 아래 4단 구조. **노드와 자동 연동**(`addLinkToNoteFile`), 맵은 먼저 디스크 보장(`ensureMapPersisted`).
4. 노트를 **오른쪽 분할**에서 열어 바로 기록 가능하게.
5. 타이머 위젯 표시(시작 시각 기준 카운트업). 활성 세션 상태를 localStorage에 저장(복구용).

**노트 제목(=시작 시각)**: 정렬·식별이 쉽도록 `YYYY-MM-DD HH:mm 집중 · <노드텍스트>` 형식 권장.
원 요청("제목은 시작 시간")을 지키되 노드텍스트를 덧붙여 목록에서 알아보게 한다. 제목→파일명 동기화(기존 기능)로 파일명도 동일해진다.

### 5.2 진행 중 (During)
- 위젯에 경과 시간(`mm:ss`, 1시간 넘으면 `h:mm:ss`) 1초 간격 갱신 + 대상 노드명 + 종료 버튼.
- 사용자는 세션 노트의 "작업 기록"에 자유롭게 기록(표·이미지·체크리스트 등 노트 전 기능 사용).
- 노드 캔버스에는 대상 노드에 **세션 진행 배지**(작은 점멸 점/타이머 아이콘)를 표시해 "지금 이 노드를 집중 중"임을 시각화(선택적, 가벼운 표시).

### 5.3 종료 (End)
1. 종료 시각 기록 → `durationSec = end - start`.
2. 세션 노트 **프론트매터의 session** 갱신(end, durationSec) + 본문 상단 **메타 블록의 "종료" 줄**을 앱이 자동으로 채움.
3. **세션 기록 확정**: 노트 프론트매터가 진실의 원천(아래 6.2). 대시보드는 `work-log/*.md`를 스캔해 집계.
4. 위젯 닫고 요약 토스트("32분 집중 완료 · <노드텍스트>"). 노드의 진행 배지 제거.

### 5.4 복구 (Recovery)
- 활성 세션은 localStorage에 `{start, nodeId, mapId, notePath, nodeText}`로 저장.
- 앱 재시작 시 활성 세션이 있으면 위젯을 복구해 `now - start`로 이어서 표시 + 작은 안내("집중 세션 진행 중 — 계속하거나 종료할 수 있어요").
- 비정상 종료로 "종료를 못 누른" 세션도 다음 실행에서 사용자가 종료 시점을 정해 마무리(또는 마지막 노트 수정시각으로 추정 제안).

## 6. 데이터 모델 — 메타는 "데이터", 본문은 "사용자 글" (수정 불가 보장)

> **결정(사용자 요청)**: 자동 생성되는 메타(시작·종료·대상)는 **수정 불가**. 이를 구조적으로 보장하기 위해 메타를 마크다운 본문 텍스트가 아니라 **프론트매터 데이터 + 그것을 읽어 그리는 읽기 전용 배너**로 둔다. 본문(편집 가능)에는 메타가 아예 없다 → 손으로 고칠 대상 자체가 없음.

### 6.1 프론트매터 `session` (진실의 원천 + 수정 불가 메타의 원본)
기존 `id / title / links`에 **`session`** 추가:
```
session: {"nodeId":"...","mapId":"...","nodeText":"출시 준비 로드맵",
          "start":"2026-06-12T14:30:00","end":"2026-06-12T15:02:11","durationSec":1931}
```
- `serializeNote`/`parseNote`가 이 필드를 왕복하도록 확장(없으면 일반 노트와 동일 → 하위호환).
- 진행 중에는 `end:null, durationSec:0`. 종료/총시간은 **앱만** 기록(사용자는 손댈 수 없음).
- **대시보드는 이 프론트매터만 읽어 집계** → 노트 파일이 곧 데이터. 별도 DB 불필요(이식성·복원력↑). 성능을 위해 `work-log/.index.json` 캐시를 둘 수 있으나 v1은 스캔으로 충분.

### 6.2 세션 노트 화면 구성 = [읽기 전용 메타 배너] + [편집 가능한 본문]
세션 노트(`work-log/*.md`)를 열면 NotePane이 **세션이면** 상단에 메타 배너를 그린다(프론트매터 `session`을 읽어 렌더, `contentEditable` 아님 → 수정 불가):
```
┌─────────────────────────────────────────────┐
│ ◷ 집중 세션              ● 24:13  [종료]      │  ← 진행 중: 라이브 타이머 + 종료
│ 시작 14:30 · 대상 「출시 준비 로드맵」          │     (종료 후: "14:30–15:02 · 32분")
└─────────────────────────────────────────────┘
```
- 진행 중에는 배너에 **라이브 경과 시간 + 종료 버튼**을 함께 둔다(플로팅 위젯과 동일 동작 — 노트를 보며 바로 끝낼 수 있음).
- 종료되면 배너가 "시작–종료 · 총시간"으로 고정. 대상 노드명 클릭 → 노드로 이동(revealNode).
- 외부 마크다운 에디터로 열면 메타는 **프론트매터에 그대로 보존**(파일 안에 있음). 본문은 깨끗한 사용자 글만.

### 6.3 본문 템플릿 (편집 가능 — 메타 없음)
```markdown
## 집중할 일
(무엇에 집중하나)

## 목표
(이 세션에서 끝낼 것)

---

## 작업 기록
(여기에 진행하며 기록)
```
섹션 헤더는 사용자가 바꿔도 무방(자기 노트). **메타만 잠근다.**

### 6.3 활성 세션(휘발성 + 복구)
- `uiStore.focusSession: {start, nodeId, mapId, notePath, nodeText} | null` (실시간 화면).
- localStorage 미러(복구용).

## 7. 타이머 위젯 UX — 위치 결정

**결정: "사이드바 하단 도킹 + 사이드바 숨김 시 좌하단 플로팅 필"** (같은 화면 앵커로 연속성).

근거:
- 집중 세션은 **앰비언트(항상 보이되 방해 안 됨)** 요소다. 그런데 **집중할 때 사이드바를 숨기는** 경우가 많다 → 사이드바 하단에만 두면 정작 집중 중 타이머가 사라진다(나쁨).
- 그래서: 사이드바가 **보일 때**는 `.sidebar-foot` 위 영역에 **도킹**(통합되어 떠다니지 않음), **숨길 때**는 동일한 **좌하단 좌표에 플로팅 필**로 전환. 두 상태의 앵커가 같아 토글해도 위치가 안 튄다.
- 좌하단은 현재 비교적 비어 있음(줌 툴바=하단중앙, 브레드크럼=좌상단). 사이드바 발(foot)도 좌하단이라 연속성 최적.

**위젯 구성(접힘 기본)**: `● 24:13  출시 준비 로드맵   [종료]`
- 좌측 점멸 점(진행 표시) + 경과 시간 + 대상 노드명(클릭 → 세션 노트 열기) + 종료 버튼.
- hover/클릭 시 살짝 확장: 시작 시각, "노트 열기", "노드로 이동" 보조 액션.
- 비활성(세션 없음) 시 위젯 자체를 표시하지 않음(군더더기 0).

**대안(검토 후 기각)**: 상단 탭바 영역 = 탭과 시선 경쟁. 화면 중앙 플로팅 = 캔버스 작업 방해. 우하단 = 일부 OS 위젯/스크롤바와 충돌 가능.

## 8. 작업 기록 대시보드 (Work History)

**여는 법**: 사이드바 하단 "작업 기록" + `⌘K`. **새 탭 종류 `'history'`**로 연다(맵·노트와 동급 탭, 경로 없는 특수 탭).

**레이아웃(위→아래)**
1. **요약 스트립**: 오늘 / 이번 주 집중 시간, 세션 수, 연속 기록(streak) — 큰 숫자 카드.
2. **연간 히트맵**: GitHub 잔디 스타일. 칸=하루, 진하기=그날 총 집중 분. 한눈에 패턴 파악(가장 직관적). 칸 hover → 그날 분/세션 수, 클릭 → 그날 세션들로 필터.
3. **최근 세션 타임라인**: 카드 리스트 — `날짜·시간 | 32분 | 노드텍스트 | 목표 한 줄`. 클릭 → 해당 세션 노트 열기. 노드명 클릭 → 노드로 이동(revealNode).
4. **노드별 누적**: 어떤 주제에 시간을 가장 썼나(가로 막대 톱 N). 클릭 → 그 노드의 세션만 필터.

**데이터 출처**: `work-log/*.md` 프론트매터 `session` 스캔 → 메모리 집계. 진행 중 세션도 실시간 반영.

**상태/엣지**: 기록 0건 → 빈 상태 안내("노드에서 '집중 세션 시작'으로 첫 기록을 남겨보세요"). 삭제된 노드의 세션 → nodeText 스냅샷으로 표시(이동은 비활성).

## 9. 엣지 케이스 / 결정 사항

- **일시정지 없음(v1)**: 시작·종료만. 잠깐 자리를 비워도 시간은 흐른다(정직한 기록). pause/idle 차감은 후속. — *이 결정이 마음에 안 들면 v1에 일시정지를 넣을 수 있음(아래 확인 요청).*
- **세션 중 대상 노드 삭제**: 세션은 유지(노트의 nodeText 스냅샷으로 계속). 연동 칩만 dead link 처리.
- **세션 중 같은 노드에 또 시작**: 단일 세션 원칙 → 기존 종료 확인.
- **자정 넘김**: durationSec는 연속 계산. 히트맵 귀속은 *시작일* 기준(단순·일관).
- **work-log 폴더를 사용자가 지움/이름변경**: 다음 시작 때 다시 보장 생성. 대시보드는 현재 폴더 기준.
- **매우 짧은 세션(예: 10초)**: 그대로 기록(필터는 대시보드에서 선택적으로).

## 10. 단계별 구현 계획 (기획 승인 후)

## 14. 비판 검토 반영 — 목적 달성을 위한 필수 보강 (2026-06-12)

세 독립 관점(행동심리 / 데이터 무결성 / 엔지니어링)으로 "이 기획이 목적을 달성하기에 충분한가"를 검토. **결론: 초안은 '정확한 시간 기록기'로는 충분하나 목적('노드=작업단위, 시간=데이터, 돌아보기')에는 미달.** 세 관점이 독립적으로 같은 곳을 가리킨 항목이 핵심이다.

### 14.1 세 관점이 수렴한 P0 — v1 필수

**A. 노드 누적 시간 칩을 v1으로 (심리+데이터 동시 지목).**
캔버스 노드 옆 "⏱ 3회·2h15m" 칩이 (심리) 대시보드를 안 가도 *항상 보이는 누적의 증거 = 습관 보상*이고, (데이터) "노드=작업단위"의 *문자 그대로의 구현*이다. 만들면 "노드→누적시간" 질의를 데이터 모델이 1급으로 다루게 되어 아래 B·C를 강제한다. → **v1.5 아님, v1.**

**B. 서브트리 롤업 (데이터 — 목적 직결).**
사용자는 보통 잎 노드("로그인 버그")에서 세션을 돌리지 부모 주제("인증")에서 안 돌린다. "어디에 시간을 썼나"를 *주제 단위*로 답하려면 부모 누적 = 자신 + 모든 자손. 그러려면 집계 시점에 부모 체인이 필요한데 초안엔 없다. → **세션 종료 시 `session`에 조상 체인 스냅샷(`ancestorIds`)을 박는다.** 맵이 닫혀/삭제돼도 롤업 가능. 칩·대시보드 모두 이걸로 주제 단위 집계.

**C. 종료 = '쌓임'을 보여주는 완료 카드 (심리 — 습관 루프의 보상 고리).**
현재 종료는 1.5초 토스트 = 소멸성 알림이지 보상이 아니다. 습관은 *행동 직후 즉각 보상*으로 굳는다. → 토스트 대신 **완료 카드**: "32분 · 오늘 누적 1h20m · 🔥streak +1 · 이 주제 누적 2h47m" + **회고 한 줄 입력란을 기본 노출**(skip 가능, placeholder는 질문형 "방금 뭐가 풀렸나?"). 보상을 대시보드까지 미루지 않고 *그 자리*에서 준다.

**D. 회고는 목표와 분리, 기본 노출 (심리).**
초안의 "회고 선택·빈칸이면 목표로 대체"는 사실상 회고를 죽인다. 회고(실제)와 목표(예상)는 다른 데이터이고 *그 간극이 회고의 핵심 가치*다. → 대체 금지. C의 완료 카드에서 기본 노출(여전히 skip 가능). `session.reflect`에 저장(읽기 전용 메타).

### 14.2 데이터 무결성 P0

**E. 세션 귀속을 NoteLink 구조로 통일.**
별도 매칭 코드 만들지 말 것. `session.link = {mapId, nodeId, mapPath?, nodeText}` — 기존 `revealNode`·인덱스 재구축·stale-path 복원(`noteLinks.ts`)을 그대로 재사용. `nodeText`는 **표시용 스냅샷이지 식별 키 아님**(동명 노드·리네임 때문). 식별은 `(mapId,nodeId)`.

**F. `sessionId` + 깨진 세션 비은닉.**
세션마다 고유 `sessionId`(파일 복사 시 중복 집계 방지의 dedup 키). 프론트매터 파서는 관대해서(JSON 실패→raw 문자열) **손상된 session을 조용히 누락**할 수 있다 → 집계 과소. 파싱 실패 세션은 **대시보드에 "읽을 수 없는 세션 N건"으로 노출**(silent drop 금지). 집계 원천은 **기존 workspace noteIndex에 `session`을 실어** 메모리 필터(매번 전량 재파싱 금지). work-log 폴더가 아니라 **`session` 마커 유무로 워크스페이스 전역 판정** → 폴더 이동에도 강건.

**G. 진행중/추정/노드없음을 1급 플래그로.**
`end:null`=진행중(집계서 분리 표기), 비정상 종료 추정은 `estimated:true`(대시보드서 점선/주석 구분). 노드 없는 임시 집중은 **v1에서 막는다**(목적='노드 귀속'에 충실) — 단 이 결정을 명시. (후속: 종료 후 "이 시간을 어느 노드에?" 귀속.)

### 14.3 엔지니어링 P0 — 구현 성패를 가르는 격리 규칙

세션 노트를 "일반 노트 인프라에 그대로 얹는" 전제는 옳지만(이식성·재사용), **일반 노트의 두 자동 effect에서 격리**해야 한다. §10 1단계에 명시 항목으로 추가:

**H. 제목→파일명 동기화에서 세션 노트 제외.** `NotePane`의 rename effect는 모든 노트에 도는데, 제목=시작시각(`:` 포함)이면 `fileNameFromTitle`이 `:`을 공백 치환 → 제목≠파일명 → **매 로드 rename 루프**. → 세션 노트는 rename effect early-return, **파일명은 생성 시 1회 확정 후 불변**, 제목 input 읽기전용. 시작시각의 진실원천은 파일명이 아니라 `session.start`.

**I. 800ms 자동저장 vs 종료 write 경합.** 종료가 `session.end`를 쓴 직후 자동저장이 옛 store로 덮어써 종료시각 유실 가능. → `session`을 `NoteDoc`에 싣되 **store에서 immutable**(setBody/setTitle이 못 건드림), 종료는 전용 action + 즉시 flush, write 직렬화(리마인더 osascript 직렬화 전례 적용). `serializeNote`는 **session 없는 일반 노트에 session 줄을 추가하지 말 것**(오염 방지) — 조건부 직렬화 + 일반노트 왕복 동일성 회귀 테스트.

**J. 시간 정확성 — 절전/DST/시간대.** `now-start`/로컬 ISO는 **노트북 절전 시 거짓 시간**(14:30 시작→다음날 깨움→18시간 기록). → `start`/`end`를 **epoch ms(또는 TZ 포함 ISO)**로 저장, duration=epoch 차. **장시간(>Nh) 세션은 종료 시 sanity 확인/보정**("18시간이 맞나요?"). 정상 종료 경로에도 적용(절전은 정상 경로에서 발생). 히트맵 귀속=시작일(단순화, ADR 명시).

**K. `'history'` 탭 vs 경로 기반 탭.** persist/hydrate/중복판정이 전부 path 기반 → 경로 없는 탭은 재시작 시 소실·중복 열림 위험. → **sentinel path(`history://`) + 싱글톤 가드**, TabKind 분기를 makeTab/select 한 곳에 격리.

### 14.4 streak 역효과 방어 (심리)

초안은 streak를 동기 장치로 *전제*만 하고 부작용 방어가 0. → (1) **연속일이 아니라 "최근 7일 중 N일" 또는 1~2일 유예(freeze)** — 하루 빠짐의 what-the-hell 이탈 방지. (2) streak·"집중일" 카운트에 **최소 유효시간(예 5분)** — 10초 세션 게이밍 차단(기록은 하되 streak엔 불산입). (3) 히트맵 빈칸/옅음을 **중립적**으로(죄책감 트리거 금지).

### 14.5 시작 마찰 완화 (심리)

초안은 시작 시 노트를 오른쪽 분할로 강제로 들이민다 = "기록 모드" 강제 진입 마찰. → **"1초 시작"이 기본**: 시작은 타이머만 돌리고 노트·연동·파일은 백그라운드 보장. **노트는 위젯의 "노트 열기"로 당겨(pull)** 연다. 진행중 템플릿도 4칸→**목표 한 줄 + 기록 2칸**으로 축소(형식 채우기 부담↓, 목표는 종료 회고와 짝지어 예상vs실제 가치 유지).

### 14.6 v1 범위 재확정

초안 §10에 더해 **v1 필수로 끌어올림**: 노드 누적 칩(A) · 서브트리 롤업(B) · 완료 카드+회고 기본노출(C·D) · 세션 노트 격리 규칙(H·I) · epoch 시간+sanity(J). 이 다섯이 빠지면 결과물은 "세션 목록"이지 목적이 말한 "집중 분석/돌아보기"가 아니다.

**한 줄 결론**: 초안은 인프라로는 견고하나 *습관 루프의 보상*(C)과 *주제 단위 귀속*(A·B)이 비어 목적의 절반이 빈다. 위 A~K를 v1에 반영하면 충분하다. 단 하나만 고른다면 — **종료를 '쌓임을 보여주는 완료 카드'로** 만드는 것(C).
