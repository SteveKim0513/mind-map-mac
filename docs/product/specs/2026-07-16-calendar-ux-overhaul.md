## 1. 문제 — 7개 개선 요청

| # | 현재 | 불만 / 요구 |
|---|---|---|
| 1 | 일간 카드의 `집중`(라벨+테두리 pill)과 `완료`(아이콘만, 투명)가 **서로 다른 모양** | 같은 자리의 두 버튼이 짝으로 안 읽힘. 텍스트를 **둘 다 넣거나 둘 다 빼서** 통일 |
| 2 | 스케줄 클릭 → `revealNode`가 곧바로 **해당 마인드맵 탭을 열고** 노드를 중앙정렬 | 클릭 한 번에 맵이 통째로 열리는 게 과함. **먼저 그 노드+하위만 미리보고**, 원할 때만 맵을 연다 |
| 3 | 주간 빈 슬롯 클릭 → 텍스트 입력 → `captureScheduled`로 **새 노드를 (첫 열린 맵에) 생성** | 매번 새 노드가 생기는 게 부담. **기존 노드를 검색·선택**이 기본, 새로 만들기는 별도 메뉴로 **오늘의 생각**에. 종일 섹션에서도 생성 가능해야 |
| 4 | 일간 뷰는 **생성 진입점 없음**(카드 리스트만) | 주간과 동일한 생성 흐름 필요 |
| 5 | 월간 셀 클릭 = **일간 이동**(생성 없음) | 주간과 동일한 생성 흐름 필요 |
| 6 | 주간은 슬롯 클릭 위치로 시각이 정해짐. 일/월은 **시각 정보가 없음** | 일/월 생성 시 **종일/시간 지정**을 자연스럽고 간결한 UX로 |
| 7 | 월간 셀은 3개까지만 표시 후 `+N`(하드코딩, `overflow:hidden`) | `+N` 대신 **셀 안 스크롤**로 전부 보기 |
| 8 | 주간 뷰에서 스케줄 등록 시 **시간 눈금과 블록 위치가 안 맞음**(버그) | 눈금↔블록 정렬 수정(§3.8) |

**공통 방향**: 캘린더를 "맵을 여는 관문"이 아니라 **일정을 미리보고·잡고·되돌아보는 시간 렌즈**로. 클릭은 무겁게 열지 말고 **미리보기(peek) → 필요 시 열기**로 단계화(원칙 3, 점진적 공개).

## 2. 해결 개요 (설계 원칙)

1. **클릭 = 미리보기 먼저, 열기는 2단계.** 스케줄 클릭 → 그 노드+하위만 읽기전용 미리보기 → 다시 누르면 우측 분할로 실제 맵.
2. **생성 = 기존 노드 우선.** "새 노드 남발" 대신 **기존 노드 검색·선택**이 기본. 새로 만들기는 **오늘의 생각.mind**(전역 캡처와 동일 인박스)로 흘려보낸다.
3. **종일 기본 · 시간 옵트인.** 일/월 생성은 종일이 기본, 시간은 필요할 때만 드러낸다(원칙 3).
4. **스키마·리마인더 불변.** 새 필드 없이(종일=자정 관례), 모든 변경은 `mapStore` 액션 경유.
5. **뷰별 밀도에 맞춘 컨테이너.** 같은 미리보기·생성 로직을 일간=서랍 / 월간=하단 패널 / 주간=팝업으로 담는다(공간 제약 차이).

## 3. 항목별 상세

### 3.1 (요청 1) 일간 `집중`·`완료` 버튼 통일

**현재**: [CalendarView.tsx:357-365](../../../src/calendar/CalendarView.tsx#L357-L365) — `.cal-daycard-focus`(아이콘+"집중"+테두리+배경 pill)와 `.cal-daycard-check`(아이콘만, 투명, 라벨 없음). [styles.css:5693-5714](../../../src/styles.css#L5693-L5714).

**변경 (확정)**: **둘 다 아이콘+라벨 pill**로 통일 — `[⏱ 집중] [✓ 완료]`(2026-07-16 사용자 확정). 하나의 베이스 클래스(`cal-daycard-act`)에 hover 색만 modifier(집중=`--primary`, 완료=`--state-done`)로 분기. 일간 뷰는 상세·여유 공간 뷰라 라벨이 발견성을 돕고(특히 아이콘만이던 `완료`가 모호했음), 두 버튼이 한 짝으로 읽힌다.

- 게이팅은 그대로: `집중`은 일정 있는 할 일에만(포커스 게이트), `완료`는 할 일 노드에만. 완료 상태면 라벨을 `완료됨`/취소 토글로.
- 주간·월간 인라인 액션은 컴팩트 밀도라 아이콘-only 유지(뷰 간 통일이 아니라 **뷰 내 짝 통일**이 이번 요구).

### 3.2 (요청 2) 스케줄 클릭 → 서브트리 미리보기 → 2차 클릭 시 우측 분할

기존 `actions.reveal(it)`(→ `revealNode`, 즉시 맵 열기)를 **2단계**로 바꾼다.

**공통 — 읽기전용 서브트리 미리보기 (`SubtreeMiniView`, 신규)**
- 순수 함수 [`layout(doc, sizes, focusRootId)`](../../../src/layout/treeLayout.ts#L34)를 호출해 그 노드+하위만 배치한 좌표를 얻고, **정적 SVG(엣지+노드)**로 렌더. 편집·store 바인딩 없음.
- 열린 맵: 해당 탭 `mapStore`의 `doc` 사용. **닫힌 맵**: `window.api.readFile(path)` + `io/formats.deserialize`로 doc 확보(캘린더가 이미 [collectAgendaCached](../../../src/calendar/collectAgendaCached.ts)로 하는 스캔과 동일 경로) → **탭을 열지 않고도** 미리볼 수 있어 현재보다 가벼움.
- 크기(`sizes`)는 읽기전용이라 측정 대신 텍스트 길이 기반 추정값 사용(구현 세부).

**1차 클릭 — 뷰별 컨테이너** (트리거 = **카드/칩 전체 클릭**, 2026-07-16 확정 — 아직 "클릭=맵 열기" 근육 기억이 없어 전용 화살표 없이 전체 클릭으로 단순화)
- **일간**: 클릭한 카드 **바로 아래로 서랍(accordion)**이 펼쳐지며 미리보기. 다시 카드를 누르면 접힘.
- **월간**: 셀은 좁으니 **달력 그리드 하단에 패널**로 미리보기(어느 날 어느 스케줄인지 헤더 표기). 다른 스케줄 클릭 시 패널 내용 교체.
- **주간**: 시간 그리드에 공간이 없으니 **플로팅 팝업**으로 미리보기. **✕ 닫기 버튼 필수**(요청 명시). 바깥 클릭/Esc로도 닫힘.

**2차 클릭 — 우측 분할로 실제 맵**
- 미리보기 안에서 **그 스케줄(루트) 노드를 다시 누르면** [`openInRight(path, content)`](../../../src/store/sessionStore.ts#L317)로 **우측 분할**에 맵을 열고 해당 노드를 포커스. `content`는 `await window.api.readFile(path)`.
- 발견성 위해 미리보기에 `[이 맵을 오른쪽에 열기 →]` 보조 버튼도 함께.

```
일간 서랍 예시
┌ 09:00  팀 회고 · 제품맵 ───────────────── [⏱ 집중] [✓ 완료] ┐
│ ┌──────────────── 미리보기(읽기전용) ─────────────────────┐ │
│ │   ● 팀 회고            ← 다시 누르면 우측 분할로 맵 열림   │ │
│ │    ├ 지난 스프린트 리뷰                                  │ │
│ │    ├ 블로커 3건                                          │ │
│ │    └ 다음 액션                          [오른쪽에 열기 →] │ │
│ └─────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 (요청 3) 주간 생성 재설계 — 기존 노드 검색·선택 + 새로 만들기(오늘의 생각) + 종일 생성

**현재**: 빈 슬롯 → 인라인 텍스트 → [`captureScheduled`](../../../src/store/mapStore.ts#L355)로 **새 root 노드를 첫 열린 맵에** 생성.

**변경 — `SchedulePicker`(신규) 팝업** (빈 슬롯 클릭 시 그 날짜·시각을 미리 채워 오픈)
- **① 기존 노드 검색·선택(기본).** 워크스페이스 전체 맵의 노드를 검색 목록으로. 선택 → 그 노드에 [`setScheduleAt(id, iso)`](../../../src/store/mapStore.ts#L585)(자동 할 일 승격). **닫힌 맵 노드도 반드시 대상**(2026-07-16 확정) — 고르면 **백그라운드로 그 맵을 로드**(탭 오픈, 필요 시 조용히)한 뒤 store 액션으로 일정을 잡는다. v1의 "열린 맵만 편집" 한계를 생성 흐름에서 해소. 로드는 `revealNode`의 맵-오픈 경로를 재사용하되 캔버스 포커스는 생략.
- **② 새로 만들기(별도 행).** 목록 맨 위 index-0에 `✚ "<검색어>" 새로 만들기 → 오늘의 생각`([NoteLinkPicker](../../../src/note/NoteLinkPicker.tsx)의 `.picker` 패턴 재사용). 선택 → **워크스페이스 루트 `오늘의 생각.mind`**([capture:targetPath](../../../electron/main.ts#L694))에 새 root 노드 append + 일정 지정. 활성 맵을 오염시키지 않고 전역 캡처(⌥Space)와 같은 인박스로 모인다.
- **③ 종일 섹션 생성.** 현재 드롭 이동만 되는 종일 스트립([CalendarView.tsx:666-688](../../../src/calendar/CalendarView.tsx#L666-L688))에 **클릭 → 같은 SchedulePicker**(종일=자정으로 생성) 배선.

```
SchedulePicker
┌ 일정 추가 · 7/16(수) 14:00 ─────────────────┐
│ 🔎 [ 노드 검색…                        ]     │
│ ─────────────────────────────────────────── │
│ ✚ "회의 준비" 새로 만들기 → 오늘의 생각       │  ← 검색어로 새 노드
│ ─────────────────────────────────────────── │
│ ● 팀 회고             제품맵                  │  ← 기존 노드(검색 결과)
│ ● 회의 자료 정리       업무맵                  │
│ ● 회고 액션           제품맵                  │
└──────────────────────────────────────────────┘
   (주간: 시각은 클릭한 슬롯으로 고정 → 시간 컨트롤 없음)
```

- 노드 검색은 **`collectAgendaCached`의 워크스페이스 스캔 캐시를 공유**(도메인 경계상 `search/`를 직접 import하지 않음, §4).

### 3.4 (요청 4) 일간 뷰 생성 — 동일 SchedulePicker

- 일간 헤더(또는 빈 상태)에서 **`+ 일정 추가`** 진입점 → 같은 `SchedulePicker`. 날짜는 그 일간의 날짜.
- 클릭 위치로 시각이 안 정해지므로 **요청 6의 시간 컨트롤**을 함께 노출.

### 3.5 (요청 5) 월간 뷰 생성 — 동일 SchedulePicker (셀-이동 계약 보존)

- 월간 셀 클릭 = 일간 이동은 **v1 계약·E2E라 유지**([계약 근거](2026-07-15-calendar-execute-insight.md) 구현노트 (2)). 생성 진입은 이와 구분되게 **셀 hover 시 우상단 `+` 버튼** → 같은 `SchedulePicker`. 날짜는 그 셀의 날짜.
- 시각 없음 → **요청 6의 시간 컨트롤** 노출.

### 3.6 (요청 6) 일/월 생성의 시간 지정 UX — 종일 기본 · 시간 옵트인

주간은 슬롯 위치로 시각이 정해지지만(§3.3), 일/월은 시각이 없다. **간결·자연스럽게**:

- `SchedulePicker` 하단에 **종일/시간 세그먼트 토글 + 시간 필드**(2026-07-16 확정). **기본은 `종일`**(원탭 완료). `시간 지정` 선택 시에만 인라인 **시간 입력 필드(HH:mm)** 노출(점진적 공개). 프리셋 칩이 아니라 필드 하나로 — 임의 시각을 바로 칠 수 있게.
- **새로 만들기 경로는 자연어도 허용** — 입력 텍스트에 "오후 3시", "내일 3시"가 있으면 기존 [`parseScheduleText`](../../../src/store/parseNodeText.ts) 재사용해 자동 시각화(전역 캡처와 동일 동작). 자연어가 없으면 토글 기본값(종일) 적용.
- 종일=`scheduleAt` 자정(00:00) 관례 → **신규 필드 없음**. 시간 지정=`rescheduleToMinute`로 시각 세팅([calendarMath.ts:95](../../../src/calendar/calendarMath.ts#L95)).

```
시간 컨트롤 (일/월에서만)
시간:  [ 종일 ● ]  [ 시간 지정 ○ ]      ← 기본 종일(원탭)
                       └ 시간 지정 선택 시 →  [ 09:00 ]   ← HH:mm 입력 필드
```

### 3.7 (요청 7) 월간 `+N` → 셀 내부 스크롤

**현재**: `dayItems.slice(0, 3)` + `+N` 버튼([CalendarView.tsx:823-832](../../../src/calendar/CalendarView.tsx#L823-L832)), `.cal-month-chips { overflow:hidden }`([styles.css:5886](../../../src/styles.css#L5886)).

**변경**: `slice(0,3)`/`+N` 제거. 칩 컨테이너를 **`overflow-y:auto`**로. 그리드 정렬 유지 위해 **셀(주 행) 높이는 고정**하고 넘치면 칩 리스트만 스크롤. **셀 높이를 92px→120px로 상향**(2026-07-16 확정)해 스크롤 없이 보이는 개수를 늘려 스크롤 마찰 완화. 스크롤바는 저대비(hover 시 노출). 칩 각각은 클릭 가능(§3.2 미리보기), 셀 빈 영역/날짜 숫자는 기존대로 일간 이동.

### 3.8 (버그) 주간 시간 눈금 ↔ 블록 위치 어긋남

**증상(사용자 보고)**: 주간 뷰에서 스케줄이 등록되면 **맨 왼쪽 시간 눈금(가이드)과 블록의 세로 위치가 안 맞는다.**

**원인(코드 확인)**: 시간 눈금(`.cal-wk-hour`)과 슬롯(`.cal-wk-slot`)은 **고정 40px/시간**인데, 블록 위치는 컬럼 높이의 **백분율**로 계산된다([CalendarView.tsx:576-577](../../../src/calendar/CalendarView.tsx#L576-L577), `top: (topMin/WEEK_GRID_MINUTES)*100%`). 주간 그리드(`.cal-wk-grid`는 `flex:1`, [styles.css:5793](../../../src/styles.css#L5793))가 `18시간×40px=720px`보다 **키가 크면** CSS grid가 컬럼을 그 높이로 늘리고, 블록 백분율은 늘어난 높이에 매핑돼 **고정-픽셀 눈금에서 아래로 점점 밀린다**(창이 클수록·시각이 늦을수록 편차↑). 스케줄이 없으면 비교 대상이 없어 안 보이다가 **등록되는 순간 드러난다.** 빈 슬롯 캡처 입력창(`top:%`, [CalendarView.tsx:741](../../../src/calendar/CalendarView.tsx#L741))도 동일 증상.

**해결**: 눈금·슬롯·블록·캡처가 **하나의 고정 px/분 스케일**을 공유하게 한다.
- `PX_PER_MIN` 상수(예: 40px/시간 = `2/3 px/min`) 도입 → 그리드 콘텐츠 높이 = `WEEK_GRID_MINUTES × PX_PER_MIN`(고정 720px)로 못 박고(컬럼이 늘어나지 않게), 블록 `top`/`height`와 캡처 `top`을 **백분율이 아닌 px**(`startMin × PX_PER_MIN`)로 배치.
- 리사이즈/드래그의 분↔px 역산([CalendarView.tsx:552-555](../../../src/calendar/CalendarView.tsx#L552-L555))도 같은 상수를 쓰게 통일(현재는 `rect.height` 기반이라 스케일이 어긋나면 함께 틀어짐).
- 순수 계산(분↔px)은 `calendarMath`에 함수로 분리해 단위 테스트. **재현 테스트 우선**([testing.md](../../../.claude/rules/testing.md) 버그 수정 계약): 큰 뷰 높이에서 특정 시각 블록의 기대 top(px)을 검증.

**저위험·독립** — 스키마·store 무관, 순수 렌더 좌표 버그. 요청 2(미리보기)보다 먼저 고쳐도 됨(§9 순서에 반영).

## 4. 데이터·아키텍처 (스키마 무변경 확인)

- **신규 필드 없음.** 종일=`scheduleAt` 자정 관례 유지(신규 `allDay` 필드를 만들면 스키마 검토·인간 승인 트랙 → 이번엔 안 함, §5). `todo`/`durationMin`은 기존 선택 필드. `version=1` 유지 → **인간 승인 불필요**.
- **모든 일정 변경은 `mapStore` 액션 경유**(`setScheduleAt`/`setScheduled`/`captureScheduled`) → `reminderOn/reminderId` 불변조건과 고아 정리 자동 보존. 캘린더가 노드 필드 직접 수정 금지.
- **도메인 경계**([architecture.md](../../../.claude/rules/architecture.md)):
  - `SubtreeMiniView`는 `canvas/` 내부를 참조하지 않는 **공유 읽기전용 컴포넌트**로 둔다(예: `src/ui/SubtreeMiniView.tsx`) — 하위 계층 순수 함수 `layout/treeLayout`만 사용.
  - 노드 검색은 `search/`를 직접 import하지 않고 `collectAgendaCached`가 쓰는 **워크스페이스 스캔 캐시를 store/공유 경유**로 재사용.
  - 우측 분할·오늘의 생각 append는 `sessionStore`/`window.api`(IPC)만 사용.
- **로컬 벽시계 ISO 고정** — 날짜·시각 계산 전부 로컬 `Date`(UTC 금지, 리마인더 라운드트립 보존).
- **신규 IPC 없음** — `readFile`/`file:save`/`capture:targetPath` 기존 채널로 충분.

## 5. 범위에서 뺀 것

- **반복 일정** — `version` 스키마 변경, 인간 승인 트랙(Phase 4, 별도 ADR).
- **`allDay` 전용 필드** — 자정 관례로 충분. (관례의 구조적 한계: "실제 자정 시작"과 "종일" 구분 불가 — 기존 한계 승계.)
- **외부 캘린더(Google/CalDAV)·EventKit** — Non-goal(로컬 우선).
- **정교한 겹침 packing / 월간 성능 캐시** — 기존 v1 한계 승계(느리면 후속).
- **미리보기 안에서의 편집** — 미리보기는 읽기전용. 편집은 우측 분할로 실제 맵을 연 뒤.

## 6. 가드레일 정합성

- **"우리는 todo 앱이 아니다"** — 새 라벨/우선순위/프로젝트 없음. 기존 노드 필드만. 일정 지정 시 자동 `todo` 승격은 [0014] 규칙 그대로.
- **G3 리트머스** — 캘린더를 꺼도 맵·노트·할 일은 온전. 캘린더는 노드의 시간 렌즈.
- **색 억제** — 새 색 없음. 기존 시맨틱(일정=주황)·상태 재사용.
- **reminder 불변조건** — 모든 경로 `mapStore` 경유.
- **원칙 3(점진적 공개)** — 클릭=미리보기 먼저, 시간=옵트인, 새로 만들기=index-0 행.
- **원칙 5(패턴 반복)** — 미리보기·생성 로직은 뷰마다 컨테이너만 다르고 동일. `.picker` 셸 재사용.

## 7. 완료 기준

- [ ] 일간 `집중`·`완료`가 **같은 스타일**(둘 다 아이콘+라벨 pill)로 렌더되고 짝으로 읽힌다.
- [ ] 스케줄 클릭이 맵을 바로 열지 않고 **그 노드+하위 읽기전용 미리보기**를 연다(일간=서랍, 월간=하단 패널, 주간=✕ 닫기 있는 팝업).
- [ ] 미리보기 안에서 스케줄 노드를 **다시 누르면 우측 분할**에 맵이 열리고 노드가 포커스된다. 닫힌 맵도 미리보기 가능.
- [ ] 주/일/월에서 빈 슬롯·진입점으로 `SchedulePicker`가 열리고, **기존 노드 검색·선택 → 그 노드에 일정**이 잡힌다.
- [ ] `새로 만들기`는 **오늘의 생각.mind**에 새 노드로 들어간다(활성 맵 아님). 종일 섹션에서도 생성된다.
- [ ] 일/월 생성에서 **종일 기본 + 시간 옵트인** 토글이 동작하고, 새로 만들기 텍스트의 자연어 시각이 파싱된다.
- [ ] 월간 셀이 `+N` 없이 **스크롤**로 전체 일정을 보여준다(셀 높이 120px 고정, 칩 스크롤). 셀 이동 클릭 계약 유지.
- [ ] 주간 그리드에서 **블록·캡처 입력이 시간 눈금과 정확히 정렬**된다(큰 뷰 높이·늦은 시각에서도). 분↔px 재현 단위 테스트 포함(§3.8).
- [ ] `MindMapDoc.version` 변경 없음(diff 확인). 모든 일정 변경 `mapStore` 경유(리마인더 무회귀).
- [ ] 순수 로직(미리보기 레이아웃 입력, 시간 파싱, 스크롤/오버플로 계산 등)은 순수 함수로 분리해 단위 테스트.
- [ ] `e2e/calendar-*.spec.ts`에 (a) 클릭→미리보기→2차클릭 우측분할, (b) 기존 노드 검색→일정, (c) 새로 만들기→오늘의 생각, (d) 일/월 종일·시간 토글, (e) 월간 스크롤 시나리오 추가.
- [ ] `/design-ui` 확인 → `make harness-check` → `make dev-safe` 실동작 검증(서랍/팝업/피커/스크롤).

## 8. 결정 완료 (2026-07-16 사용자 확정)

1. **(요청 1) 버튼 통일** — ✅ **둘 다 아이콘+라벨 pill**(§3.1).
2. **(요청 3·미리보기) 닫힌 맵** — ✅ **닫힌 맵 노드도 대상**. 미리보기는 디스크 읽기, 일정 지정은 **백그라운드 로드 후 적용**(§3.2·§3.3).
3. **(요청 6) 시간 입력** — ✅ **종일/시간 세그먼트 + HH:mm 시간 필드**(프리셋 칩 아님), 새로 만들기엔 자연어 파싱 병행(§3.6).
4. **(요청 2) 미리보기 트리거** — ✅ **카드/칩 전체 클릭**(전용 화살표 없음). 근육 기억 없어 안전(§3.2).
5. **(요청 7) 월간 셀 높이** — ✅ **92px→120px 상향**(§3.7).

- **`PX_PER_MIN` 값·리사이즈 스냅** — 40px/시간 유지 vs 상향(가독성), 스냅 15분 유지(§3.8).
