# 세션 노트 자리표시자 — 네이티브 placeholder 전환

> 작성: 2026-06-12 · 상태: 명세 (합의 후 구현) · 출처: [REVIEW §6.3](../REVIEW-2026-06-focus-and-dashboard.md)
> 선행: 목표 추출 정합성은 [goal.ts/sessionNote.ts](../REVIEW-2026-06-focus-and-dashboard.md#62)로 이미 해결됨. 이 스펙은 *편집 경험*만 다룬다.

## 1. 문제

세션 노트 템플릿(`src/focus/sessionNote.ts`)은 각 섹션(🎯/✅/🔨/🅿️) 아래에 **이탤릭 텍스트 힌트**(`_무엇에 집중하나 — 한 문장으로_`)를 본문 콘텐츠로 깐다. 이 때문에:

- 힌트가 **진짜 콘텐츠처럼 보여**, 입력 위치(어디에 목표를 쓰나)가 불명확하다.
- E2E에서 관찰: 힌트 줄/헤딩 근처를 클릭해 타이핑하면 캐럿이 **헤딩·힌트에 합쳐지기** 쉽다.
- 힌트 텍스트가 디스크에 남아, 미입력 섹션도 "내용이 있는 것처럼" 직렬화된다.

이상적 동작: **입력하면 사라지는 회색 placeholder**(헤딩 아래 빈 줄에 가이드가 떠 있다가, 타이핑 시작 시 소멸).

## 2. 제약 (구현 전 반드시 인지 — 검증에서 확인)

1. **markdown 빈 문단 소실**: tiptap 네이티브 `@tiptap/extension-placeholder`는 *빈 문단 노드*에만 뜬다. 그런데 `tiptap-markdown`은 저장·재로드 시 **빈 문단을 버린다**. 즉 헤딩만 쌓인 템플릿은 첫 저장 후 섹션 사이에 빈 문단이 없어 placeholder가 사라지고, 헤딩에서 Enter를 쳐야 본문을 쓸 수 있다.
2. **한글 IME 위험**: "타이핑 시작 시 힌트 문단을 비운다"를 `handleTextInput`/keydown으로 구현하면 **조합 중(composition) 한글 입력**과 충돌해 첫 글자가 깨질 수 있다. 입력 경로를 가로채는 방식은 IME 안전성 검증 없이는 금지.

## 3. 해결 (제안 — 두 안, 합의 필요)

### A안 — 네이티브 placeholder + 빈 문단 보존 (정공법)
- `BODY_TEMPLATE`을 헤딩 + 빈 문단 구조로 바꾸고, **빈 문단을 보존하는 직렬화**를 추가한다(`tiptap-markdown` 빈 문단을 `\n\n<br>\n\n`류 마커나 커스텀 노드로 라운드트립, 또는 세션 노트 한정 직렬화 분기).
- `Placeholder.configure({ showOnlyCurrent:false, placeholder })` — `placeholder` 함수가 빈 문단의 직전 헤딩 이모지로 `SECTIONS[].hint`를 매핑(소스: `sessionNote.ts`).
- 범위: `NoteEditor`에 `scaffold?: boolean` prop 추가, `NotePane`에서 `scaffold={isSession}` 전달(섹션 힌트는 세션 노트에서만).
- 난이도: 中~高 (빈 문단 라운드트립이 핵심 리스크).

### B안 — 힌트 문단 데코레이션 (절충, 저위험)
- 힌트는 지금처럼 디스크에 텍스트로 남긴다(markdown 안전). 단 세션 노트에서 **텍스트가 `PLACEHOLDER_HINTS`와 정확히 일치하는 문단**을 ProseMirror 데코레이션으로 **회색·placeholder 스타일** 처리 → 콘텐츠가 아니라 가이드로 명확히 읽힌다.
- 비우기는 입력 가로채기(IME 위험) 대신 **선택 진입 시 문단 전체 선택**(클릭하면 힌트가 통째로 선택돼, 바로 타이핑하면 덮어써짐) 정도의 안전한 보조만.
- 난이도: 低~中. markdown·IME 리스크 없음. "사라지는" 느낌은 A안보다 약함.

**권장**: B안으로 빠른 체감 개선을 먼저 내고, A안(빈 문단 보존)은 별도로 검증. 둘 다 `sessionNote.ts` 단일 소스를 재사용한다.

## 4. 범위에서 뺀 것

- 일반(비세션) 노트의 placeholder 동작 변경 — 현행 단일 PLACEHOLDER 유지.
- 슬래시 메뉴·툴바 등 다른 에디터 기능.
- 목표 추출 로직 — 이미 정합화 완료(이 스펙 범위 밖).

## 5. 완료 기준

- 세션 노트의 각 섹션 가이드가 **콘텐츠가 아닌 placeholder로 명확히** 구분된다(시각).
- 사용자가 목표를 쓰면 가이드와 섞이지 않고 깔끔히 입력된다(덮어쓰기/소멸).
- **한글 IME로 첫 글자부터 정상 입력**된다(회귀 없음). 격리 앱 E2E로 확인.
- 미입력 섹션이 저장·재로드돼도 깨지지 않는다(A안은 빈 문단 라운드트립 검증, B안은 텍스트 유지).
- 일반 노트 편집 경험은 변화 없음.
