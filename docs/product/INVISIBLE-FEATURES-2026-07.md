## 1. 왜 지금 "안보이는 기능"인가

### 1-1. 정의

**안보이는 기능 = 사용자가 "쓴다"고 의식하지 않지만, 없으면 즉시 불편·불신·손실로 드러나는 기반 기능.**

보이는 기능(노드 만들기, 노트 쓰기, 일정 걸기)은 사용자가 **의도적으로** 부른다. 안보이는 기능은
사용자가 부르지 않아도 **알아서** 작동한다 — 자동 저장, 크래시 복구, 자료 자동 정돈, 자연어 인식,
조용한 성능 최적화, 파일 위생. 잘 되면 아무도 눈치채지 못하고, 무너지면 앱을 떠난다.

### 1-2. 만족도와의 관계 (Kano 모델)

안보이는 기능은 만족도 곡선에서 두 지점에 집중된다.

| 유형 | 특성 | 이 제품의 안보이는 기능 예 | 만족도 레버 |
|---|---|---|---|
| **당연 품질 (Must-be)** | 있으면 무감, **없으면 강한 불만·이탈** | 자동 저장, 데이터 안전, 세션 복원, 파일 무결성 | 이탈 **방어** (바닥을 지킴) |
| **매력 품질 (Attractive)** | 없어도 불만 없지만, **있으면 "알아서 해주네" 감동** | 자연어 일정 인식, 자동 정돈, 성장 넛지, 자동 명명 | 애착 **공격** (기억에 남김) |

핵심 통찰: **보이는 기능을 아무리 잘 만들어도, 당연 품질이 한 번 무너지면(맵이 날아가면) 전부 무의미하다.**
그래서 안보이는 기능 투자는 **만족도의 하한선을 끌어올리는 일**이고, 이 제품에서는 특히 결정적이다 — 이유는 다음 절.

### 1-3. 우리 제품에서 특히 중요한 이유

경쟁사 분석([RESEARCH](reports/RESEARCH-mindmap-competitors.md))에서 사용자가 반복적으로 떠나는 **1순위 지점은
"데이터 손실·락인 공포"**였다 (MindMeister: "취소하면 맵이 되돌릴 수 없이 사라진다", "3시간 작업 손실",
헬프센터에 "I Have Lost All Maps!" 문서 존재). 우리의 대응은 **"로컬 파일이 원본"**이다
([PRODUCT-DEFINITION §왜 우리인가](PRODUCT-DEFINITION.md)).

그런데 **"로컬 파일이 원본"이라는 약속은 안보이는 기능이 지킬 때만 진짜다.** 파일이 저장 중 깨지거나,
두 창이 서로 덮어쓰거나, 클라우드 동기화 충돌로 조용히 사라지면, 우리는 경쟁사와 똑같은 배신을 저지른다 —
오히려 "네 파일이니 네 책임"이라 더 아프다. 즉 **안보이는 기능은 이 제품의 핵심 차별점(소유·신뢰)을
실제로 작동시키는 엔진**이다. 세 핵심 경험과의 연결:

- **속도**(생각의 속도로) → 마찰 제거·조용한 성능이 지탱한다.
- **통합**(맵·노트·일정 한 앱) → 리마인더 동기화 복원력·양방향 참조 무결성이 지탱한다.
- **소유**(내 파일은 내 것) → 데이터 안전·파일 위생·무결성이 지탱한다. ← **여기가 가장 약하고, 가장 중요하다.**

## 2. 안보이는 기능의 5계층 + 현황 카탈로그 (이미 잘 하고 있는 것)

우리 앱은 이미 상당한 안보이는 기능을 갖추고 있다. 아래는 **코드에서 실제 확인된** 현황이다 —
기획의 출발점은 "없는 것을 채우기"가 아니라 "있는 것 위에 쌓기"다.

### A. 데이터 신뢰 계층 (소유를 지킨다)
- 파일별 독립 디바운스 **자동 저장**: 맵 1s / 노트 800ms + `⌘S` 수동 저장. ([Pane.tsx:104](../../src/panes/Pane.tsx#L104), [NotePane.tsx:149](../../src/note/NotePane.tsx#L149))
- 타이머 발화 시점에 `filePath`를 **다시 읽어** rename 직후 stale 경로 쓰기 방지.
- 탭 닫기(개별·다른탭·모두) 시 **dirty 탭 즉시 flush** — 마지막 ~1초 편집 유실 방지. ([sessionStore.ts:103](../../src/store/sessionStore.ts#L103))
- rename/move 전 **`flushSaves` 선행** — 디스크 조작이 자동저장과 레이스 방지.
- **삭제 중 경로 자동저장 억제**(`beginDelete/endDelete`) — 삭제된 파일을 자동저장이 되살리는 회귀 가드.
- 파일별 **경계 검증**: 모든 쓰기 IPC를 `assertInsideWorkspace`가 감싸 path-traversal 차단. ([main.ts:481](../../electron/main.ts#L481))

### B. 복원력 계층 (실수·크래시에서 되돌린다)
- **세션 복원**: 재시작 시 열린 탭·분할 상태 복원. 손상된 파일 1개가 전체 복원을 막지 못하게 **skip + 건너뛴 수 안내**. ([sessionStore.ts:597](../../src/store/sessionStore.ts#L597))
- **Undo/Redo** 100단계, 연속 편집 병합. ([mapStore.ts:175](../../src/store/mapStore.ts#L175))
- **휴지통**: 파일·폴더 삭제를 `.trash/`로 이동 + 매니페스트 복원(동반 이미지 폴더도 함께). ([main.ts:726](../../electron/main.ts#L726))
- **리마인더 고아 정리(reaping)**: 중복 리마인더는 최신만 남기고 삭제, `nodeId` 태그로 소유권 추적 → undo 후에도 복구. ([reminderSync.ts:215](../../src/sync/reminderSync.ts#L215))
- **지수 백오프 heartbeat**(5s→15s→45s→120s): AppleScript 브리지 장애를 자동 복구.

### C. 자동 정돈 계층 (자료를 예쁘게 알아서 정리한다)
- **"제목 없음" 자동 명명**: 첫 중심 주제 입력 시 파일명 자동 생성(제어문자 제거·60자 컷·1회). ([autoName.ts](../../src/io/autoName.ts))
- **레거시 마이그레이션**: raw-hex 색 → 시맨틱 키, 누락 필드 방어적 backfill. ([formats.ts:32](../../src/io/formats.ts#L32))
- **에셋 폴더 자동 추적**: 노트 rename/move/trash 시 이미지 폴더가 따라가고 본문 이미지 경로 자동 재작성 + 구버전 폴더 숨김 이름 승격. ([main.ts:418](../../electron/main.ts#L418))
- **rename 충돌 de-dupe**: 기존 파일 클로버 방지 + " 2" 자동 접미사. ([main.ts:601](../../electron/main.ts#L601))
- 레이아웃 자동 정렬(트리) + FLIP 애니메이션(점프 대신 활강). ([NodeView.tsx:118](../../src/canvas/NodeView.tsx#L118))

### D. 의도 감지·적응 계층 (알아서 알아차린다)
- **자연어 일정 인식**: `@내일 3시`·`@다음주 월요일` → 일정 자동 설정(보수적 `@` 마커 방식). ([parseNodeText.ts:29](../../src/store/parseNodeText.ts#L29))
- **해시태그 색상**: `#red`·`#빨강` → 태그 색 자동 지정.
- **URL 본문 추출**: 링크 → Readability 파싱 → Markdown(원본 HTML 미주입, 15s 타임아웃).
- **성장 넛지**: 탭 5개→분할 제안, 유사 노트 3개→템플릿 제안, 일정 노드 3개→리마인더 제안(각 1회). ([GrowthNudges.tsx](../../src/ui/GrowthNudges.tsx))
- **Quick Keys 자동 학습**: 자주 쓰는 명령에 `⌥1-9` 자동 배정. ([commandUsage.ts](../../src/ui/commandUsage.ts))
- **Smart Views**: 최근 수정(mtime)·즐겨찾기 자동 뷰.

### E. 조용한 성능·보안 계층 (빠르고 안전하게, 티 안 나게)
- **시맨틱 줌 LOD**: 0.42배 이하에서 칩·뱃지 숨겨 대형 맵 성능 확보. ([Canvas.tsx:510](../../src/canvas/Canvas.tsx#L510))
- **조건부 폴링**: 리마인더 인바운드 폴은 **창이 보일 때 + 추적 노드가 있을 때만** → idle 비용 ~0.
- **이미지 자동 축소**: 붙여넣기 이미지 최장변 2400px 캡·JPEG 재인코딩. ([imageInsert.ts:7](../../src/note/imageInsert.ts#L7))
- **osascript 직렬화 + 배치 조회**: 동시 호출 wedge 방지, 23s→<1s 속도 개선. ([reminders.ts:55](../../electron/reminders.ts#L55))
- **safeStorage 키 암호화**(OS 키체인) + **argv 인젝션 방지**(execFile, shell 아님).

> 요약: **당연 품질의 상당 부분은 이미 견고하다.** 특히 자동저장·삭제 레이스·리마인더 복원력은 회귀
> 테스트로 보호되는 3종 세트다. 아래 제안은 이 위에서 **아직 비어 있는 신뢰의 틈**과 **매력 품질의 여백**을 채운다.

## 3. 갭 & 제안 (더 하면 좋은 것)

각 제안은 `IF-NN` ID, Kano 유형, 연결된 핵심 경험, 노력(S/M/L), 리스크로 태그한다.
근거가 되는 갭은 코드 스캔 또는 [FEATURE-INVENTORY](FEATURE-INVENTORY.md)의 "알려진 한계"에서 확인된 것이다.

### 3-A. 데이터 신뢰 — 소유 약속을 진짜로 (최우선)

**IF-01 · 원자적 쓰기 (Atomic write)** — 당연 품질 · 소유 · S · 리스크 낮음
현재 저장은 `fs.writeFile(대상파일, 내용)`으로 **원본에 직접** 쓴다([main.ts:290](../../electron/main.ts#L290)·[:577](../../electron/main.ts#L577)).
저장 도중 크래시·전원 차단·디스크 풀이 나면 `.mind`/`.md` 파일이 **잘려서 손상**될 수 있다 — 원본이 진실
원천인 앱에서 가장 치명적인 실패다. **해법**: `대상.tmp`에 쓰고 `fsync` 후 `rename`(원자적 교체). 무의존·저위험.
→ **"로컬 파일이 원본" 약속을 실제로 안전하게 만드는 1번 안보이는 기능.**

**IF-02 · 로컬 버전 히스토리 / 자동 스냅샷 ("타임머신")** — 매력+당연 하이브리드 · 소유 · L · 리스크 낮음
Undo는 100단계지만 **재시작하면 사라진다**. 휴지통은 파일 단위 삭제 복구만 된다. **"어제 실수로 지운
가지"를 되살릴 방법이 없다.** MindMeister 이탈의 핵심 공포를 정면으로 뒤집는 기능: 저장 시점마다(또는
N분·중요 변경마다) 이전 버전을 워크스페이스 숨김 폴더 `.history/`에 스냅샷하고, **"이 맵의 이전 버전"**
패널에서 특정 시점으로 복원. `.trash`와 동일한 무의존·dot-prefix 패턴이라 **스키마 변경 없음**(파일 스냅샷일 뿐).
→ 경쟁사가 "돈 내고 클라우드로 파는" 안전망을 **로컬·무료·오프라인**으로 제공하는 헤드라인 차별점.

**IF-03 · 단일 인스턴스 락 (Single-instance lock)** — 당연 품질 · 소유 · S · 리스크 낮음
`requestSingleInstanceLock`이 없어(코드 스캔 확인) 같은 워크스페이스를 **두 인스턴스가 열면 자동저장이
서로 덮어쓸 수 있다.** Electron 표준 API로 두 번째 실행을 기존 창 포커스로 전환. 무의존.

**IF-04 · 클라우드 폴더 충돌 감지 (External-change detection)** — 당연 품질 · 소유 · M · 리스크 중간
소유 스토리의 **가장 약한 고리**: 폴더 동기화(iCloud/Dropbox)는 사용자 몫인데, 두 기기에서 편집하면
동기화 도구가 조용히 `파일 2` 충돌본을 만들거나 최신본을 덮는다. **해법**: 파일을 열 때·자동저장 직전에
디스크의 `mtime`/해시를 메모리 사본과 대조 → 외부 변경이 감지되면 저장 전에 **"이 파일이 밖에서
바뀌었어요 — 내 버전 유지 / 디스크 버전 불러오기"** 안내. 조용한 데이터 손실을 막는다.

### 3-B. 자동 정돈 & 복원력 — 자료가 알아서 깔끔하게

**IF-05 · 죽은 링크 자동 정리 (Dead-link GC)** — 당연 품질 · 통합 · S · 리스크 낮음
노트를 삭제하면 노드에 **죽은 노트 링크가 잔존**한다([INVENTORY §7 한계](FEATURE-INVENTORY.md)). 노트 삭제·이동 시 이를
참조하는 노드 링크를 자동 정리하거나, 열 때 조용히 GC → 양방향 참조 무결성.

**IF-06 · 휴지통 보관 정책 (Trash retention)** — 일원적 품질 · 소유 · S · 리스크 낮음
휴지통 **자동 만료가 없어**(스캔 확인) 무한 축적된다([INVENTORY §13 한계](FEATURE-INVENTORY.md)). 설정 가능한 보관
기간(예: 30일 후 자동 비우기, 기본 꺼짐/안내) → 워크스페이스 위생. 파괴적이므로 **명시적 옵트인**.

**IF-07 · 집중 세션 유휴 감지 (Focus idle detection)** — 일원적 품질 · 통찰 · M · 리스크 낮음
집중 세션에 **유휴 감지가 없어**([INVENTORY §11c 한계](FEATURE-INVENTORY.md)) 자리를 비워도 시간이 계속 쌓인다 →
통찰 대시보드 데이터의 신뢰를 갉는다. 입력·창 활성 기준 유휴 임계(예: 10분) 감지 → "자리 비우셨나요?"
확인 후 유휴 구간 차감. streak 게이밍 방지 로직과 결이 같다.

**IF-08 · 자동 레이아웃 미세정돈 ("예쁘게")** — 매력 품질 · 정리 · M · 리스크 중간
이미 트리 자동 정렬이 있지만, 사용자가 손댄 뒤 어긋난 간격·겹침을 **한 번에 예쁘게 재정돈**하는
"정돈" 액션(간격 균일화·서브트리 밸런스·선택적 스냅)을 강화. "자료를 예쁘게 자동 정리"라는 기대에 직결.

### 3-C. 지능·성능·보안 — 조용히 더 똑똑하고 빠르게

**IF-09 · 시스템 테마 자동 추종** — 매력 품질 · 속도(네이티브 느낌) · S · 리스크 낮음
**시스템 다크/라이트 자동 추종이 없다**(수동 전환만, [INVENTORY §11](FEATURE-INVENTORY.md)). `nativeTheme` 구독으로
OS 설정 따라가기(옵션) → macOS 네이티브 감각 강화.

**IF-10 · 검색·캘린더 증분 인덱스 캐시** — 일원적 품질 · 속도 · M · 리스크 낮음
전역 검색(`⌘⇧F`)과 캘린더 월간 뷰가 **열 때마다 워크스페이스 전체를 재스캔**한다(캐시 없음, [INVENTORY §9](FEATURE-INVENTORY.md)·[§11d](FEATURE-INVENTORY.md)).
`mtime` 기반 증분 인덱스 → 큰 워크스페이스에서 즉시 응답. "조용한 성능"의 정석.

**IF-11 · 내보내기 메타데이터 보존** — 일원적 품질 · 소유 · M · 리스크 낮음
Markdown/OPML 내보내기에 **색·일정·노트 연결이 빠진다**(텍스트 구조만, [INVENTORY §10](FEATURE-INVENTORY.md)).
소유의 완성은 "내 데이터를 온전히 꺼낼 수 있음"이다 → 메타데이터 포함 내보내기 옵션(왕복 충실도).

**IF-12 · web:fetch SSRF 방어 + webPreferences 명시 하드닝** — 당연 품질(보안) · 소유 · S · 리스크 낮음
URL 추출(`web:fetch`)에 **사설 IP/메타데이터 엔드포인트 차단이 없다**(127.0.0.1·10.x·169.254 등, 스캔 확인).
또한 `webPreferences`가 `contextIsolation`/`nodeIntegration`/`sandbox`를 **명시하지 않고 기본값에 의존**한다
([main.ts:155](../../electron/main.ts#L155)) — 현재 안전하나 버전 회귀에 취약. 명시적 하드닝 + SSRF 가드. → [security.md](../../.claude/rules/security.md) 강화.

**IF-13 · 타임존 견고성** — 당연 품질 · 실행 · M · 리스크 중간(스키마 주의)
일정이 로컬 시간으로 저장돼 **시간대 변경 시 어긋날 수 있다**([INVENTORY §6 한계](FEATURE-INVENTORY.md)). 저장 표현·표시
정책 정비. `scheduleAt` 표현이 바뀌면 `MindMapDoc.version` 영향 가능 → **결정 기록·인간 승인 트랙**.

**IF-14 · 넛지·Quick Key 임계값 적응화** — 매력 품질 · 속도 · S · 리스크 낮음
넛지 임계(5탭/3노트/3일정)·Quick Key 임계(2회)가 **하드코딩 상수**다(스캔 확인). 사용 패턴 기반 완만한
적응 또는 설정 노출 → 과잉/과소 넛지 방지.

### 3-D. 앞으로 열리는 안보이는 AI 기능 (인프라 이미 존재)

AI 프로바이더 키 저장 인프라는 이미 있으나 소비 기능이 없다([INVENTORY §14](FEATURE-INVENTORY.md)). AI는 **가장
"안보이는" 형태**로 쓸 때 가치가 크다 — 사용자가 프롬프트를 치는 게 아니라 **알아서** 해주는 것:

- **자동 제목·요약**: 긴 노트에 제목/한 줄 요약 자동 제안(자동 명명 IF의 AI 확장).
- **자동 태그·색 제안**: 노드 내용에 어울리는 태그 색 추천(해시태그 색의 지능형 확장).
- **정돈 제안**: "이 세 노드는 한 섹션으로 묶을까요?" 같은 구조 정돈 넛지.

모두 **매력 품질**이며, 어느 핵심 경험(속도/통합/소유)에 묶을지는 [PRODUCT-DEFINITION §AI 방향 전환](PRODUCT-DEFINITION.md) 확정 후
별도 명세(`docs/product/specs/`)로 분리한다. 본 기획서에서는 **방향만** 기록한다(P2 이후).

## 5. 성공 지표 (안보이는 기능을 어떻게 측정하나)

안보이는 기능은 "잘 되면 티가 안 나서" 측정이 어렵다. **없었을 때의 나쁜 일이 안 일어나는지**를 대리 지표로 본다.

- **데이터 무결성**: 파일 로드 시 파싱 실패율(0에 수렴), 손상으로 세션 skip된 파일 수(→ 0). ([sessionStore.ts:604](../../src/store/sessionStore.ts#L604) 로그 활용)
- **복구 사용률**: 버전 히스토리 복원·휴지통 복원·undo 사용 횟수(안전망이 실제로 쓰이는지 = 신뢰의 증거).
- **조용함**: 넛지 노출 후 수락률 vs 무시율(넛지가 방해가 아니라 도움인지), idle 차감 발생 빈도.
- **성능**: 전역 검색·캘린더 월간 뷰 오픈 지연(캐시 전후 비교), 대형 맵 프레임 유지.
- **정성(dogfooding)**: 제작자가 "데이터가 날아갈까 걱정 없이" 매일 쓰는가([PRODUCT-DEFINITION 성공 기준](PRODUCT-DEFINITION.md)).

> 모든 계측은 **로컬 로그**([logger.ts](../../electron/logger.ts))에만 남기고 사용자 파일 내용은 절대 기록하지 않는다([security.md](../../.claude/rules/security.md)). 서버 전송 없음.

## 6. 리스크 · 인간 승인 필요 항목

- **레이스 상호작용**: 자동저장·flush·삭제억제·rename은 얽힌 3종 세트다. IF-01/03/04는 이 경로를 건드리므로
  `deletingPaths.test.ts`·`sessionStore.test.ts` 회귀 확인 필수(구현 전 재현 테스트 우선 — [testing.md](../../.claude/rules/testing.md)).
- **스키마 영향(인간 승인)**: IF-13(타임존)은 `scheduleAt` 표현 변경 시 `MindMapDoc.version` 마이그레이션 →
  [AGENTS.md 인간 승인](../../AGENTS.md) 트랙. 버전 히스토리(IF-02)는 파일 스냅샷이라 스키마 무영향(승인 불필요).
- **리마인더 불변조건**: IF-05(죽은 링크 정리)가 노드 링크를 만질 때 `reminderOn/reminderId` 로직은 건드리지
  않는다 — 분리 시 리마인더 고아 위험([ARCHITECTURE 불변조건](../../ARCHITECTURE.md)).
- **파괴적 기본값 금지**: IF-06(휴지통 자동 만료)은 반드시 **옵트인**. 기본은 안내만.
- **무의존 원칙**: P0(IF-01/02/03)은 신규 의존성 없이 구현 가능 확인. 새 의존성 필요 시 [인간 승인](../../AGENTS.md).
- **E2E 의무**: IF-01/02/03은 데이터 안전 시나리오라 E2E 필수([testing.md 배포 게이트](../../.claude/rules/testing.md)) —
  "저장 중 강제 종료 후 재시작 시 파일 온전", "버전 복원", "두 번째 실행 시 기존 창 포커스".
