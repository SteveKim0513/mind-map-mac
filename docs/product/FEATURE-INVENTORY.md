## 1. 맵 캔버스 — 노드 편집 ⚠️

**무엇**: 키보드 중심으로 노드를 생성·편집·삭제한다. 한 캔버스에 루트(중심 토픽) 여러 개 허용.

**진입점**
- 자식 추가: `Tab` / 노드 "+" 버튼 / 우클릭 메뉴 / 선택 툴바 "+"
- 형제 추가: `Enter`(선택 있음) / 우클릭 메뉴
- 루트 추가: `Enter`(선택 없음) / 빈 캔버스 더블클릭(해당 위치에 생성)
- 편집: `Space` · `F2` / 더블클릭 / 우클릭 메뉴
- 삭제: `Delete` · `Backspace` / 우클릭 메뉴 / 선택 툴바 휴지통

**핵심 규칙**
- 새 노드는 즉시 편집 모드로 진입. 빈 텍스트로 편집을 끝내면 자동 삭제(자식 없을 때).
- 자식 추가 시 접힌 부모는 자동으로 펼쳐진다. 부모가 `scheduled`면 새 자식도 상속.
- 삭제는 확인 없이 즉시 실행되고 자손까지 재귀 삭제 (undo로 복구). 관련 연결선·섹션도 정리. 리마인더가 걸린 노드는 macOS 리마인더도 삭제.
- 다중 선택 삭제 시 부모-자식이 함께 선택돼 있으면 최상위만 삭제(이중 삭제 방지).
- 단일 행 텍스트만 지원 (Shift+Enter 줄바꿈 없음).
- **자연어 일정·태그 자동 인식**(REDESIGN-VISION §3-2, 2026-07-10 · "@" 마커 필수화 2026-07-14):
  텍스트 커밋 시 "@내일 3시"·"@다음 주 월요일"처럼 상대날짜/요일 앞에 **"@"를 붙인 경우에만**
  감지해 `scheduled`+`scheduleAt`을 자동 설정, `#색상명`(영문 키 또는 한글 별칭) 토큰을 감지해
  `color`를 자동 지정한다. `src/store/parseNodeText.ts`. 처음엔 "@" 없이 "오늘"/"내일" 등이 텍스트
  어디에 있든 매칭했으나, "오늘 : 회의 메모"처럼 날짜 단어를 라벨/헤더로 쓰는 습관과 자주 충돌해
  실사용 리포트 후 `#색상`과 대칭되는 명시적 마커 방식으로 변경했다. "@" 없이 날짜/요일 표현만
  있으면(예: "내일 회의") 절대 매칭하지 않음 — 과탐지보다 무반응이 안전하다는 원칙. 이미 일정·색이
  있는 노드는 재편집해도 덮어쓰지 않아 SchedulePopover·색 그리드가 "자동 인식을 고치는 보조
  경로"로 남는다. 인식된 텍스트는 그대로 유지되고(잘라내지 않음) 기존 일정 칩(`.gchip.sched`)·색
  틴트가 인식 신호를 대신한다 — 별도의 인라인 하이라이트 렌더링은 공수 대비 낮은 가치로 판단해
  범위 밖(계획 참고).

**알려진 한계**: 멀티라인 텍스트 미지원. 삭제 확인 팝업 없음(의도된 설계 — undo 신뢰). 제목은 저장 시 trim됨(공백만 입력 시 빈 제목 — [UX 감사 B2](reports/UX-AUDIT-2026-06.md)에서 수정). 자연어 일정 파싱은 상대날짜·요일만 지원(절대 날짜·반복 일정 미지원 — [계획](../exec-plans/completed/2026-07-10-nl-schedule-parsing.md)).

## 2. 맵 캔버스 — 구조 조작 ⚠️

**무엇**: 드래그 재배치(부모 변경), 형제 순서 변경, 접기/펼치기, 다중 선택.

**진입점**
- 재배치: 노드 드래그(5px 이상) → 가까운 노드(130px 내)가 드롭 타깃으로 강조
- 형제 순서: `Alt+↑` / `Alt+↓`
- 접기/펼치기: 노드 옆 −/+ 버튼 / `⌘←`(접기) · `⌘→`(펼치기) / 우클릭 메뉴
- 다중 선택: `Shift+클릭`
- 키보드 이동: 방향키 (←부모 / →첫 자식 / ↑↓형제)

**핵심 규칙**
- 자기 자손 아래로는 이동 불가(사이클 방지). 다중 선택 상태에서 드래그하면 선택 전체가 함께 이동.
- 루트 노드 드래그는 재배치가 아니라 **자유 위치 이동**(manualPos) — 이후 자동 정렬의 앵커가 된다.
- 접힌 노드는 숨겨진 자손 개수 뱃지를 표시.
- 레이아웃 변경 시 0.28s 스프링(FLIP) 애니메이션.

**알려진 한계**: 러버밴드(영역 드래그) 다중 선택 없음. 선택 툴바는 단일 선택일 때만 표시.

## 3. 노드 속성 ⚠️

**무엇**: 색(태그), 아이콘, 완료 상태, 인라인 메모(2줄), 다중 URL 링크.

**진입점**
- 색: 선택 툴바 색 도트 / 우클릭 메뉴 색 그리드 — 같은 색 재클릭 시 해제
- 완료: `⌘Enter` / 선택 툴바 체크 / 우클릭 메뉴 — 취소선 + 페이드 표시
- 아이콘: 노드 팝오버(6종 라인 아이콘: star·check·flag·bulb·pin·target)
- 메모·링크: 노드 팝오버(400ms 디바운스 자동 저장) / 우클릭 메뉴

**핵심 규칙**
- 색은 hex가 아닌 **시맨틱 키**('red'~'brown' 8종)로 저장 → 팔레트 조정 시 파일 수정 불필요. 레거시 hex는 로드 시 자동 마이그레이션.
- 완료 토글은 `updatedAt`을 갱신해 리마인더 동기화 대상이 된다. 메모·링크 변경은 동기화 대상 아님.
- 링크는 `links[]` 배열(다중) + 레거시 `link` 필드 공존. 노드 옆에 도메인 칩으로 표시.

**알려진 한계**: 커스텀 색·아이콘 추가 불가(프리셋 고정). 아이콘 진입점이 노드 팝오버 안에만 있어 발견성 낮음(UX 감사 대상).

## 4. 연결선 · 섹션 ⚠️

**무엇**: 트리 구조 밖의 자유 연결(크로스링크)과, 여러 노드를 묶는 색칠 영역(섹션).

**진입점**: 노드 2개 선택 → 우클릭 "두 노드 연결" / 2개 이상 선택 → 우클릭 "섹션으로 묶기 (N)"

**핵심 규칙**
- 연결선: 중복(양방향 동일 쌍) 무시. 연결 위에 메모 라벨을 달고 드래그로 위치 조정 가능. 접힌 노드의 연결은 보이는 조상으로 라우팅.
- 섹션: 멤버를 둘러싼 blob 경계 + 제목·색 편집. 멤버가 삭제되면 자동 정리, 멤버 0이면 섹션 삭제. 레이아웃에서 멤버 주변 20px 여유 확보.
- 다중 선택 컨텍스트 메뉴는 "구조"(연결·섹션)/"속성"(색·완료) 그룹 레이블로 구분된다(단일 노드 메뉴는 이미 편집/속성/첨부/일정·집중/보기 5그룹으로 구분돼 있었음 — IA-STRATEGY §5-5, 2026-07-10).

**알려진 한계**: 연결/섹션 모두 진입점이 다중 선택 + 우클릭뿐 — 발견성 낮음(UX 감사 대상).

## 5. 보기 (줌·팬·레이아웃·필터) ⚠️

**무엇**: 자동 트리 레이아웃 위에서 줌/팬, 서브트리 집중(focus), 색상 필터.

**진입점**
- 줌: `⌘+휠`(핀치) / 하단 툴바 −·%·+ / 메뉴·`⌘=` `⌘-` `⌘0`(화면 맞춤) / `Z`(선택 서브트리로 확대)
- 팬: 빈 캔버스 드래그 / 트랙패드 두 손가락
- 집중: 우클릭 "집중" → 브레드크럼("전체 / 상위로")으로 탈출, `Esc`로도 해제
- 색상 필터: 하단 툴바의 색 도트(맵에 사용된 색만 표시) 클릭 토글 — 활성 시 "상위/하위 포함" 토글 노출, `Esc` 해제
- 재배치: 하단 툴바 새로고침 버튼(강제 리레이아웃)

**핵심 규칙**
- 레이아웃: 왼→오 트리, 형제 서브트리는 겹침 없는 수직 밴드 배치. 루트는 manualPos 앵커 또는 자동 스택.
- 줌 범위 0.2~3.0. **0.42 이하에서 LOD 모드**(칩·뱃지 숨김)로 대형 맵 성능 확보.
- 줌/팬 상태는 파일에 저장되지만 undo 대상은 아님.
- 색상 필터 시 보이지 않는 부모의 자손은 새 루트처럼 배치(forest layout).

**알려진 한계**: 레이아웃 방향 고정(우측 단방향 — 방사형/양방향 없음). 색상 필터는 색이 하나도 안 쓰인 맵에서는 도트 자체가 숨겨져 기능 존재를 알기 어려움(UX 감사 대상).

## 6. 일정 + macOS 리마인더 동기화 ⚠️

**무엇**: 노드에 날짜·시간을 걸면 macOS 리마인더 앱의 "MindMap" 전용 리스트와 **양방향 동기화**된다. 제품의 핵심 차별 기능.

**진입점**: 선택 툴바 달력 버튼 / 우클릭 메뉴 → SchedulePopover (날짜·시간 입력 + 빠른 칩: 오늘/내일/주말/다음주, 아침 09:00/점심 13:00/저녁 18:00/밤 21:00)

**핵심 규칙**
- 동기화 대상: 제목, 마감일시, 완료 상태. **필드 단위로 독립 해소** — 한쪽만 바뀐 필드는 그 값을 채택하고, 같은 필드가 양쪽에서 바뀐 경우만 수정 시각으로 tiebreak(동시 편집 손실 방지 — [명세](specs/2026-06-12-reminder-field-level-sync.md)).
- 아웃바운드: 편집 후 1.2초 디바운스. 인바운드: 45초 폴링(창이 보일 때만).
- AppleScript 호출은 전부 직렬화(동시 1개) — 리마인더 앱 행 방지. 텍스트는 argv로 전달(인젝션 방지).
- 소유권은 리마인더 본문의 `mindmap:<nodeId>` 태그로 식별 → undo 후에도 복구 가능. 같은 노드의 중복 리마인더는 최신만 남기고 자동 정리(reaping).
- 헬스 체크: 권한 거부/타임아웃을 구분해 UI에 표시("권한 필요 — 설정 › 자동화"), 백오프(5s→15s→45s→120s)로 자동 복구.
- 노드 삭제·일정 해제·복제 시 리마인더 정리 규칙 있음(복제 노드는 새 리마인더 발급).
- 일정을 켜면 자손 노드에도 `scheduled` 상속.

**알려진 한계**: 반복 일정 미지원(일회성만). 시간대 변경 시 기존 일정이 어긋날 수 있음(로컬 시간 저장). 리마인더 앱에서의 변경은 폴링 기반이라 앱이 꺼져 있으면 다음 실행 때 반영. macOS 외 플랫폼은 전부 no-op.

## 7. 노트 시스템 ⚠️

**무엇**: TipTap 기반 Markdown 노트. 독립 `.md` 파일로 존재하고, 마인드맵 노드와 명시적으로 연결할 수 있다.

**진입점**
- 새 노트: 사이드바 "새 노트" / 노드 우클릭 "노트 추가·연결"(→ `.notes/` 숨김 폴더에 생성, 오른쪽 분할에서 자동 열림)
- 노드→노트 연결: 노드 우클릭 "노트 추가·연결" (기존 노트 검색 or 신규 생성)
- 노트→노드 연결: 노트 패널 헤더 "연동" 버튼 (열린 맵의 노드 검색, 최대 80개)
- URL로 노트: 사이드바 "링크로 노트 만들기" → 웹페이지 본문 추출(Readability) → Markdown 변환

**핵심 규칙**
- 에디터: 제목 H1–H3, 리스트, 체크리스트, 인용, **코드 블록**(문법 강조 + 언어 드롭다운, `src/note/CodeBlock.tsx` — v0.7.12), 구분선, 링크, **이미지**, **표**(3×3 헤더 행, 셀 Tab 이동, GFM 파이프 마크다운으로 왕복 — [명세](specs/2026-06-12-note-tables.md)), **메타 블록**(구조화 속성, 아래 참조). `/` 슬래시 메뉴(한/영 검색, 캐럿에 위치하며 화면 하단에선 위로 뒤집힘, 스크롤하면 닫힘). 자동 저장 800ms 디바운스.
- **블록 드래그 재정렬**: 최상위 블록(단락·헤더·목록·인용·코드블록) 왼쪽에 hover 시 ⠿ 핸들 표시 → 드래그로 순서 변경, undo 가능 (`src/note/DragHandle.ts` — v0.7.21, Notion 스타일).
- **리스트 내어쓰기(Shift+Tab)의 "형제 노드가 함께 딸려가는" 현상 수정**(2026-07-14): ProseMirror `liftListItem` 기본 동작은 내어쓰는 항목 뒤에 같은 깊이의 형제 항목이 더 있으면 그 형제를 내어쓰는 항목의 자식으로 재배치해버린다("b1 > [b2,b3,b4]"에서 b3만 내어쓰면 b4가 b3의 자식이 됨) — 사용자에게는 "그 항목이 갑자기 스스로 부모 노드가 된 것"처럼 보여 반복적인 들여쓰기/내어쓰기가 예측 불가능해진다. `src/note/listOutdent.ts`가 뒤따르는 형제가 있을 때만 별도 트랜잭션으로 원래 부모 밑에 그대로 남기고, 내어쓰는 항목만 최상위로 옮긴다(커서 위치도 보존). 형제가 없는 일반적인 경우(마지막 항목 내어쓰기)는 라이브러리 기본 동작 그대로.
- 이미지: 툴바 버튼·붙여넣기·드래그&드롭으로 삽입, 2400px/0.92 품질로 자동 축소. **파일이 있는 노트는 외부 파일로 저장**(`.notename.assets/image-*.ext` — 이름 앞에 점을 붙인 숨김 폴더, Finder 등 외부 앱에도 노출되지 않는다. 클릭 시 lightbox로 확대 — `src/note/imageInsert.ts`, `src/note/ImageLightbox.tsx`, v0.7.15 이후). 저장된 적 없는 세션 노트는 base64 인라인으로 폴백. **폴더는 노트 이름 변경·이동·삭제·복원 시 항상 함께 따라가고 본문의 이미지 경로도 자동으로 다시 써진다**(`electron/main.ts`의 `findAssetsDir`/`relocateAssetsDir`). v0.8.3 이전 버전이 만든 점 없는 폴더(`notename.assets`)는 그대로 동작하며, 해당 노트를 다음에 이름 변경·이동·삭제하면 자동으로 숨김 이름으로 승격된다 — 별도 일괄 마이그레이션은 없음(2026-07-14).
- **메타 블록**: 노트 상단에 구조화 속성(텍스트·날짜·select·URL·숫자 5종 필드) 삽입. 전역 템플릿(설정 › 고급 설정 › 정보 양식)을 노트에서 선택해 최상단에 고정 삽입, 템플릿에 필드가 추가되면 해당 템플릿을 쓰는 모든 노트에 자동 반영. `_meta` frontmatter로 templateId+값만 저장(스키마는 항상 템플릿에서 조회) — `src/note/NoteMetaBlock.tsx`, `src/store/metaStore.ts` (v1, 2026-07). 삽입 버튼은 툴바의 "양식+"(REDESIGN-VISION §3-6, 2026-07-10 개명 — 기존 "메타+").
- 제목↔파일명: 제목을 편집하면 600ms 후 `.md` 파일명·탭·사이드바가 따라간다(단방향 — [명세](specs/2026-06-12-note-title-filename-sync.md)).
- 파일 포맷: 프론트매터(id, title, links — JSON 값) + 순수 Markdown → 다른 에디터로 열어도 읽힌다. 프론트매터 없는 일반 .md도 import 가능.
- 연결되면: 노트 쪽엔 노드 칩(클릭 → 해당 맵·노드로 점프), 노드 쪽엔 노트 칩(클릭 → 플로팅 프리뷰 팝업, "수정"으로 오른쪽 분할 열기, **"연동 해제"로 해제**). 한 노드에 여러 노트 연결 시 팝업에서 탭 전환. 해제는 노트 패널 칩의 X로도 가능(양방향 대칭).
- `.notes/` 폴더의 노트는 사이드바에 숨겨지고, "사이드바에 보이기"로 일반 노트로 승격 가능.

**알려진 한계**: 노트 삭제 시 노드에 죽은 링크 잔존. 노트 간 위키링크(`[[ ]]`) 미지원. URL 추출이 렌더러에서 동기 실행이라 무거운 페이지에서 UI 멈춤 가능. 노트 연결 픽커는 모달(열려 있는 동안 ⌘K/⌘P 차단, Esc로 닫힘 — [UX 감사 B1](reports/UX-AUDIT-2026-06.md)에서 수정). 메타 블록 값 연동 검색·리마인더·마크다운 내보내기 렌더링은 2차 범위(미구현).

## 8. 앱 셸 ⚠️

**무엇**: 워크스페이스(폴더) 기반 사이드바, 멀티탭 + 좌우 분할, 홈 화면.

**진입점**
- 워크스페이스: 기본 `~/Documents/MindMaps`, 사이드바에서 폴더 변경
- 분할: 탭을 드롭존으로 드래그 / `⌘K` → "화면 분할" / 보기 메뉴 → "화면 분할 토글"
- 사이드바 토글: 좌상단 버튼 / 메뉴 / `⌘K`
- 홈: 모든 탭을 닫으면 표시 (최근 12개 파일 + 새 마인드맵)

**핵심 규칙**
- 사이드바 구조(2026-07 개편): 상단 "오늘 · 돌아보기"를 전체 너비 내비게이션 행으로 분리, 라이브러리 섹션 헤더에 생성 버튼 4종(마인드맵·노트·URL·폴더) 우측 정렬, 하단 바는 "관리형" 화면 아이콘 묶음(휴지통 + Note Template, 새 관리 화면은 여기 편입)과 설정 버튼을 시각적으로 분리(과거의 "+" 팝업 생성 메뉴는 제거) — 생성(Create)과 이동(Navigate) 의도를 분리(IA-STRATEGY §5-2, 2026-07-10).
- **Smart Views**(REDESIGN-VISION §3-3, 2026-07-10): 상단 내비게이션 행에 "최근 수정"·"즐겨찾기" 2개 추가(Apple Notes Smart Folder 패턴 — 새 UI 없이 "오늘"·"집중 기록"과 시각적으로 동일한 평범한 행). "최근 수정"은 `workspace:tree` 응답에 추가된 파일별 `mtimeMs`를 상위 20개 정렬해 보여줌(새 IPC 채널 아님, 기존 채널 응답 확장). "즐겨찾기"는 파일 행에 마우스를 올리면 나타나는 별표로 토글, `<workspace>/.pins.json`에 절대경로 배열로 저장(`.trashmeta.json`과 동일 패턴), 삭제된 파일의 죽은 핀은 다음 조회 시 조용히 정리됨. 두 뷰 모두 휴지통·Note Template 패널과 같은 `.trash-panel` 셸을 재사용(`src/ui/RecentView.tsx`, `src/ui/FavoritesView.tsx`) — [계획](../exec-plans/completed/2026-07-10-smart-views.md).
- 사이드바는 폴더·`.mind`·`.md`만 표시(숨김 파일 제외), 폴더→맵→노트 순 정렬. 인라인 이름 변경, 드래그로 폴더 이동, ⌘클릭 다중 선택 후 일괄 삭제(휴지통으로).
- 분할은 최대 2그룹(좌/우). 우측 그룹이 비면 자동으로 분할 해제. 같은 파일을 양쪽에 동시에 열 수 없음(기존 탭 활성화).
- 탭이 5개 이상 열리면 화면 분할을 1회 토스트로 제안한다(`src/ui/GrowthNudges.tsx`, IA-STRATEGY §5-6). 비슷한 제목의 노트가 3개 이상 쌓이거나(기본값 "제목 없음" 계열 제외) 열린 맵에 스케줄 노드가 3개 이상 생기면 각각 노트 템플릿·리마인더 동기화를 제안 — 전부 1회성, 닫으면 다시 안 뜸.
- 세션(열린 탭·분할 상태)과 최근 파일은 localStorage에 저장 → 재시작 시 복원.
- 파일 삭제·이름 변경 시 열린 탭과 자동 저장 충돌을 정리(flushSaves 선행).

**알려진 한계**: 분할 만들기의 발견성이 낮음(드래그 또는 팔레트만, 성장 넛지로 일부 완화). 3분할 이상 불가(의도된 제한).

## 9. 검색 · 커맨드 팔레트 · Quick Open ⚠️

**진입점**: 노드 검색 `⌘F` / 커맨드 팔레트 `⌘K` / 파일 빠른 열기 `⌘P` / 전체 검색 `⌘⇧F`

**핵심 규칙**
- `⌘F`: 활성 맵의 노드 텍스트+메모+링크를 부분 일치 검색. Enter/Shift+Enter로 결과 순환, 결과 노드 자동 선택+포커스.
- `⌘⇧F`(`GlobalSearch.tsx`): 워크스페이스 전체 — 모든 `.mind`의 노드 + 모든 `.md`의 제목·본문·메타 값을 대상으로 하는 전역 검색. 열 때 1회 전체 스캔 후 메모리 필터링.
- `⌘K`: 명령(새 맵, 분할, 테마, 내보내기 등 컨텍스트별) + 전체 파일 통합 검색. **활성 맵이 있으면 노드 텍스트도 "노드" 그룹으로 함께 검색**하고(`CommandPalette.tsx`), 쿼리가 있으면 항상 하단에 "전체에서 찾기(⌘⇧F)" 힌트를 보여준다 — 클릭하면 같은 쿼리로 `GlobalSearch`를 이어서 연다(IA-STRATEGY §5-1, 2026-07-10).
- **자동 학습 단축키(Quick Keys)**(REDESIGN-VISION §3-7, 2026-07-10): 명령을 2회 이상 실행하면 다음부터 빈 검색어 목록 맨 위로 올라오고 `⌥1`~`⌥9` 배지가 붙는다 — 팔레트가 열려 있으면 검색어 입력 중이어도 `⌥+숫자`로 그 명령을 바로 실행(macOS Tahoe Spotlight Quick Keys와 동일 개념). 사용 횟수는 `localStorage`(`cmdUsage`, `src/ui/commandUsage.ts`)에 명령 id별로 누적 — 노드·파일 검색 결과는 대상이 아니고 "명령" 그룹만 학습한다.
- `⌘P`: 파일만 빠르게 (이름+폴더 검색, 최대 50개). 파일명으로 못 찾으면 빈 상태에 같은 "전체에서 찾기" 탈출구를 보여준다.

**알려진 한계**: 노트 패널에서는 `⌘F` 불가(맵 전용). `⌘⇧F`는 워크스페이스가 크면 열 때마다 다시 스캔(캐시 없음).

## 11c. 집중 세션 · 작업 기록 ⚠️

**무엇**: 노드를 대상으로 집중 작업 시간을 측정·기록하고 대시보드로 돌아본다. 마인드맵에 "실행/시간" 축을 더하는 기능 ([명세](specs/2026-06-12-focus-session.md), [decisions/0006](../decisions/0006-focus-session-and-rename.md)).

**진입점**: 노드 우클릭 "집중 세션 시작" / 타이머 위젯 종료 / 사이드바 하단 "작업 기록"(대시보드) / 노드 집중 칩 클릭(대시보드).

**핵심 규칙**
- 시작: `work-log/` 폴더 보장 후 세션 노트 생성(제목=시작시각·노드, 읽기전용 메타 배너), 노드 자동 연동, 조상 체인 스냅샷. 좌하단 타이머 pill(사이드바 표시 시 도킹/숨김 시 플로팅).
- 진행 중: 노트 상단 읽기전용 배너에 라이브 타이머+종료. 사용자는 본문(목표·작업 기록)에 자유 기록.
- 종료: epoch 기반 durationSec(절전/DST 견딤, 12h 초과는 추정 플래그). 완료 카드(총시간·오늘 누적·연속·이 주제 누적 + 회고 한 줄). 한 번에 하나의 세션.
- 데이터: 노트 프론트매터 `session`(sessionId·link·ancestors·start/end·durationSec·reflect)이 진실의 원천 → 워크스페이스 인덱스가 실어 대시보드 메모리 집계. 깨진 세션은 노출.
- 노드 칩: 캔버스 노드에 "⏱ N회·누적시간"(서브트리 롤업) → 마인드맵이 노력 히트맵.
- 대시보드(오버레이): 요약(오늘/주간/세션/연속) + 연간 히트맵 + 최근 세션 타임라인 + 주제별 누적 막대.
- streak: 연속일 + 1일 유예, 5분 미만 세션은 streak 불산입(게이밍 방지).

**알려진 한계**: 일시정지·유휴 감지 없음(v1). 노드 없는 임시 세션 불가(노드 귀속 필수). 자정 넘긴 세션은 시작일에 귀속.

## 11b. 자동 업데이트 ⚠️

**무엇**: 정식 빌드는 GitHub Releases에서 새 버전을 자동 확인·다운로드하고, 받아지면 재시동을 안내한다 (electron-updater — [명세](specs/2026-06-11-auto-update.md)).

**진입점**: 자동(시작 10초 후 + 4시간마다, 백그라운드) / 수동은 앱 메뉴 → "업데이트 확인…"

**핵심 규칙**
- 다운로드 완료 시에만 대화상자("새 버전 준비됨 — 지금 재시동 / 나중에"). "나중에"면 다음 종료 시 적용.
- 백그라운드 확인·다운로드 실패는 사용자에게 침묵(로그만). 수동 확인일 때만 "최신입니다"·실패를 안내.
- 정식 빌드(`app.getName()==='MindMap'` + packaged)에서만 동작 — MindMap Dev·`npm run dev`는 비활성.
- 무결성: zip의 sha512를 yml과 대조(electron-updater 내장) + Squirrel.Mac이 Developer ID 서명 검증 → 미서명/위변조 업데이트는 적용 불가.

**알려진 한계**: macOS arm64만. 델타(차등) 업데이트는 미사용(전체 다운로드). `MINDMAP_UPDATE_URL` env로 피드 대체(E2E 전용).

## 11. 테마 · 환경설정 ⚠️

- 라이트/다크: `⌘⇧L` / 메뉴 / 사이드바 설정 / `⌘K`. `data-theme` 속성 + CSS 토큰으로 전환.
- 글자 크기: fontScale 0.8~1.5 (10% 단위).
- **시스템 테마 자동 추종 없음** (수동 전환만 — 로드맵 후보).

## 12. Undo/Redo ⚠️

- `⌘Z` / `⌘⇧Z`, 히스토리 100단계. 모든 구조 변경 포함, 뷰(줌/팬) 변경은 제외.
- undo로 살아난 노드의 리마인더는 다음 동기화 사이클에서 자동 재생성.

## 13. 휴지통 ⚠️

**무엇**: 사이드바에서 파일·폴더를 삭제하면 즉시 파괴하지 않고 워크스페이스 `.trash/` 폴더로 이동, 매니페스트 기반으로 복원할 수 있다 (`src/store/trashStore.ts`, `src/ui/TrashPanel.tsx` — v0.7.11).

**진입점**: 사이드바 하단 바 휴지통 아이콘 → 전용 패널(항목별 복원 / 완전 삭제, 전체 비우기).

**핵심 규칙**
- 삭제된 항목은 원래 경로를 매니페스트에 기록 → 복원 시 원위치로 되돌아간다.
- 완전 삭제(휴지통 건너뛰기) 옵션 지원.
- 사이드바 하단 바 배지가 휴지통 항목 수를 표시 (`useTrash` 스토어가 워크스페이스 변경 시 갱신).

**알려진 한계**: 자동 비우기(보관 기간) 없음 — 수동으로만 비움.

## 14. AI 프로바이더 설정 (인프라) ⚠️

**무엇**: 설정 화면에서 Claude·OpenAI API 키를 저장하고 활성 프로바이더를 전환할 수 있다. 현재는 키 저장·마스킹 표시·활성 전환 인프라만 존재하며, 이 키를 소비하는 실제 AI 기능(요약·생성 등)은 아직 구현되지 않았다 (v0.7.23–0.7.24).

**진입점**: 설정(⌘,) → "AI 기능" 섹션 → Claude/OpenAI 키 입력 및 활성 프로바이더 선택.

**핵심 규칙**
- 키는 렌더러에 평문으로 노출되지 않고 `safeStorage.encryptString`으로 암호화해 `userData/`에 저장, IPC(`ai:setKey`/`ai:openai:setKey`/`ai:getActive`/`ai:setActive` 등)로만 접근.
- 두 프로바이더 키를 동시에 보유 가능하나 활성(active)은 항상 하나.
- UI에는 마스킹된 키만 표시(`ai:getMasked`/`ai:openai:getMasked`).

**알려진 한계**: 저장된 키를 실제로 사용하는 기능이 아직 없다 — 순수 설정 인프라 단계.

## 15. 노트 템플릿

**무엇**: 자주 쓰는 노트 구조(회의록, 회고 등)를 템플릿으로 저장해두고, 다른 노트를 쓸 때 커서 위치에 그대로 불러와 삽입한다. 설정에서 기능 자체를 켜고 끌 수 있다 ([명세](specs/2026-07-09-note-templates.md)).

**진입점**
- 노트 편집기 툴바 "템플릿+" 버튼(검색 가능한 드롭다운) → 항목 선택 시 커서 위치에 본문 삽입.
- 사이드바 하단 "Note Template" 아이콘 버튼(뱃지: 보유 템플릿 수) → 목록·검색·새 템플릿 작성·삭제.
- 설정 → "노트 템플릿" 켜짐/꺼짐 세그먼트 토글.

**핵심 규칙**
- 템플릿은 워크스페이스의 숨김 시스템 폴더 `.templates/`에 보통 `.md` 노트로 저장(폴더 위치 자체가 구분자 — 별도 확장자·프론트매터 플래그 없음). `.trash`와 동일하게 dot-prefix라 사이드바 트리 워크에서 자동 제외된다.
- 폴더는 워크스페이스를 열 때, 그리고 설정 토글을 켤 때 없으면 즉시 재생성된다(휴지통의 "필요 시 지연 생성"과 달리 상시 존재 보장).
- 삽입은 `editor.chain().focus().insertContent(markdown)`으로 본문만(프론트매터 제외) 커서 위치에 넣는다 — 구조화 메타데이터를 넣는 "양식+"(항목 6 참조, 구 "메타+")와는 완전히 분리된 별개 기능.
- 토글을 끄면 툴바의 "템플릿+" 버튼과 사이드바의 "Note Template" 버튼이 함께 즉시 숨겨진다(내부적으로 하나의 `templatesEnabled` 상태가 두 UI를 제어).
- 양식+ 드롭다운에도 동일한 검색창을 리트로핏 — 항목이 늘어나도 스크롤이 길어지지 않게.
- **탭·편집 화면 구분**: 템플릿 탭은 일반 노트와 다른 아이콘(`tab-ic--template`, `src/panes/TabBar.tsx`)으로 표시되고, 편집 화면 제목 옆에 "템플릿" 배지(`.note-template-badge`)가 항상 보인다. 템플릿은 마인드맵 그래프에 속하지 않으므로 "노드에 연동" 버튼은 숨겨지고, 빈 본문 placeholder도 템플릿 전용 문구("템플릿 내용을 작성하세요…")를 쓴다 — 다중 탭 상황에서 노트/템플릿을 혼동하지 않도록 하는 조치 (2026-07-10 UX 감사 F-1/F-2).
- 툴바 "템플릿+ → 새 템플릿 만들기"와 사이드바 패널의 "+ 새 템플릿"은 둘 다 즉시 새 탭으로 연다(과거엔 툴바 쪽이 토스트만 띄우고 조용히 만들어 사용자가 직접 사이드바에서 찾아야 했음 — F-3 수정). 기본 제목은 일반 노트의 "제목 없음"과 겹치지 않도록 "새 템플릿"을 쓴다(F-4 수정).
- **템플릿에는 메타 블록("양식+")을 추가할 수 없다**: 삽입은 본문 텍스트만 옮기므로 템플릿에 붙은 메타 블록은 어차피 다른 노트로 전달되지 않고, 삽입 대상 노트가 같은 템플릿의 메타 블록을 이미 갖고 있으면 혼란을 준다. `NotePane.tsx`가 템플릿 탭에서는 `NoteMetaBlocks` 렌더와 `EditorToolbar`의 `templates`/`onAddMeta` prop을 모두 꺼서 "양식+" 버튼 자체가 나타나지 않는다.
- **양식+/템플릿+ 툴바 버튼 아이콘 차별화**: 라벨이 비슷해("X+") 헷갈리기 쉬웠던 두 버튼에 아이콘을 추가 — 양식+는 `table` 아이콘(`--primary`, 설정의 "정보 양식" 표기와 동일 색), 템플릿+는 `template` 아이콘(`--state-note`, 템플릿 정체성 색) (IA-STRATEGY §5-3 / REDESIGN-VISION §3-6, 2026-07-10 "메타+"에서 개명).

**알려진 한계**: 기존 노트를 "템플릿으로 저장"하는 단축 경로 없음(템플릿 전용 패널에서 새로 작성만 가능). 템플릿 하위 폴더·변수 플레이스홀더(`{{날짜}}` 등)·단축키 미지원.

## 16. 전역 즉시 캡처 (⌥Space)

**무엇**: 앱이 백그라운드에 있거나 메인 창이 닫혀 있어도(완전 종료 전까지) 전역 단축키로 작은
캡처 입력창이 뜨고, 텍스트를 입력해 Enter를 누르면 워크스페이스 루트의 "오늘의 생각.mind"에
새 루트 노드로 즉시 쌓인다 — [계획](../exec-plans/completed/2026-07-10-global-capture.md).

**진입점**
- 전역 단축키 `⌥Space`(macOS 전용, `electron/main.ts`의 `globalShortcut`).
- `⌘K` "빠른 캡처 열기" 명령 — 단축키를 몰라도/충돌해도 같은 기능에 도달할 수 있는 보조 경로.

**핵심 규칙**
- 캡처 창은 별도의 작은 `BrowserWindow`(테두리 없음, 투명 배경, 항상 위, 크기 고정)로, 기존
  렌더러 번들을 `?capture=1` 쿼리로 로드해 `CaptureWindow.tsx`만 그린다 — `.mind` 스키마 지식은
  여전히 `src/io/formats.ts`에만 있고 `electron/main.ts`는 파일 경로만 안다(아키텍처 규칙 유지).
- 창은 닫히지 않고 숨겨진다(재사용) — 다음 호출 시 즉시 뜬다. 포커스를 잃으면(다른 곳 클릭)
  자동으로 숨겨진다(Spotlight와 동일).
- Enter로 저장, Esc로 취소(저장 안 함). 저장 시 3-2의 자연어 파싱(`parseScheduleText`/
  `parseHashtagColor`)이 그대로 적용돼 "내일 3시 치과 #red"처럼 캡처해도 일정·색이 자동으로
  붙는다 — 맵에 직접 입력한 것과 동일 경로.
- 단축키 등록 실패(다른 앱이 이미 `⌥Space`를 선점)는 앱을 막지 않고 조용히 비활성화되며,
  설정 → 고급 설정에 상태 줄로 노출된다("Alt+Space를 다른 앱이 사용 중이라 꺼져 있어요").
- macOS 전용(다른 플랫폼은 애초에 앱 대상이 아님). `globalShortcut`은 시스템 권한(손쉬운 사용
  등)이 필요 없다(입력 시뮬레이션이 아니라 OS가 키 조합을 이 앱으로 라우팅하는 것뿐).

**알려진 한계**: 단축키 커스터마이징 UI 없음(고정 `⌥Space`, 충돌 시 안내만). 트레이 상주 없음 —
앱을 완전히 종료(⌘Q)하면 캡처도 꺼진다. "오늘의 생각.mind"가 메인 창에서 동시에 열려 있는
상태로 캡처하면 마지막 저장이 이긴다(드문 시나리오로 판단, v1은 감내).
