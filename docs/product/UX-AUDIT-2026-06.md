## 버그 (재현 확인)

### B1. 노트 연결 픽커의 Esc가 입력창 포커스에 종속 — P1 → ✅ 수정 (2026-06-11)
- 증상: "노트 추가·연결" 픽커에서 `Esc`가 동작하지 않는 경우 발생 (푸터에 "esc 닫기" 힌트가 있는데도). Esc 핸들러가 검색 입력창의 onKeyDown에만 있어서, 포커스가 입력창을 벗어나면(픽커 내부의 버튼 아닌 영역 클릭 등) Esc가 무력화됨.
- 부수 발견: 픽커가 떠 있는 동안 `⌘K`로 커맨드 팔레트가 **픽커 위에 겹쳐** 열림(모달 배타성 없음, 스크린샷 13).
- 수정: 픽커에 윈도우 캡처 단계 Esc 리스너 추가(ContextMenu·SchedulePopover와 동일 패턴), App의 ⌘K/⌘P 핸들러에 픽커 열림 가드 추가. **빌드 앱에서 검증 완료** (Esc 닫힘, 픽커 위 ⌘K 차단).

### B2. 공백만 입력한 노드가 빈 노드로 잔존 — P2 → ✅ 수정 (2026-06-11)
- 정밀 원인: 빈 텍스트 자동 삭제의 trim 판정은 이미 있었음. 삭제가 **불가한** 노드(예: 유일한 루트)에서 공백 텍스트가 trim 없이 그대로 저장되는 것이 원인.
- 수정: `commitText`가 저장 시 trim하도록 변경 + Vitest 회귀 케이스 추가 (17/17 통과).

### B3. 커밋 직후 50ms 내 Enter 무시 → 형제 생성 누락 — P3
- 편집 커밋(Enter) 직후 50ms 가드 때문에 아주 빠른 연속 Enter 시 형제 노드 생성이 조용히 누락된다. 자동화/매크로 수준 속도라 실사용 빈도는 낮지만, "키보드 속도가 제품 정체성"이므로 인지 필요.

## UX 이슈

### U1. ~~선택 툴바 아이콘 대비 부족~~ — ❌ 기각 (테스트 환경 아티팩트)
- 스크린샷 09에서 아이콘이 의도 색(#31302e)의 ~21% 불투명도로 측정돼 P1로 분류했으나, 픽셀 분석 결과 원인은 제품이 아니라 **테스트 환경**: 가려진(occluded) 창에서 Chromium이 CSS 애니메이션을 정지시켜 등장 애니메이션(`selPop`, opacity 0→1)이 중간에 얼어붙은 상태가 찍힌 것. 포커스된 실사용 창에서는 0.2초 후 정상 대비.
- 교훈으로 기록: 스크린샷 기반 시각 판정은 애니메이션 완료 보장 후에.

### U2. 우클릭 메뉴가 하단 툴바에 가려짐 — P1 → ✅ 수정 (2026-06-11)
- 화면 중앙~하단 노드에서 우클릭 시 메뉴(항목 14개, 약 420px)가 하단 줌 툴바와 겹쳐 마지막 항목 **"삭제"가 가려진다** (스크린샷 10 픽셀 확인).
- 원인: 메뉴는 z-index 70, 툴바는 20이지만 **서로 다른 스태킹 컨텍스트**(메뉴는 캔버스 쪽, 툴바는 팬 쪽)라 툴바가 위에 그려짐. 위치 클램프는 뷰포트 가장자리만 고려.
- 수정: 위치 클램프의 하단 여백을 8px → 76px(툴바 밴드)로. 항목 수 축소(중복 진입점 정리)는 U4와 함께 별도 검토.

### U3. "제목 없음" 파일 누적 — P2 → ✅ 수정 (2026-06-11)
- 홈 CTA "새 마인드맵"은 이름 입력/변경 단계 없이 "제목 없음.mind"를 만들고 끝. 실제 워크스페이스에서 "제목 없음", "제목 없음 2" 누적 확인.
- 수정: **첫 중심 토픽 텍스트가 파일 이름이 됨** (이름 다이얼로그 없이 — 속도 원칙 유지). 명세: [specs/2026-06-11-untitled-autoname.md](specs/2026-06-11-untitled-autoname.md). 격리 환경에서 E2E 검증("제목 없음.mind" → "출시 준비 로드맵.mind").

### U4. 진입점 분산 (발견성) — P2 → 🔶 1차 해소 (2026-06-11)
- 표준을 [동작별 진입점 매트릭스](specs/2026-06-11-entrypoint-matrix.md)로 명문화. 위반 2건 수정:
  - **노트 연동 해제를 노드 쪽에 추가** — 노트 팝업 헤더 "연동 해제" 버튼 (E2E 검증: 칩 제거 + frontmatter links 비움)
  - **화면 분할을 보기 메뉴에 추가**
- 아이콘 지정(메뉴→팝오버 경로)은 매트릭스상 위반 아님으로 판정. 남은 것: 우클릭 메뉴 14항목 그룹핑 — 팀 합의 후.

### U5. 비정상 종료 시 세션 복원 누락 가능성 — 검증 필요
- 정상 흐름에서는 탭·분할 상태가 잘 복원되는 것을 확인했다. 그러나 렌더러 강제 종료 직후 재실행에서 한 차례 홈 화면이 떴다(탭 미복원). 테스트 환경의 다중 인스턴스 간섭일 수 있어 단정하지 않음 — 단일 인스턴스에서 재검증 필요.

## 기술 관찰 (이슈 아님, 인지용)

- **단일 인스턴스 락 없음**: dev 빌드와 패키지 앱을 동시에 띄우면 같은/다른 userData 조합에 따라 세션·설정이 꼬일 수 있다. `app.requestSingleInstanceLock()` 검토 가치 있음.
- **⌘Z는 네이티브 메뉴 전용**: 인페이지 키 핸들러에 없음. 사용자 영향은 없으나 E2E 자동화 시 메뉴 경유 필요.
- **recent/세션이 localStorage 기반**: dev(localhost)와 패키지(file://)는 오리진이 달라 서로 이어지지 않음. 사용자 데이터가 아닌 UI 상태라 수용 가능.

## 잘 작동하는 것 (지킬 것)

- **키보드 루프의 속도**: Enter/Tab/방향키로 맵을 만드는 핵심 루프가 즉각적이고 자연스럽다. 새 노드 즉시 편집 진입, 빈 노드 자동 삭제(공백 제외, B2)도 의도대로.
- **자동 저장 신뢰성**: 렌더러를 강제로 죽여도 디바운스(1초) 이전 작업분 외 데이터 손실 없음 — "데이터 소유" 약속과 부합.
- **빈 맵 엠티 스테이트**: "Enter를 눌러 중심 주제를 만드세요" 안내 명확. (더블클릭 생성도 안내에 추가할지는 선택.)
- 탭의 dirty 인디케이터(파란 점), 사용된 색만 노출하는 하단 색상 필터 도트, 다크 모드 일관성.
