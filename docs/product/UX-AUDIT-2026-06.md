# UX 여정 감사 (2026-06)

> 작성: 2026-06-11 · 방법: 빌드된 앱(`npm run build` 산출물)을 Playwright로 직접 구동·관찰 (스크린샷 19장)
> 범위: 여정 1(맵 작성) 전체, 여정 3(노트 연결) 부분. 여정 2(리마인더)는 실데이터 보호를 위해 reminderOn 활성화 없이 UI까지만 — 전체 검증은 [QA-CHECKLIST](../release/QA-CHECKLIST.md)로.

## 버그 (재현 확인)

### B1. 노트 연결 픽커가 Esc로 닫히지 않음 — P1
- 재현: 노드 우클릭 → "노트 추가·연결" → `Esc` → 닫히지 않음. 푸터에 "esc 닫기" 힌트가 있는데도.
- 더 큰 문제: 픽커가 떠 있는 동안 `⌘K` `⌘P` `⌘F` 전역 단축키가 전부 막힘. 바깥 클릭으로만 탈출 가능.
- 같은 방식의 Esc 입력이 컨텍스트 메뉴·스케줄 팝오버는 정상적으로 닫았으므로 픽커 고유의 핸들러 문제로 보임 (NoteLinkPicker의 키 핸들러 확인 필요).

### B2. 공백만 입력한 노드가 자동 삭제되지 않음 — P2
- 빈 텍스트로 편집을 끝내면 노드가 자동 삭제되는데, 공백 한 칸(" ")이면 살아남아 빈 모양의 노드가 맵에 잔존.
- 빈 텍스트 판정에 `trim()` 미적용으로 추정 (`mapStore`의 commitText / 자동 삭제 경로).

### B3. 커밋 직후 50ms 내 Enter 무시 → 형제 생성 누락 — P3
- 편집 커밋(Enter) 직후 50ms 가드 때문에 아주 빠른 연속 Enter 시 형제 노드 생성이 조용히 누락된다. 자동화/매크로 수준 속도라 실사용 빈도는 낮지만, "키보드 속도가 제품 정체성"이므로 인지 필요.

## UX 이슈

### U1. 선택 툴바 아이콘 대비 부족 — P1
- 노드 선택 시 뜨는 툴바의 아이콘 8개가 모두 옅은 회색으로, 전부 **비활성 상태처럼** 보인다 (스크린샷 09). 색상 도트조차 회색.
- 제안: 기본 상태 대비 상향(`--text-secondary` 수준), hover 강조. [COLOR-SYSTEM](../design/COLOR-SYSTEM.md)의 인터랙션 계층 토큰 활용.

### U2. 우클릭 메뉴가 하단 툴바에 가려짐 — P1
- 화면 중앙~하단 노드에서 우클릭 시 메뉴(항목 14개, 약 420px)가 하단 줌 툴바와 겹쳐 마지막 항목 **"삭제"가 가려진다** (스크린샷 10, 창 높이 820px 기준).
- 메뉴 위치 보정이 뷰포트 가장자리만 고려하고 하단 툴바를 피하지 않음. 항목 수 축소(중복 진입점 정리)와 병행 검토.

### U3. "제목 없음" 파일 누적 — P2
- 홈 CTA "새 마인드맵"은 이름 입력/변경 단계 없이 "제목 없음.mind"를 만들고 끝. 사이드바 생성 경로만 인라인 이름 변경 모드로 진입한다.
- 실제 워크스페이스에서 "제목 없음", "제목 없음 2"가 이미 누적된 것을 확인. 생성 경로 간 일관성 필요 — 홈 CTA도 이름 변경 모드로 시작하거나, 첫 루트 노드 텍스트를 파일명으로 제안.

### U4. 진입점 분산 (발견성) — P2
- [기능 인벤토리 부록 B](FEATURE-INVENTORY.md) 그대로 확인됨. 특히: 아이콘 지정은 노드 팝오버 안에만, 노트 연결 해제는 노트 패널에서만, 화면 분할은 드래그/⌘K뿐.
- 컨텍스트 메뉴(14개 항목)와 선택 툴바(8개 버튼)의 중복도 정리 대상 — "무엇이 어디에 있는가" 원칙을 한 번 정하고 일괄 적용 권장.

### U5. 비정상 종료 시 세션 복원 누락 가능성 — 검증 필요
- 정상 흐름에서는 탭·분할 상태가 잘 복원되는 것을 확인했다. 그러나 렌더러 강제 종료 직후 재실행에서 한 차례 홈 화면이 떴다(탭 미복원). 테스트 환경의 다중 인스턴스 간섭일 수 있어 단정하지 않음 — 단일 인스턴스에서 재검증 필요.

## 기술 관찰 (이슈 아님, 인지용)

- **단일 인스턴스 락 없음**: dev 빌드와 패키지 앱을 동시에 띄우면 같은/다른 userData 조합에 따라 세션·설정이 꼬일 수 있다. `app.requestSingleInstanceLock()` 검토 가치 있음.
- **⌘Z는 네이티브 메뉴 전용**: 인페이지 키 핸들러에 없음. 사용자 영향은 없으나 E2E 자동화 시 메뉴 경유 필요.
- **recent/세션이 localStorage 기반**: dev(localhost)와 패키지(file://)는 오리진이 달라 서로 이어지지 않음. 사용자 데이터가 아닌 UI 상태라 수용 가능.

## 잘 작동하는 것 (지킬 것)

- **키보드 루프의 속도**: Enter/Tab/방향키로 맵을 만드는 핵심 루프가 즉각적이고 자연스럽다. 새 노드 즉시 편집 진입, 빈 노드 자동 삭제(공백 제외, B2)도 의도대로.
- **자동 저장 신뢰성**: 렌더러를 강제로 죽여도 디바운스(1초) 이전 작업분 외 데이터 손실 없음 — "데이터 소유" 약속과 부합.
- **빈 맵 엠티 스테이트**: "Enter를 눌러 중심 주제를 만드세요" 안내 명확. (더블클릭 생성도 안내에 추가할지는 선택.)
- 탭의 dirty 인디케이터(파란 점), 사용된 색만 노출하는 하단 색상 필터 도트, 다크 모드 일관성.
