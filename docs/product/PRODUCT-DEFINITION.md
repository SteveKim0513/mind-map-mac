# 제품 정의 (1페이지)

> 작성: 2026-06-10 · 상태: 초안 — 팀 합의 필요
> 근거: [RESEARCH-mindmap-competitors.md](reports/RESEARCH-mindmap-competitors.md) (검증된 경쟁사 불만 분석)

## 한 줄 정의

**생각(마인드맵), 기록(노트), 할 일(일정·리마인더)을 하나의 로컬 파일 기반 macOS 앱에서 잇는 키보드 중심 사고 도구.**

## 타깃 사용자

- Apple 생태계의 **개인 지식 작업자** (기획자, 개발자, 연구자, 작가)
- 클라우드 락인을 싫어하고 **내 파일은 내가 소유**하길 원하는 사람
- 마우스보다 키보드가 빠른 사람

협업 팀은 (현재 버전의) 타깃이 아니다.

## 왜 우리인가 — 검증된 시장 불만과의 대응

경쟁사 분석에서 사용자가 반복적으로 떠나는 지점 4가지가 확인됐다. 각각에 대한 우리의 답:

| 시장의 검증된 불만 | 우리의 답 |
|---|---|
| 데이터 손실·락인 공포 (MindMeister: "취소하면 맵이 사라진다") | **로컬 파일이 원본** (.mind = JSON, 노트 = Markdown). 서버 없음. Markdown/OPML 내보내기 내장 |
| 분산 과금 피로 (맵·노트·캘린더·리마인더 따로 결제) | **맵 + 노트 + 일정 + 리마인더가 한 앱** — 통합이 핵심 차별점 |
| 대형 맵 성능 저하 (Xmind·Miro 공통) | 네이티브 데스크탑 + LOD 렌더링. **대형 맵 성능을 지속 측정 대상으로 삼는다** |
| 플랫폼 폐쇄성 | 우리도 macOS 전용(MindNode와 동일한 약점) — 대신 **표준 포맷(MD/OPML)으로 데이터는 항상 열려 있다**로 방어 |

## 핵심 경험 3가지 (이 순서로 지킨다)

1. **속도** — 키보드만으로 생각의 속도로 맵을 만든다 (Tab/Enter/화살표).
2. **통합** — 맵의 노드가 곧 일정(macOS 리마인더 양방향 동기화)이고, 노드에 노트가 붙는다. 앱을 떠날 필요가 없다.
3. **소유** — 모든 데이터는 사용자의 폴더에 사람이 읽을 수 있는 파일로 존재한다.

이 셋은 제품이 **어떻게** 좋아야 하는지(경험 품질)를 말한다. 아래 "기능의 3가지 축"은 기능이
**무엇을 위한 것인지**(사고 과정에서의 역할)를 말한다 — 서로 다른 질문에 답하는 별개의 기준이다.

## 기능의 3가지 축 (2026-07-15, [결정 0011](../decisions/0011-three-axis-thinking-model.md))

이 도구가 사용자의 사고 과정을 돕는 방식을 세 축으로 정의한다. 화면의 기능 그룹핑·계층은 이 축을
기준으로 한다(표면적 유사성이 아니라 "어떤 축에 속하는가"로 묶는다).

| 축 | 정의 | 답하는 질문 | 대표 기능 |
|---|---|---|---|
| **정리 (Organize)** | 생각을 포착하고 구조화한다 | "이게 뭐고, 다른 것과 어떻게 연결돼 있나?" | 노드 생성·편집·구조 조작, 색·아이콘, 메모·링크, 연결선·섹션 |
| **실행 (Execute)** | 정리된 것 중 해야 할 일을 실제로 해낸다 | "언제 할 거고, 하는 동안 뭐가 도와주나, 끝났나?" | 완료 체크, 일정(리마인더 동기화), 집중 세션 |
| **통찰 (Insight)** | 더 깊이 쓰고, 되짚어 이해한다 | "이게 무슨 의미고, 지금까지 뭘 해왔나?" | 노트 시스템(노드-노트 연결), 집중 기록 대시보드 |

세 축은 순환한다: **정리 → 실행 → 통찰 → (다시) 정리.** 집중 세션은 "실행" 축에 속하며, 그 개념적
출발점은 **일정이 지정된 노드**다 — 일정을 걸고(실행 시작) 때가 되면 집중 세션이 실행을 돕고, 끝나면
노트·대시보드로 되짚는다(통찰). 전체 기능-축 매핑과 화면 구조 재검토는
[FEATURE-HIERARCHY-VISION-2026-07.md](FEATURE-HIERARCHY-VISION-2026-07.md) 참고.

## 범위에서 뺀 것 (v1 Non-goals)

## AI 기능 — 방향 전환 (2026-07-10)

AI 기능은 더 이상 비목표가 아니다. **핵심 확장 축으로 계획 중** — 이미 설정에 있는 Claude/OpenAI 프로바이더 인프라(FEATURE-INVENTORY #14)가 그 출발점이다. 구체적 범위(요약·생성·대화 등)와 어느 핵심 경험(속도/통합/소유)에 연결할지는 아직 확정 전이며, 확정되는 대로 별도 명세(`docs/product/specs/`)와 [IA-STRATEGY-2026-07.md](IA-STRATEGY-2026-07.md)의 기능 계층 재배치로 이어간다.
