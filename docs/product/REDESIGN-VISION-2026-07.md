> 작성: 2026-07-10 · 상태: **초안 — 방향 합의 필요 (구현 전)**
> 근거: Apple HIG·Notes·Reminders·Freeform·Spotlight 벤치마크 조사, 토스 공개 디자인 원칙(toss.tech) 조사 — 각 항목에 출처 표기
> 관계: [IA-STRATEGY-2026-07.md](IA-STRATEGY-2026-07.md)를 대체하지 않는다 — 그 문서는 "있는 걸 재배치"였고, 이 문서는 "경험 자체를 다시 설계"다. IA 문서의 완료 항목(검색 통합·메뉴 그룹핑 등)은 이 비전의 **1단계 안에 흡수**된다.

## 0. 왜 지난 작업은 "바뀐 게 없어" 보였는가

지난 IA 작업(검색 3종에 힌트 추가, 사이드바 아이콘 묶기, 메뉴에 레이블 달기)은 전부 **"이미 있는 화면 요소의 재배치"**였다. 배경색을 묶고 아이콘을 붙이고 레이블을 추가하는 건 구조적으로는 개선이지만, 사용자가 실제로 겪는 **행동 하나(생각을 적는다 → 정리한다 → 가끔 시간을 건다 → 가끔 깊게 쓴다)**는 조금도 안 바뀌었다. Apple과 토스가 "탈바꿈"으로 느껴지는 이유는 화면을 재배치해서가 아니라, **사용자가 매번 내려야 했던 결정 자체를 없앴기 때문**이다 — "어느 폴더에 넣지", "날짜를 어떻게 입력하지", "이 설정이 뭐 하는 거지" 같은 질문이 아예 발생하지 않게 만든다. 이번 재설계는 **화면 재배치가 아니라 결정 제거**를 목표로 한다.

## 2. 핵심 재설계 — 사고 모델 전환

**지금**: "파일 관리자 + 탭 + 여러 종류의 문서" (Finder에 가까움)
**목표**: "생각을 위한 하나의 캔버스, 파일은 그 뒤에 조용히 있음" (Freeform + Spotlight에 가까움)

이 전환은 데이터 모델(.mind/.md 파일, 워크스페이스)을 바꾸는 게 아니라 — 그건 이미 검증된 핵심 자산(로컬 파일 소유권, PRODUCT-DEFINITION의 3대 기둥 중 "소유")이라 안 건드린다 — **그 위에 있는 진입·전환 경험**을 바꾸는 것이다.

## 3. 구체적 재설계 제안

### 3-1. 전역 즉시 캡처 (A1 + A4 + T2) — 가장 임팩트 큰 단일 변경

> **구현 완료 (2026-07-10)** — `⌥Space`, "오늘의 생각.mind"로 캡처, `⌘K` "빠른 캡처 열기"도 병행.
> [계획](../exec-plans/completed/2026-07-10-global-capture.md), [인벤토리 §16](FEATURE-INVENTORY.md#16-전역-즉시-캡처-space).

- **문제**: 지금 "생각 하나를 적으려면" 먼저 어느 맵을 열지 골라야 한다. Toss의 3초 규칙으로 보면 "어느 맵에?"는 답하는 데 3초 넘게 걸리는, 안 물어봐도 될 질문이다.
- **제안**: 전역 단축키(예: `⌥Space`, macOS 다른 앱과 충돌 없는 조합 확정 필요) → 화면 어디서든 작은 캡처 바가 뜬다. 타이핑하고 Enter → **"오늘의 생각"이라는 항상 존재하는 워크스페이스 루트 맵**(Notes의 "이어쓰는 Quick Note"와 동일 개념)에 노드로 즉시 들어간다. 나중에 사이드바에서 그 노드를 실제 맵으로 드래그하면 된다(기존 드래그 재배치 로직 재사용).
- 이건 새 기능이 아니라 **"새 마인드맵" 버튼이 지금 강요하는 결정(제목 뭐로 할지, 어디 저장할지)을 제거**하는 것.

### 3-2. 자연어 입력 — 날짜·태그를 타이핑의 부산물로 (A1)

> **구현 완료 (2026-07-10)** — 인식된 텍스트를 자르거나 인라인 하이라이트하지는 않기로 스코프
> 축소(기존 일정 칩·색 틴트가 신호를 대신함). **2026-07-14 수정**: "오늘"/"내일" 등을 문장 어디서든
> 인식하던 방식이 라벨 습관("오늘 : 회의 메모")과 자주 충돌한다는 실사용 리포트로, `#색상`과
> 대칭되는 `@`(예: "@내일 3시") 마커를 필수화했다. [계획](../exec-plans/completed/2026-07-10-nl-schedule-parsing.md).

- 노드 편집 중 "내일 3시", "다음 주 월요일" 같은 문구를 감지해 `scheduled`+`scheduleAt`을 자동으로 채우고, 인식된 부분은 노드 텍스트에서 옅게 하이라이트만 남긴다(Reminders와 동일). SchedulePopover는 없어지지 않고 **"자동 인식이 틀렸을 때 고치는 보조 경로"**로 격하된다(A3의 "기본 하나, 복잡한 건 한 클릭 아래"와 같은 구조).
- 색상도 `#`로 시작하는 토큰을 타이핑하면 태그 팔레트에서 매칭되는 색으로 자동 지정 — 지금처럼 우클릭 메뉴의 색 그리드를 열 필요가 기본 경로에서는 없어진다.

### 3-3. Smart Views — 사이드바를 "장소+필터"의 단일 모델로 (A2)

> **구현 완료 (2026-07-10)** — "최근 수정"·"즐겨찾기" 2개 추가("미완료"는 제외, 아래 참고).
> [계획](../exec-plans/completed/2026-07-10-smart-views.md).

- "오늘", "최근 수정", "즐겨찾기(핀)", "미완료" 같은 저장된 조회를 사이드바 트리 **안에**(지금처럼 트리 밖 별도 전체너비 행이 아니라) 일반 폴더와 구분 안 되는 모양으로 넣는다. 아이콘 색만 살짝 다르게(이미 있는 시맨틱 토큰 재사용 — 새 색 도입 안 함).
- Note Template·휴지통도 이 관점에서는 "관리형 필터"의 일종 — 3-4에서 다룰 설정 축소와 함께 재배치 검토.

### 3-4. 설정 화면 — "옵션 더 보기" 패턴 (A3)

> **구현 완료 (2026-07-10)** — 테마·워크스페이스·노트 템플릿만 기본 노출, 나머지는 "고급 설정 보기".

- 지금 설정 메인 화면은 테마·글자크기·워크스페이스·노트 템플릿·AI·메타 템플릿·사용 안내·업데이트… 전부 한 화면에 평평하게 나열. Apple 인쇄 대화상자처럼: **테마·워크스페이스·노트 템플릿 3개만 기본 노출**, 나머지 전부(AI, 메타 템플릿 관리, 업데이트 내역, 사용 안내)는 "고급 설정 보기" 한 번 눌러야 나오는 아래쪽 확장 영역으로.
- 이게 지난 IA 작업의 "AI 설정을 없앨지 말지" 논쟁 자체를 무의미하게 만든다 — 없애는 대신 **기본적으로 안 보이게** 접으면 되고, AI 기능이 실제로 나오면 그때 이 화면 밖(툴바 등 실제 쓰는 자리)으로 승격시키면 된다.

### 3-5. 통합 시트 컴포넌트 (토스 바텀시트 패턴)

> **축소판 구현 완료 (2026-07-10)** — 아래 §5 리스크에서 이미 "엔지니어링 규모가 크다"고 짚은
> 대로, 8개 컴포넌트를 하나의 `Sheet` 프리미티브로 재구현하는 전면 리팩터는 하지 않았다. 대신
> **공통 모션·그림자 토큰화**만 진행: `--shadow-panel`(대형 모달 3종 중복 제거), `.meta-add-menu`를
> `--shadow-2`+`modalPop`으로 정렬, `.ctx-menu`의 독립 `ctxIn` 애니메이션을 폐기하고 공유
> `modalPop`+`--spring`으로 교체. CommandPalette/QuickOpen/GlobalSearch/SchedulePopover/
> NodePicker는 감사 결과 이미 `--shadow-2`/`--spring` 계열을 쓰고 있어 추가 변경 없음. 전면
> `Sheet` 프리미티브화는 여전히 별도 exec-plan이 필요한 고위험 항목으로 남겨둔다.

- 지금 뜨는 플로팅 UI를 세어보면: 우클릭 메뉴, SchedulePopover, NodePicker, MetaAddButton 드롭다운, TemplateAddButton 드롭다운, CommandPalette, QuickOpen, GlobalSearch — **8개의 서로 다른 구현**이 각자 다른 위치 계산·모션·배경을 갖고 있다. 토스가 "가벼운 동작 → 부모 화면으로 복귀"를 전부 바텀시트 하나의 컴포넌트로 통일하듯, 이것들을 **하나의 `Sheet` 프리미티브**(트리거 기준 위치 계산, 동일한 12ms~120ms 등장 모션, 동일한 배경·그림자)로 재구현한다. 사용자에게는 "이 앱은 뭘 열어도 똑같이 느껴진다"는 일관성으로 나타난다.

### 3-6. 마이크로카피 전수 감사 (T1, T3, T4, T5)

> **부분 구현 완료 (2026-07-10)** — 용어 교체("메타+"→"양식+", "메타 템플릿"→"정보 양식")는
> 원래 1단계 범위에서 제외였으나, 앱이 아직 v0.7.x·소수 사용자 단계라 재학습 비용이 낮다고
> 판단해 진행. 에러 메시지 전수 재작성·작은 축하 문구 추가는 **아직 미착수**(다음 라운드 후보).

- "메타+"/"메타 블록"/"메타 템플릿" → 내부 구현 용어. 사용자에게 "정보 양식"/"노트 속성" 같은 결과 중심 언어로 교체 검토(단, 기존 사용자 학습 비용 있음 — §5 리스크 참조).
- 에러 메시지 전수 재작성: "실패했습니다" 류를 전부 "무엇이 문제였는지 + 다음에 뭘 하면 되는지" 2문장 템플릿으로.
- 파괴적 동작(삭제·동기화 해제)의 확인 문구는 **그대로 명확하게 유지**(위 토스 그늘 참고 — 여긴 쉽게 만들지 않는다).
- 작은 감정적 순간 추가: 집중 세션 종료, 첫 리마인더 동기화 성공 같은 "성취" 지점에 담백한 축하 한 줄(토스의 "노력이 보상받는 순간" 패턴) — 절제된 톤 유지(앱의 기존 "모션 없거나 ≤120ms" 원칙과 같은 절제).

### 3-7. ⌘K를 진짜 앞문으로 (A5)

> **구현 완료 (2026-07-10)** — Quick Keys(자동 학습 ⌥1-9)만 진행. "리마인더 동기화 켜줘" 같은
> 자연어 액션 실행은 원래 앱에 없던 기능을 예시로 든 것이라 범위에서 제외(이미 있는 액션
> 커맨드들은 기존 그대로 실행 가능 — 새로 만들 "액션"이 없었음).

- 이미 노드 검색까지 확장했으니(직전 세션), 다음 단계는 **행동 실행**까지: "리마인더 동기화 켜줘", "오늘 할 일 보여줘" 같은 액션을 ⌘K 안에서 바로 실행. 반복 사용하는 명령엔 자동으로 짧은 단축 문자를 배정(Spotlight의 Quick Keys와 동일 개념).

### 3-8. 조용히 있다가 필요할 때만 — Focus 위젯 (Apple Focus mode)

> **구현 완료 (2026-07-10)** — 독킹 위젯 기본 접힘 + 클릭 확장, 휴지통·Note Template 배지를
> 조용한 점으로 교체.

- 사이드바에 항상 도킹된 집중 타이머 위젯을 "설정 한 번 → 평소엔 작은 표시만" 모델로. 진행 중일 때만 작은 점 표시, 클릭하면 펼쳐짐.
- 숫자 배지도 재점검: 휴지통·템플릿처럼 "몇 개 있는지가 중요한 정보가 아닌 곳"은 Notes의 조용한 점(●) 표시로, "오늘 할 일 3개"처럼 숫자 자체가 의미 있는 곳만 숫자 배지 유지.

## 4. 안 바꾸는 것 (명시적으로)

- **디자인 시스템(색·타이포·토큰) 전면 교체 안 함** — `OPERATING-PLAN.md`가 이미 "이미 자산 수준, 재작업 안 함"으로 못박음. 이번 재설계는 기존 3계층 색 시스템 위에서 형태·모션·카피로 차별화한다.
- **파일 포맷·워크스페이스 모델 안 바꿈** — 로컬 파일 소유권은 제품 정의의 핵심 기둥, 건드릴 이유 없음.
- **macOS 전용·키보드 우선 정체성 안 바꿈** — Apple 벤치마크를 쓰는 이유가 바로 이 정체성과 맞기 때문. 토스의 모바일 바텀시트는 "가벼운 동작→복귀" **원리**만 가져오고, 실제 구현은 macOS 팝오버/시트 관례(HIG)를 따른다.

## 5. 리스크 · 트레이드오프

- **자연어 날짜/태그 파싱(3-2)은 새로운 오탐 위험**을 도입한다 — "내일 회의 있음"처럼 스케줄 의도가 아닌 문장도 매칭될 수 있음. 파싱은 명확한 패턴(요일/시간/상대날짜)만 인정하고, 애매하면 그냥 텍스트로 둔다(과탐지보다 무반응이 안전).
- **전역 캡처 단축키(3-1)는 OS 권한/다른 앱과의 충돌 검토 필요** — macOS 전역 단축키 등록은 손쉬운 사용 권한이 걸릴 수 있어 별도 조사 필요.
- **마이크로카피 용어 교체(3-6)는 기존 사용자의 학습된 어휘를 바꾸는 것** — "메타+"에 이미 익숙해진 사람에게는 혼란. 점진적 전환(툴팁으로 새 이름 병기 후 전환) 검토.
- **설정 접기(3-4)는 AI 설정처럼 "찾아야 하는" 사람에게 발견성이 낮아짐** — 검색(⌘K에서 설정 항목도 검색되게)으로 보완 필요, 안 그러면 §1의 Apple System Settings 사례(검색에 의존하게 만든 실패)를 반복.
- **통합 시트 컴포넌트(3-5)는 엔지니어링 규모가 크다** — 8개 플로팅 UI를 하나로 리팩터링하는 건 이번 재설계에서 가장 위험도 높은 항목, 별도 exec-plan 필요.

## 6. 단계별 로드맵 (제안)

- **1단계 (2주, 저위험)**: 3-6 마이크로카피 감사(용어 교체는 제외, 톤/에러 메시지만) · 3-8 위젯 축소 · 3-4 설정 접기(AI/메타 템플릿 관리를 "고급"으로)
- **2단계 (3-4주, 중위험)**: 3-3 Smart Views 통합 · 3-7 ⌘K 액션 실행 확장
- **3단계 (설계 선행 필요, 고위험)**: 3-1 전역 캡처(OS 권한 조사 먼저) · 3-2 자연어 파싱(오탐 방지 룰 먼저 설계) · 3-5 통합 시트 컴포넌트(exec-plan 필수)
- 각 단계 종료 시 실사용 확인(`make dev-safe`) + 최소 1개 UX 감사 라운드.
