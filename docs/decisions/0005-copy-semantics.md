# 0005. 복사 의미론 — 내용은 전부, 연동 정체성은 제외

날짜: 2026-06-11 · 상태: 채택 (제품 오너 결정)

## 맥락

⌘C/⌘V가 다중 링크(`links[]`)와 일정(`scheduled`/`scheduleAt`)을 버리고 있었다. "미리알림 중복 방지를 위한 의도 아닌가"라는 질문이 나왔고, 복제(duplicateNode)는 반대로 전부 보존하고 있어 동작이 갈라져 있었다.

## 결정

**노드의 사본(복사·복제 공통)은 보이는 내용을 전부 가져가되, 미리알림 연동 정체성(`reminderOn`/`reminderId`)은 절대 가져가지 않는다.**

- 일정 날짜(`scheduleAt`)는 내용이다 — 복사된다. 동기화 엔진은 `reminderOn`인 노드만 건드리므로 날짜 복사로 리마인더가 중복 생성될 일은 없다(사본에서 토글을 켜야 새 리마인더 발급).
- 중복 방지는 연동 필드 제외만으로 완전하다 — 날짜를 버리는 것은 중복 방지에 기여하지 않으면서 사용자에게는 조용한 데이터 손실로 보인다.

## 결과

클립보드와 복제의 의미론이 통일됨. 사본이 늘면 '오늘' 칩이 여러 개 보일 수 있으나, 이는 사용자가 만든 사본의 정직한 상태다.
