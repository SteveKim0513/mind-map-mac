# 0003. 노트는 독립 Markdown 파일 + 명시적 노드 링크

날짜: 2026-06-10 (소급 기록) · 상태: 채택

## 맥락

노트를 (a) 노드에 내장된 필드로 둘지, (b) 독립 파일로 두고 노드와 연결할지 선택해야 했다. 내장하면 구현이 단순하지만 노트가 맵에 종속되고, 다른 도구로 열 수 없다.

## 결정

노트는 **독립 `.md` 파일**(프론트매터 id/title/links + 순수 Markdown)이고, 노드와의 관계는 **명시적 링크**(`links[]`의 `{mapId, nodeId}`)다. 노드에서 만든 노트는 `.notes/` 숨김 폴더에 위성으로 두되 일반 노트로 승격 가능. 노드 쪽에는 칩 → 프리뷰 팝업으로 표시.

## 결과

"데이터 소유" 원칙과 일치 — 노트는 어떤 Markdown 에디터로도 열린다. 대신 링크 무결성을 우리가 관리해야 하며, 현재 노트 삭제 시 죽은 링크가 남는 문제가 미해결(인벤토리 §7 한계 참고).
