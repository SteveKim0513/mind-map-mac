# 0014. 완료·일정·집중은 "할 일(todo) 노드" 전용 — 일반 노드는 순수 생각

날짜: 2026-07-16 · 상태: 채택 (결정 0011 §4 실행 그룹 개정)

## 맥락 — 어떤 문제/선택지가 있었나

마인드맵의 기본은 브레인스토밍(생각 생성)이다. 그런데 `done`이 **아무 노드에나** 붙어 실행 흐름 밖에
떠 있었고(사용자 지적: "done이 좀 따로 있는 느낌"), [결정 0011](0011-three-axis-thinking-model.md)은
완료→일정→집중을 "하나의 흐름"으로 규정하지만 UI에선 완료가 모든 노드에 노출돼 그 흐름과 어긋났다.
노드 메뉴도 실행 그룹 때문에 무거웠다.

## 결정 — 무엇을 택했나

노드에 명시적 **`todo` 상태**를 도입하고, **완료·일정·집중은 "할 일(todo) 노드"에서만 노출**한다.

- **일반 노드(생각)**: 정리·구조·통찰만. 완료/일정/집중은 **아예 숨김(비활성 아님)**. 실행 입구는
  "할 일로 전환" 하나.
- **할 일 노드**: 완료(체크박스 글리프)·일정·(일정 있으면)집중. 스타일로 시각 구분.
- **일정을 잡으면 자동으로 할 일이 된다**(일정은 곧 실행) — `@내일 3시` 자연어·캘린더 캡처·수동 일정 모두.
- **되돌리기(할 일→일반)**: 완료·일정·리마인더를 정리(반드시 `reminderOn/reminderId` 불변조건 경유).
- 결정 0011의 "실행" 그룹을 이 규칙으로 개정 — 실행은 todo 노드에서만 보인다. (정리·구조·통찰은 그대로 모든 노드.)

## 결과 — 트레이드오프, 다시 볼 조건

- 장점: 일반 노드 메뉴가 대폭 간결(진행적 공개, 원칙 3). "우리는 todo 앱이 아니다"를 **더 강하게** —
  실행은 의도적 옵트인, 지도는 순수 생각 도구로 남음. `done`의 "붕 뜬" 문제 해소.
- 트레이드오프: 브레인스토밍 중 아무 항목이나 즉시 체크하던 용법은 "할 일로 전환"(1제스처: ⌘Enter 또는
  메뉴) 후 가능. **의도적 경계**라 수용(체크한다는 것 자체가 "실행하기로 결정"한 순간).
- 데이터: `todo?: boolean` 선택 필드, **`MindMapDoc.version` 무범프**([0012](0012-node-duration-field.md) 선례).
  로드 시 `done || scheduled || reminderOn`인 노드를 todo로 backfill.
- 다시 볼 조건: 사용자가 경계를 번거로워하면 "체크=자동 승격"으로 완화 가능(이 결정 갱신).
