# 0011 — 기능의 3가지 축: 정리·실행·통찰 (0007 G3 보강)

> 날짜: 2026-07-15 · 상태: 채택 · 관련: [0007-focus-positioning](0007-focus-positioning.md)(보강, 대체 아님),
> [0006-focus-session-and-rename](0006-focus-session-and-rename.md), [PRODUCT-DEFINITION](../product/PRODUCT-DEFINITION.md)

## 맥락

`docs/product/PRODUCT-DEFINITION.md`의 "핵심 경험 3가지"(속도·통합·소유)는 제품이 **어떻게** 좋아야
하는지(경험 품질)를 말한다. 그런데 지금 UI는 기능을 **무엇을 위한 것인지**(사고 과정에서의 역할) 기준
으로 묶지 않고, "속성"/"첨부"/"일정 · 집중" 같은 표면적 유사성(둘 다 팝오버다, 둘 다 노드에 뭔가를
붙인다)으로 묶어왔다. 그 결과 완료(done)는 "속성"에, 일정·집중은 "일정 · 집중"에 갈라져 있었고,
집중 세션은 색·아이콘·메모·자식추가·삭제와 나란히 선택 툴바에 떠 있는 하나의 독립 아이콘이었다 —
"이걸 왜 쓰는가"에 대한 답이 화면 구조에 전혀 드러나지 않았다.

사용자가 제시한 재정의: **이 도구의 핵심은 정리 · 실행 · 통찰 세 축이고, 집중 모드는 "실행" 축 안에서
일정이 지정된 노드를 실제로 해낼 때 도와주는 모드**로 다시 자리잡는다. 이건 새로 여는 질문이 아니라 —
0007 시점엔 이 관계 자체가 논의된 적이 없었다(공백). 다만 0007의 가드레일 G3("집중 기능을 꺼도
제품이 온전한가")는 이 재정의와 함께 갈 수 있는지 먼저 확인해야 한다.

## 결정

### 1. 세 축을 제품의 기능 분류 기준으로 채택한다

| 축 | 정의 | 답하는 질문 |
|---|---|---|
| **정리 (Organize)** | 생각을 포착하고 구조화한다 | "이게 뭐고, 다른 것과 어떻게 연결돼 있나?" |
| **실행 (Execute)** | 정리된 것 중 해야 할 일을 실제로 해낸다 | "언제 할 거고, 하는 동안 뭐가 도와주나, 끝났나?" |
| **통찰 (Insight)** | 더 깊이 쓰고, 되짚어 이해한다 | "이게 무슨 의미고, 지금까지 뭘 해왔나?" |

세 축은 순환한다: **정리 → 실행 → 통찰 → (다시) 정리**. 노드를 적고 구조화하다(정리) 그중 할 일이
생기면 일정을 걸고(실행 시작) 때가 되면 집중 세션으로 실제로 해내고(실행), 끝나면 노트에 깊이
기록하거나 대시보드로 되짚어보고(통찰), 그 통찰이 다시 새로운 정리로 이어진다.

### 2. 집중 세션은 "실행" 축에 귀속되고, 그 시작 지점은 일정이다

집중 세션의 개념적 출발점을 **일정이 지정된 노드**로 정의한다 — "완료해야 할 일 → 언제 할지 정함
(일정) → 그때 실제로 하는 걸 도와줌(집중) → 완료 체크"가 하나의 흐름이다. 완료(done) 토글도 이
흐름의 마지막 단계이므로 "실행" 축으로 옮긴다(기존엔 "속성" 그룹에 색·아이콘과 함께 있었음).

### 3. G3는 보강되어 유지된다 — 완전 게이트는 아니다

0007 G3("집중 기능을 꺼도 제품이 온전한가")는 그대로 유지한다. **일정 없는 노드에서도 집중 세션은
여전히 기술적으로 가능하다** — 강제 게이트(일정 없으면 버튼 비활성화)는 걸지 않는다. 대신:

- 집중 세션의 **1차 진입 경로**는 일정 흐름 안(SchedulePopover의 "지금 집중 시작")으로 옮긴다.
- 선택 툴바의 독립 "집중" 아이콘처럼, 일정과 무관하게 떠 있는 진입점은 줄인다 — 우클릭 메뉴의
  "실행" 그룹(백업 경로)에는 남긴다.
- 이렇게 하면 "일정 → 집중"이 **권장되는 자연스러운 경로**가 되지만, 예외(일정 없이 바로 몰입하고
  싶은 탐색성 작업)를 막지는 않는다 — G3가 요구하는 "꺼도 핵심이 안 무너진다"를 만족한다.

### 4. 노드 우클릭 메뉴를 축 기준으로 재편한다

기존 5그룹(편집/속성/첨부/일정 · 집중/보기)을 4그룹으로:

| 새 그룹 | 항목 | 축 |
|---|---|---|
| **구조** | 자식 추가, 형제 추가, 편집, 접기/펼치기, 이 노드만 보기, 이 노드로 확대, 복제, 삭제 | (축 무관 — 노드 자체의 뼈대) |
| **정리** | 색, 아이콘…, 메모 추가, 링크 추가 | 정리 |
| **실행** | 완료 표시/해제, 날짜·시간 설정…/스케줄 지정·해제, 집중 세션 시작 | 실행 |
| **통찰** | 노드에 노트 연결 | 통찰(로 들어가는 다리) |

세부 사항·전체 기능-축 매핑·선택 툴바/사이드바 재검토는 [FEATURE-HIERARCHY-VISION-2026-07.md](../product/FEATURE-HIERARCHY-VISION-2026-07.md) 참고.

## 결과 (트레이드오프 / 다시 볼 조건)

- **얻는 것**: 화면 구조가 "이 기능이 왜 여기 있는지"를 스스로 설명한다 — 완료·일정·집중이 하나의
  흐름으로 보이고, 노트 연결이 "더 깊이 들어가는 통로"로 명확해진다.
- **트레이드오프**: 카테고리가 사용자의 실제 멘탈 모델과 다르면(예: 메모가 "정리"가 아니라 "통찰"에
  더 가깝다고 느낄 수 있음) 재분류가 다시 필요할 수 있다 — 이번 분류는 사용자가 직접 제시한 축
  정의를 기준으로 한 1차 결과이며, 실사용 후 조정 대상이다.
- **다시 볼 조건**: 일정 없이 집중 세션을 시작하는 빈도가 실제로 높다면(예: 탐색성 작업이 잦다면),
  "실행 축 = 일정에서 출발" 전제 자체를 재검토한다.
