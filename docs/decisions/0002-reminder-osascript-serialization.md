# 0002. 리마인더 AppleScript 호출 직렬화 + 본문 태그 소유권

날짜: 2026-06-10 (소급 기록, 커밋 6b62ef9·c632d73) · 상태: 채택

## 맥락

macOS 리마인더 앱의 AppleScript 인터페이스는 동시 호출 시 행(hang)이 잦았고, 동기화가 멈추는 버그의 주원인이었다. 또 undo로 노드의 `reminderId`가 사라지면 고아 리마인더가 남았다.

## 결정

1. 모든 osascript 호출을 Promise 체인으로 **직렬화**(동시 1개, 기본 6초 타임아웃).
2. 텍스트 인자는 스크립트 문자열 보간 대신 **argv로 전달**(인젝션 방지).
3. 소유권은 리마인더 **본문의 `mindmap:<nodeId>` 태그**로 식별 — ID 매핑이 깨져도(undo 등) 복구 가능하고, 같은 노드의 중복 리마인더를 reaping으로 정리할 수 있다.
4. 장애는 헬스 상태 머신(ok/down/denied) + 백오프 재시도로 격리하고 UI에 표시.

## 결과

처리량은 낮아졌지만(직렬 실행) 동기화 노드 수 규모에서는 문제없음. 폴링 기반이라 리마인더 앱 쪽 변경은 최대 45초 지연 — EventKit 네이티브 연동으로 갈지는 사용자 규모가 커지면 재검토.
