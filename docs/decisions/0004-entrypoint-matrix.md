# 0004. 동작별 진입점 매트릭스 채택 + 우클릭 메뉴는 그룹 구분선(A안)

날짜: 2026-06-11 · 상태: 채택 (제품 오너 결정)

## 맥락

같은 동작의 진입점이 화면마다 다르게 배치되어 있었고, 기준이 없어 새 기능마다 버튼 위치가 논쟁거리였다. UX 감사(U4)에서 표준 부재가 확인됨.

## 결정

1. [동작별 진입점 매트릭스](../product/specs/2026-06-11-entrypoint-matrix.md)를 팀 표준으로 채택. 핵심: **우클릭 메뉴 = 그 노드에 가능한 모든 동작의 카탈로그**, 선택 툴바 = 고빈도 6±2, 팝오버 = 속성 상세, 메뉴바 = 앱·문서 수준, ⌘K = 전체의 미러. 새 동작은 매트릭스에 행을 추가하는 것이 절차.
2. 우클릭 메뉴는 **A안(그룹 구분선)**: 14항목을 유지하되 구조 / 속성(색·아이콘·완료) / 첨부(메모·링크·노트) / 일정 / 보기·기타 5개 그룹으로 구분. 서브메뉴 축소안(B안)은 hover 비용·발견성 저하로 기각.

## 결과

진입점 배치가 규칙 확인 문제로 바뀜. 메뉴는 +8px 수준으로 약간 길어짐 — 항목이 더 늘면 B안(서브메뉴)을 재검토한다.
