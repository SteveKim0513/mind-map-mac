# 0006. 집중 세션 도입 + "이 노드에 집중" → "이 노드만 보기" 개명

날짜: 2026-06-12 · 상태: 채택

## 맥락

집중 세션(시간 추적) 기능을 추가하며 "집중"이라는 단어가, 기존 캔버스 보기 필터(`setFocus` — 그 노드 서브트리만 표시)와 충돌했다. 또 대시보드를 어떻게 띄울지(탭 vs 오버레이) 결정이 필요했다.

## 결정

1. **개명**: 캔버스 보기 필터의 라벨을 "이 노드에 집중" → **"이 노드만 보기"**로 변경(동작·내부명 `setFocus`는 유지). "집중"은 더 의미 있는 작업-집중 기능에 양보. 새 기능 메뉴는 "집중 세션 시작".
2. **대시보드 = 오버레이**(탭 아님): 탭 시스템이 전부 경로 기반(persist/hydrate/중복판정)이라 경로 없는 'history' 탭은 분기가 번지고 복원·중복 위험이 있었다(검토 P1-3). 검색·커맨드 팔레트와 같은 전역 오버레이로 구현 → 더 단순·견고. `uiStore.historyOpen`.
3. **세션 데이터 = 노트 프론트매터의 `session`**(epoch ms 시간) + 워크스페이스 인덱스가 이를 실어 대시보드가 메모리 집계. 별도 DB 없음. `sessionId`로 중복 방지, 깨진 세션은 누락 대신 노출.
4. **세션 노트 격리**: 일반 노트의 자동 effect(제목→파일명 동기화, 800ms 자동저장)에서 분리 — rename 제외·제목 읽기전용, 종료 write는 열린 탭 store도 동기화해 경합 방지.

## 결과

명세 §14(3관점 비판 검토)의 필수 보강을 v1에 반영. 시간은 절전/DST에 견디는 epoch 저장 + sanity 보정. 서브트리 롤업으로 "주제 단위" 집계가 가능해 마인드맵이 노력 히트맵이 된다. 상세: [specs/2026-06-12-focus-session.md](../product/specs/2026-06-12-focus-session.md).
