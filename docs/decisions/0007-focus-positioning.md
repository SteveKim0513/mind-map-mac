# 0007 — 집중/대시보드의 위치: "사적인 거울 + 의도된 보고서"

> 날짜: 2026-06-12 · 상태: 채택 · 관련: [REVIEW-2026-06-focus-and-dashboard](../product/reports/REVIEW-2026-06-focus-and-dashboard.md), [work-dashboard-redesign](../product/specs/2026-06-12-work-dashboard-redesign.md)

## 맥락

집중 세션·작업 기록 대시보드는 "관리자에게 보여주고 평가받는 기록"이라는 맥락을 갖는다. 이는 제품의 정체성(*사적인 사고 도구* — 소유·속도·통합)과 긴장한다. 평소 화면이 "잘 보이게" 최적화되면 측정되지 않는 사고(탐색·브레인스토밍)가 위축된다. 개별 후속 결정(유휴 자동 추적 → 자동 점수화 → 알림 압박)은 각자 합리적이라 사후에 막을 명분이 약하다.

## 결정

**제품의 무게중심은 "사적인 사고 도구"로 고정한다. 관리자에게 보여주는 것은 *모드*가 아니라, 사용자가 의도적으로 만들어 내보내는 *한 장의 산출물*이다.**

두 순간을 분리한다:

- **평소(99%)**: 대시보드 = 나만 보는 거울. 지지적·낮은 압박 톤. 앱은 아무 데도 전송하지 않는다.
- **보고(의도적)**: "요약 노트로 내보내기"로 만든 한 장. 그냥 내 `.md` 파일이라 보여주기 전에 내가 검토·편집한다. 평가로 가는 **유일한 다리는 내보내기 한 번**뿐.

## 가드레일 (강제 조항)

- **G1 — 추적은 영구히 명시적 opt-in.** 자동/유휴 시간 추적·백그라운드 측정을 추가하지 않는다. 사용자가 "시작"을 누른 세션만 존재한다.
- **G2 — 대시보드 톤 = 정직한 객관 리포트.** 공백·미달을 숨기지 않는다. 게임화(streak 등)는 압박형 표현을 쓰지 않고, 신뢰성을 해치지 않는 보조로만. (예: "🔥 연속"이 아니라 "최근 7일 중 집중일".)
- **G3 — "집중 기능을 꺼도 제품이 온전한가"가 리트머스.** 집중/대시보드는 부가 축이며 핵심(생각·기록·할 일)을 종속시키지 않는다. 새 기능 리뷰의 1번 질문.

## 만들지 않는 것 (이 결정의 직접 귀결)

관리자 전용 뷰 · 라이브 공유 · 팀 리더보드 · 자동 점수 · "상사가 봅니다" 류 압박 프레이밍 · 자동/유휴 추적.

## 귀결 (구현)

- ~~"요약 노트로 내보내기"를 1급 산출물로~~ → **보강(2026-06-12)**: 정적 내보내기 노트를 **제거**. 대시보드 화면 자체가 이미 신뢰할 수 있는 기록이라 별도 정적 덤프는 잉여였다. "보여줄 산출물"은 *살아있는 화면*(대시보드 + 신설 "오늘" 계획 뷰)으로 충족 → 계획→실행(집중)→회고의 한 루프. 명세: [specs/2026-06-12-today-view](../product/specs/2026-06-12-today-view.md).
