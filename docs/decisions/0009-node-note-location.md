# 0009 — 노드에서 만든 노트는 그 노드 맵과 같은 폴더에 (숨김 .notes/ 폐기)

> 날짜: 2026-06-14 · 상태: 채택 · 대체: 노드 노트를 숨김 `.notes/`에 두던 기존 동작

## 맥락

노드의 "노트 추가·연결" → "새 노트 만들어 연결"은 노트를 **숨김 `.notes/` 폴더**에 만들었다(트리 스캐너가 `.`로 시작하는 항목을 건너뛰므로 사이드바에 안 보임, 노드 칩으로만 접근). 의도는 "사이드바를 어지럽히지 않기"였으나, 사용자 기대와 반대였다 — **"사이드바에서 안 보여야 하는 건 집중(work-log) 노트뿐"**.

## 결정

1. **노드에서 만든 노트는 그 노드 *맵과 같은 폴더*에 생성한다.** (맵 경로의 디렉터리; 없으면 워크스페이스 루트.) 일반 노트처럼 사이드바에 보인다.
   - 그냥 "새 노트"(사이드바)는 종전대로 현재 위치에 생성.
2. **사이드바에서 숨기는 건 집중 세션 노트(`work-log/`)뿐.** 다른 어떤 노트도 숨기지 않는다. work-log 숨김은 *이름*이 아니라 *루트의 그 폴더 경로*로 판정(사용자가 다른 곳에 `work-log` 폴더를 만들어도 보인다).
3. **레거시 이전**: 기존 `.notes/`에 있던 노드 노트는 워크스페이스 로드 시 **루트로 자동 이동**(보이게). 이동 실패해도 인덱스에는 남아 노드 칩으로 접근 가능.

## 결과

- 노드 노트가 사이드바에 보이고, 위치 규칙이 직관적(노드 노트 = 맵 폴더, 일반 = 그 자리).
- `.notes/` 인프라(attached:list IPC, 인덱스 병합, promote 버튼)는 레거시 호환용으로만 남음(이전 후 빈 폴더 → 무동작).
- 향후 `.notes/` 의존 코드는 제거 후보.
