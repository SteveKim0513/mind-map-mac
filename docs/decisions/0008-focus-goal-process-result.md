# 0008 — 집중 세션: 목표·과정·결과를 프로세스에서 구조화 포착 (노트 템플릿 폐기)

> 날짜: 2026-06-12 · 상태: 채택 · 대체: [specs/2026-06-12-session-note-placeholders](../product/specs/2026-06-12-session-note-placeholders.md)(폐기), [0006](0006-focus-session-and-rename.md) 보강

## 맥락

초기 집중 세션은 세션 노트에 4단 템플릿(🎯 목표 / ✅ 완료정의 / 🔨 기록 / 🅿️ 파킹랏)을 깔고, **목표를 본문에서 파싱**(`extractGoal`)했다. 문제:

- 목표가 *선택사항이자 노트 섹션에 묻혀* 대부분 비워짐 → 완료 카드의 "목표 대비 성과", 대시보드의 "마감 vs 실제"가 빈 깡통이 됨. 기능 가치의 전제가 무너짐.
- 본문 파싱은 취약(이탤릭 마커 `_`↔`*` 불일치로 자리표시자 누출 버그 — 검증에서 발견·수정한 바 있음).
- 템플릿 4단은 빠른 세션엔 "숙제"처럼 무거움.

## 결정

목표·과정·결과를 **노트 템플릿이 아니라 프로세스의 세 지점에서 구조화 데이터로** 포착한다:

- **목표(goal)** — 세션 *시작 시 팝업* 한 줄 입력 → `session.goal`. 선택(비우고 시작 가능), Esc 취소. 진입점(우클릭·선택 툴바)은 이 팝업을 연다.
- **과정(process)** — 세션 노트는 **자유 작업 로그**. 템플릿 없음, 빈 본문 + 가이드 placeholder("이 세션에서 한 일을 기록…"). 진행 시간·기록이 곧 과정.
- **결과(result)** — *종료 시 완료 카드*의 한 줄 회고 → `session.reflect`.

목표는 노트 배너에 읽기전용으로 표시돼 과정 내내 보인다. 완료 카드는 `🎯 목표 → 성과` 대비를 띄운다.

## 결과 (트레이드오프 / 다시 볼 조건)

- **단순+강력**: 시작은 Enter 한 번(즉시성 유지), 목표는 기본적으로 잡혀 측정 루프가 항상 작동.
- **코드 단순화**: `extractGoal`/`goal.ts`, 노트 4단 템플릿(`sessionNote` SECTIONS), 자리표시자 데코레이션(`sessionHints.ts`)을 **제거**. 파싱 취약성 소멸.
- **완료 카드의 "세션 노트 열기" 버튼 제거** — 세션 중 노트는 이미 열려 있어 군더더기였다.
- 폐기: "과정"의 정량 측정은 현재 시간·로그 유무 수준. 더 구조화된 과정 측정(체크포인트 등)이 필요해지면 재검토.
