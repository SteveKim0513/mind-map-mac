핵심 원칙 3-레이어 분리: ① 중립(구조) · ② 의미/상태(예약·아이콘 동반) · ③ 장식 태그(사용자 선택). 같은 색이 "스케줄"도 됐다가 "태그"도 되던 혼선을 끊고, 태그 8색을 색상환에 균등 배치해 9px 점에서도 구분되게 했습니다. 우상단 버튼으로 다크/라이트를 직접 비교해 보세요.

현행의 문제 (Before)

· 하늘(#62aef0) ↔ 브랜드 파랑(#0075de) 충돌 → 태그가 "선택됨"처럼 보임
· 보라(#d6b6f6) 너무 옅어 9px 점에서 "색 없음" 회색과 구분 안 됨
· 주황·초록·핑크가 스케줄·완료·검색 상태색과 동일 풀 → 의미 충돌
· 6색 채도/명도 제각각 + 색상환 불균등 + 다크모드 변형 없음 + 생 hex가 4곳 중복

실제 피커 모양 · 실제 크기 (18px 점):

파랑 계열을 전부 비움 → 파랑은 오직 브랜드(선택/포커스) 전용이라, 태그 단 노드가 절대 "선택됨"으로 안 보임. 8색 모두 명도 L≈55–62%로 통일해 한 가족으로 읽히고, 다크모드에선 밝기↑·채도↓ 변형으로 자동 전환.

적용 범위 · 코드 정리

· 4곳에 중복된 생 hex 배열 → src/theme/palette.ts 한 곳에서 export, 전 컴포넌트가 import
· 죽은 --accent-brown 제거, --accent-sky/purple 등 토큰명을 --tag-*로 정규화
· 의미색은 --state-*로 분리해 태그 풀과 물리적으로 격리
· 다크모드: 모든 레이어에 변형 추가 (현재는 구조 토큰만 존재)
