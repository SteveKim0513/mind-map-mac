# UI 디자인 원칙

> 모든 UI 작업 전 이 문서를 읽는다. 디자인 결정이 원칙과 충돌하면 원칙이 우선한다.
> 새 UI 컴포넌트·스타일 작업 시 `/design-ui` 스킬을 먼저 실행한다.

## 핵심 철학

**Simple, Powerful, Intuitive** — 기본은 단순하게, 기능은 강력하게, 쓰는 법은 자명하게.

1. **화면에 보이는 모든 것은 이유가 있어야 한다.** 제거해도 사용자가 잃는 게 없다면 제거한다.
2. **상태는 1초 안에 읽혀야 한다.** 무엇이 활성이고 비활성인지, 저장됐는지 아닌지를 색·아이콘·위치로 즉시 알 수 있어야 한다.
3. **첫 인상은 심플, 깊이 파면 강력해야 한다.** 복잡한 기능은 필요할 때만 드러낸다.
4. **macOS 네이티브 느낌.** Apple HIG 패턴을 따른다. 키보드로 모든 작업이 가능해야 한다.
5. **패턴은 반복되어야 한다.** 같은 역할의 요소는 앱 전체에서 같은 모양이어야 한다.

## 시각 언어

### 계층 구조

```
제목       14-15px  font-weight: 600  --ink
본문 라벨  13-13.5px font-weight: 400  --ink-secondary
보조 텍스트 12-12.5px                  --ink-muted
힌트·메타  11-12px                    --ink-faint
최소 폰트  11px  (이 이하는 읽기 어려워 금지)
```

### 색상

- **색상 값을 컴포넌트에 직접 하드코딩하지 않는다.** CSS 변수(`var(--primary)` 등)만 사용한다.
- 새 색상이 필요하면 `docs/design/COLOR-SYSTEM.md`의 3계층 규칙을 따른다.
- 예외 허용 목록: 활성 초록(`#34c759`), 위험 빨강(`#ff3b30`) — 이 두 값만 raw hex로 허용하며, 사유를 주석으로 남긴다.

### 간격

```
요소 간 gap        4–8px   (같은 그룹)
섹션 간 gap        12–16px (다른 그룹)
카드 내부 패딩     8–10px
모달 내부 패딩     16–20px
```

### 보더·반경

- 테두리: `var(--hairline)` (1px, 저대비) — 구분이 필요할 때만.
- 활성 상태의 테두리: `color-mix(in srgb, var(--primary) 30–40%, var(--hairline))`.
- 반경: `var(--r-md)` (8px) — 카드, 입력, 버튼. 작은 요소는 `var(--r-sm)` (4–6px).

## 인터랙션 원칙

### 버튼

| 용도 | 스타일 |
|---|---|
| 주요 확정 동작 | Primary (solid background, `var(--primary)`) |
| 보조·취소 | Ghost (transparent bg, `var(--hairline)` border) |
| 파괴적 동작(삭제) | Ghost + hover → `#ff3b30` (danger variant) |
| 링크처럼 보여야 하는 | Link-style button (background: none, color: `var(--primary)`) |

- 버튼 높이: 일반 32px, 컴팩트 26px (인라인/카드 내부).
- 버튼 간격: 4–6px.

### 입력 필드

- 기본: `var(--hairline)` 테두리, `var(--canvas-soft)` 배경.
- 포커스: `var(--primary)` 테두리.
- 에러: `#ff3b30` 테두리 + 인라인 에러 텍스트 (12px, `#ff3b30`).
- 에러 메시지는 팝업이 아닌 필드 바로 아래에 표시한다.

### 상태 표시

- 활성(enabled) 상태: `var(--state-done)` 또는 `#34c759` 초록 점 (● 7px).
- 비활성: 회색 점 (`var(--ink-faint)`).
- 선택(radio/toggle): 채워진 원(`●`) 대 빈 원(`○`), `var(--primary)` 색상.
- 로딩: 스피너 대신 버튼 비활성화 + 텍스트 변경("저장 중…").

## 정보 밀도

- **한 화면에서 스크롤 없이 핵심 정보 전달을 목표로 한다.**
- 설정 모달은 380–420px 폭, 최대 높이 80vh.
- 카드는 한 줄(32px 내외)을 기본으로, 편집 시에만 확장한다.
- 힌트 텍스트는 필요할 때만 (편집 모드 진입 시 등) 노출한다.

---

## 점진적 공개 (Progressive Disclosure)

```
기본 화면    → 가장 자주 쓰는 설정만
상태 요약    → 현재 상태(활성/비활성)가 한눈에
편집 진입    → 클릭/탭으로 확장, 미사용 시 접힌 상태
고급 옵션    → 별도 섹션 또는 별도 화면
```

- 비어있는 상태(empty state)도 UI다. "아무것도 없음"이 아니라 "이렇게 시작하세요"를 보여준다.

## 하지 말 것

- `style={{ color: '#1a2b3c' }}` 같은 인라인 raw 색상.
- 동일 역할에 서로 다른 버튼 모양.
- 상태 변화 없이 모달/토스트만 띄우는 액션.
- 툴팁에 중요 정보 숨기기 (툴팁은 보조 설명용).
- 입력 검증을 저장 버튼 클릭 후에만 실행 (실시간 또는 blur 시 검증 권장).
