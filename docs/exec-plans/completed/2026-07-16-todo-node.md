# 할 일(todo) 노드 구현

날짜: 2026-07-16
상태: completed · 검증: 단위(마이그·setTodo·자동승격) + E2E 3개 통과, 시각 스크린샷 확인

명세: [specs/2026-07-16-todo-node.md](../../product/specs/2026-07-16-todo-node.md) · 결정: [0014](../../decisions/0014-todo-node.md)

## 목표
생각(일반 노드)과 실행(할 일 노드)을 `todo` 상태로 가른다. 완료·일정·집중은 할 일 노드에서만.

## 범위
포함: `todo?` 필드 + 마이그레이션, `setTodo`(전환/되돌림) + 일정 자동 승격, 메뉴·툴바·노드 게이팅, 체크박스 글리프, ⌘Enter, 테스트/E2E/문서.
제외: 우선순위·라벨·리스트, 서브트리 통째 전환, 반복 일정.

## 위험
- 되돌리기 시 리마인더 고아 → 반드시 `mapStore`의 언스케줄 경로 재사용.
- 기존 done/scheduled 노드 마이그레이션 누락 → backfill + 테스트.
- 다중 선택 완료 경로에서 todo 미승격.

## 구현 단계
- [ ] `types.ts` `todo?: boolean`
- [ ] `io/formats.ts deserialize` backfill(`done||scheduled||reminderOn`→todo) + 단위 테스트
- [ ] `mapStore` `setTodo(id,on)`(off=완료·일정·리마인더 정리) + `setScheduled(true)`/`setScheduleAt`/`captureScheduled`/`toggleDone`/자연어파싱에서 `todo=true` 승격 + 단위 테스트
- [ ] `ContextMenu` 실행 그룹 todo 전용 + "할 일로 전환"/"일반 노드로 되돌리기"
- [ ] `SelectionToolbar` 완료·집중 todo 전용 + "할 일로" 버튼(일반 노드)
- [ ] `NodeView` `todo` 클래스 + 체크박스 글리프(완료 토글), CSS
- [ ] `useKeyboard` ⌘Enter: 일반→전환, 할 일→완료 토글
- [ ] E2E `e2e/todo-node.spec.ts`
- [ ] 문서: FEATURE-INVENTORY·Manual·decisions/README·ARCHITECTURE

## 검증
- `make verify` + E2E + `make dev-safe` 스크린샷(일반 vs 할 일 노드·메뉴).

## 발견한 사실
- **`addChild`/`addSibling`의 스케줄 상속 제거**: 기존엔 스케줄 노드의 새 자식/형제가 `scheduled`를 상속했는데, todo 모델에선 "scheduled인데 todo 아님"이라는 불일치(체크박스 없이 일정 칩만) + "명시적 전환" 철학과 충돌. → 상속을 제거해 **새 자식·형제는 순수 생각(일반 노드)**. (테스트 없어 안전.) 기존 서브트리로의 `setScheduled` 캐스케이드는 유지(그쪽은 todo도 함께 세팅).
- 시각 검증: 할 일 노드=제목 앞 체크박스(완료 시 초록 채움+취소선)+일정 칩, 일반 노드=체크박스·칩 없음. 일반 노드 메뉴에서 완료·일정·집중이 사라지고 "할 일로 전환"만 남음(스크린샷 확인).
