# 노트 메타 블록 기능
날짜: 2026-07-08
상태: active

## 목표

노트 상단에 구조화된 속성(메타) 블록을 추가하는 기능.
사용자가 전역 템플릿을 만들고, 노트에서 선택해 최상단에 자동 삽입한다.

---

## 사용자 여정

### A. 템플릿 만들기 (설정)
```
설정(⌘,) → "메타 템플릿" 탭
→ "+ 새 템플릿" 버튼
→ 템플릿 이름 입력 (예: "프로젝트 정보")
→ 필드 추가: 이름 + 타입 선택
→ 저장
```

**필드 타입**
| 타입 | 예시 |
|---|---|
| 텍스트 | 담당자, 브랜드명 |
| 날짜 | 시작일, 마감일 |
| 선택 | 상태 (진행 중 / 완료 / 보류) |
| URL | 참고 링크 |
| 숫자 | 우선순위, 예산 |

### B. 노트에 삽입
```
노트 편집 중 → 툴바 "메타 +" 버튼 클릭
→ 등록된 템플릿 목록 드롭다운
→ 클릭 → 노트 최상단에 메타 블록 삽입
   (커서 위치 무관, 기존 내용 아래로 밀림)
```

### C. 메타 값 편집
```
메타 블록의 필드 값 클릭 → 인라인 편집
Tab: 다음 필드 이동
Enter / Esc: 확정·취소
```

### D. 메타 블록 삭제
```
메타 블록 우측 상단 "×" 버튼 → 블록 삭제
(노트 본문은 유지)
```

---

## UI 설계 원칙

1. **최상단 고정** — 메타 블록은 항상 노트의 제일 위. 드래그로 이동 불가.
2. **컴팩트 기본** — 기본은 테이블 형태 2열(라벨 | 값). 클릭 시 인라인 편집.
3. **점진적 공개** — 빈 값은 흐릿하게("클릭해서 입력"). 편집 진입 시 입력란 활성화.
4. **여러 블록 허용** — 템플릿이 다르면 여러 개 추가 가능. 같은 템플릿 중복 추가 시 확인 토스트.
5. **본문과 시각적 구분** — 옅은 배경(`var(--canvas-soft)`) + 하단 구분선.

```
┌─────────────────────────────────────────┐
│ 📋 프로젝트 정보              [×]       │  ← 템플릿명 + 닫기
├──────────────┬──────────────────────────┤
│ 담당자       │ 김지호                   │
│ 시작일       │ 2026-07-08               │
│ 상태         │ 진행 중 ▾                │
│ 참고 링크    │ https://...              │
└──────────────┴──────────────────────────┘
```

## 저장 방식

### 템플릿 저장
- 위치: `{userData}/meta-templates.json`
- IPC: `meta:getTemplates` / `meta:saveTemplates`
- Store: `src/store/metaStore.ts`

### 노트 내 메타값 저장
- 마크다운 파일 상단 YAML 프론트매터로 저장:
  ```markdown
  ---
  _meta:
    - template: 프로젝트 정보
      fields:
        담당자: 김지호
        시작일: "2026-07-08"
        상태: 진행 중
  ---

  (노트 본문)
  ```
- Tiptap에서 `MetaBlock` 커스텀 노드로 렌더링
- 저장/불러오기 시 `src/io/noteFormat.ts`에서 프론트매터 파싱/직렬화

## 가정

- 템플릿은 앱 전역 공유 (워크스페이스별 분리 없음 — 1차)
- 프론트매터 `_meta` 키는 앱 전용 (사용자가 직접 편집 가능하나 앱이 보장)
- Tiptap에서 최상단 삽입은 `insertContentAt(0, ...)` 로 구현 가능

## 위험

| 위험 | 대응 |
|---|---|
| 기존 프론트매터가 있는 `.md` 파일 충돌 | `_meta` 키만 파싱, 나머지 프론트매터 유지 |
| 프론트매터 파싱 실패 시 노트 손상 | 파싱 실패 → 원본 그대로 표시, 에러 토스트 |
| Tiptap 최상단 고정 복잡도 | `MetaBlock`을 doc의 첫 번째 child-only node로 제한 |
